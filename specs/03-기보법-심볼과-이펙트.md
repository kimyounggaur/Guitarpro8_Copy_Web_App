# 도메인 사양서: 기보법 심볼과 이펙트 표기 (Notation Symbols & Effects)

> Guitar Pro 8 클론 웹앱 설계의 단일 기준 문서(Single Source of Truth).
> 범위: 마디 레벨 심볼, 노트 레벨 심볼, 기타 이펙트 표기, 가사, 코드 이름/다이어그램 배치, 스타일시트(Stylesheet) 시스템, 표기 디자인 옵션.
> 출처: Guitar Pro 8 공식 매뉴얼 (334p) 중 주로 p.101~167(Symbols/Extras), p.162~174(Repeats & Directions), p.324~334(단축키 부록), 신기능 소개(p.20~40).

## 0. 공통 UI 원칙 (Edition Palette 인터랙션 모델)

- **에디션 팔레트(Edition Palette)**: 메인 화면 왼쪽 패널. 표준 악보/타브(tablature)/슬래시(slash) 기보에 필요한 **모든 심볼 버튼**이 모여 있음. 팔레트 표시/숨김 단축키: `F2`.
- 팔레트 그룹 순서(10개): ① 현재 편집 보이스, ② 멀티보이스, ③ 디자인 모드, ④ 가사, ⑤ 코드, ⑥ **Bar symbols**, ⑦ **Note symbols**, ⑧ **Effect symbols**, ⑨ **Notation symbols**, ⑩ Automation symbols.
- **인터랙션 모델**: 버튼 클릭 → 현재 선택된 노트(들)에 해당 속성이 즉시 적용(토글). 추가 설정이 필요한 심볼은 클릭 시 **전용 다이얼로그**가 열림.
- **아이콘 우하단의 작은 삼각형(▸)**: 해당 아이콘에 **우클릭 또는 롱클릭**으로 접근 가능한 추가 설정이 있음을 표시(예: n-tuplet 비율 선택, 트레몰로 피킹 음가 선택, 비브라토 진폭).
- **대부분의 심볼은 재생(playback)에 반영됨** — 오디오 엔진이 심볼을 해석. 예외(표기 전용)는 §4.34 요약표 참조.
- 악보 위의 클릭 가능한 요소는 **마우스오버 시 파란색 하이라이트**로 표시되고, 클릭하면 메뉴/팔레트와 동일한 편집 다이얼로그가 열림. 본 도메인 관련 클릭 가능 요소: 페르마타, 디렉션(점프/타깃), 얼터네이트 엔딩, Repeat close, 코드, 프리 텍스트, 스윙 필(triplet feel).
- 왼손잡이 사용자를 위한 설명 규칙: 매뉴얼의 "오른손/왼손"은 왼손잡이 연주자의 경우 반대로 적용.
- 오류 표시: 박자표(time signature) 기준으로 불완전/초과 마디는 **빨간색**으로 렌더링(편집 중인 마디 제외). `Bar > Anacrusis`(못갖춘마디) 체크 시 첫/마지막 마디는 불완전해도 빨간색 표시 제외.

**웹앱 구현 시사점**: 팔레트는 "선택 → 속성 토글" 커맨드 디스패처. 각 심볼은 note/beat/bar/masterBar 중 하나의 엔티티에 붙는 속성(enum/struct)으로 모델링하고, 렌더러와 오디오 스케줄러가 동일 속성을 각각 해석하는 구조가 필요.

## 1. 마디(Bar) 레벨 심볼

팔레트 "Bar symbols" 그룹 전체 목록: Clef, Key Signature, Time Signature, Triplet Feel, Free time bars, Double bar-lines, Simple repeat sign, Double repeat sign, Repeat open, Repeat close, Repetitions(alternate endings), Directions, 8va, 8vb, 15ma, 15mb.
(Clef/Key/Time signature는 악보 구조 도메인에서 상세 기술 — 여기서는 마디 심볼 팔레트 구성 요소로만 명시. 관련 단축키: Time Signature `Ctrl+T`, Key Signature `Ctrl+K`. GP는 4개 음자리표(G, F, C3, C4) + 4개 옥타브 클레프를 지원.)

마디 레벨 속성(반복 기호, 조표, 박자표)은 **마스터 트랙 속성** — 모든 트랙에 동시에 적용됨.

### 1.1 트리플렛 필 (Triplet Feel)
- **동작**: 2분할(binary/straight)로 기보된 마디를 3분할(ternary/swing) 느낌으로 **재생만** 변경. 기보는 그대로(악보를 가볍게 유지). 여러 모티프 제공, 가장 일반적인 것은 **"triplet feel 8th"**(4/4를 블루스 셔플로 재생).
- **UI**: Time Signature 에디터 내의 전용 **Triplet feel editor**에서 모티프 선택. 악보의 스윙 필 표시를 직접 클릭해도 편집 가능.
- **단축키**: `Ctrl+/` 또는 `Ctrl+Shift+/` (macOS `⌘/`, `⇧⌘/`). 커맨드 팔레트 prefix: `triplet-feel`.
- **웹앱 시사점**: 재생 전용 변환 — 렌더링 데이터는 불변, Web Audio 스케줄러에서 8분음표(또는 선택 음가) 오프비트의 타이밍만 스윙 비율로 시프트.

### 1.2 프리 타임 마디 (Free Time Bars)
- **동작**: 리듬/템포를 자유롭게 연주하는 마디로 지정.
- **렌더링**: 해당 마디의 **바라인이 점선**으로, **박자표가 괄호 안**에 표시됨.
- **단축키**: `Alt+Shift+L` (macOS `⌥⇧L` 또는 `|`). 커맨드 팔레트 prefix: `free-time`.
- **웹앱 시사점**: masterBar에 `freeTime: boolean`. 렌더러는 점선 바라인 + 괄호 박자표; 재생 엔진은 기보 리듬을 그대로 따르되 표기 의미만 다름.

### 1.3 더블 바라인 (Double Bar-Lines)
- **동작**: 파트 변경/조·박자 변경 등 중요한 변화 지점 표시. **박자표가 바뀔 때마다 자동 삽입**되며, 팔레트 버튼으로 임의 위치에 수동 추가 가능.
- **단축키**: 커맨드 팔레트 prefix `double-barline`.
- **웹앱 시사점**: masterBar에 `doubleBar: boolean` + 박자표 변경 시 자동 세팅 로직.

### 1.4 심플 리피트 사인 (Simple repeat sign, "%", simile)
- **동작**: **직전 1개 마디를 그대로 반복**함을 나타냄(입력/가독성 경량화). **첫 마디에는 사용 불가**.
- **단축키**: `Shift+%` (macOS `%` / `⇧%`). 커맨드 팔레트 prefix: `simile`.

### 1.5 더블 리피트 사인 (Double repeat sign)
- **동작**: **직전 2개 마디**를 반복. **1~2번 마디에는 사용 불가**.
- **단축키**: `Ctrl+%` / `Ctrl+Shift+%` (macOS `⌘%` / `⇧⌘%`). 커맨드 팔레트 prefix: `double-simile`.
- **웹앱 시사점(1.4~1.5)**: masterBar 또는 bar에 `simileMark: none|single|double`. 렌더러는 % 기호만 그리며 재생 시 이전 마디 내용을 복제 재생. 유효성 검사(첫 마디/첫 두 마디 금지) 필요.

### 1.6 반복 열기 (Repeat Open)
- **동작**: 해당 마디의 **여는 바라인을 대체**(더블 바라인 + 점 2개). 이후 마디들이 Repeat Close에서 지정한 횟수만큼 반복됨. 악보 첫 마디에서는 생략 가능한 경우가 있음(암묵적 시작 반복).
- **단축키**: `[` 또는 `Shift+[`. 커맨드 팔레트 prefix: `repeat-open`.

### 1.7 반복 닫기 (Repeat Close)
- **동작**: **닫는 바라인을 대체**하고 재생을 **마지막 Repeat Open으로 되돌림**. 클릭 시 **Repeat close 다이얼로그**가 열려 **반복 횟수(passes)** 입력.
- **렌더링**: 반복 횟수 숫자가 **Repeat close 마디 위에 표시**. 단 **정확히 2회일 때는 숫자 미표시**.
- **예시**: 반복 1회 → 재생 순서 `1-2-1-2-3`. 3회 반복 → `A-B-A-B-A-B-C`.
- **고급 규칙**: **중첩 반복(nested repeats) 지원**(예: `A-B-B-C-A-B-B-C-A-B-B-C`), **여러 Repeat Close가 하나의 Repeat Open을 공유** 가능(예: `A-A-B-A-A-B-C-...` × 3).
- **단축키**: `]`. 커맨드 팔레트 prefix: `repeat-close`.
- **웹앱 시사점(1.6~1.7)**: masterBar에 `repeatOpen: boolean`, `repeatClose: passCount`. 재생 전 **악보 언롤링(unrolling) 패스**에서 반복 구조를 선형 시퀀스로 전개. 중첩/다중 close를 처리하는 스택 기반 언롤러 필요.

### 1.8 얼터네이트 엔딩 / 볼타 (Alternate Endings, "Repetitions")
- **동작**: 활성 마디의 시작에 볼타 브래킷을 추가 — 그 마디가 **몇 번째 반복 패스에서 연주되는지** 지정. 반복 기호와 함께 사용. 클릭 시 **Alternate endings 다이얼로그**에서 적용할 패스 번호 선택(복수 선택 가능).
- **수치**: 패스 번호 범위 **1~8**. 한 마디가 여러 패스 번호를 가질 수 있음(예: B가 1,3패스).
- **예시**: `1-2-1-3` (마디2=1번 엔딩, 마디3=2번 엔딩), 4패스 예: `A-B-A-C-A-B-A-D`.
- **암묵 확장 규칙**: 어떤 마디에 엔딩이 지정되면, 별도 표시가 없는 한 **뒤따르는 마디들도 같은 패스에 속함**(예: `A-B-C-A-D`).
- **단축키**: 커맨드 팔레트 prefix `alternate-endings`. 악보의 엔딩 표기 직접 클릭으로도 편집.
- **웹앱 시사점**: masterBar에 `alternateEndings: Set<1..8>` 비트마스크. 언롤러가 현재 패스 번호와 비트마스크를 대조해 마디 포함/제외 결정 + 암묵 확장 규칙 구현.

### 1.9 디렉션 (Directions — Segno/Coda/Fine 점프)
- **동작**: **Directions 에디터**(팔레트 아이콘 또는 Bar 메뉴)에서 **타깃(target)** 과 **점프(jump)** 를 배치. 재생이 마디 간 점프하도록 함.
  - **타깃**: Coda, Double Coda, Segno, Segno Segno, Fine (5종).
  - **점프**: Da Capo, Dal Segno, Dal Segno Segno, Da Coda, Da Double Coda, Al Coda, Al Double Coda, Al Fine 계열 등 — 매뉴얼 기준 "**11가지 코다 점프 유형**" 추가 제공.
- **재생 엔진 규칙(정확 구현 필수)**:
  1. **Da Capo**: 악보 처음으로 복귀(`A-B-C-A-B-C`). **Anacrusis(못갖춘마디)가 있으면 두 번째 마디로 점프** — 픽업 마디는 1회만 연주.
  2. **Dal Segno(D.S.)**: **Segno** 타깃 마디로 점프. Segno 심볼은 첫 패스에서는 무시(`A-B-C-B-C`).
  3. **Dal Segno Segno**: **Segno Segno** 타깃 참조. Segno 점프가 이미 수행되었거나 Segno 점프가 없을 때 수행.
  4. **반복 구간 내 Dal Segno 계열 점프는 마지막 패스에서 수행.**
  5. **Da Coda / Da Double Coda**: Coda/Double Coda 타깃 마디로 점프(`A-C`, 중간 마디 무시). **반복 구간 내 Da Coda 계열 점프는 첫 패스에서 수행.**
  6. **Da Coda 활성화 규칙**: "Al Coda" 지시가 먼저 수행되어야 활성화. Al Coda가 없으면 **마지막 출현 시 자동 활성화**.
  7. **Fine**: 실제 종지 지점. **점프가 이미 한 번 수행된 경우에만 정지**(예: D.C. al Fine → `A-B-C-D-A-B`).
  8. **"Al Fine / Al Coda / Al Double Coda"는 다음 패스에서 해당 타깃 해석을 강제**. "Al ..." 지시가 없으면 마지막 출현이 해석됨.
  9. 관례: Segno는 뒤로 점프, Coda는 앞으로 점프.
- **UI**: 악보의 디렉션 심볼(점프/타깃) 직접 클릭으로 편집 가능.
- **단축키**: `D`. 커맨드 팔레트 prefix: `direction`(Set direction target and jump). Expression Text(`>`)로도 디렉션 이름 직접 입력 가능.
- **스타일시트 연동**: "Add and adjust spacing between bars for Coda" — Coda 섹션 앞에 조절 가능한 마디 간격 추가(§8.5.2).
- **웹앱 시사점**: masterBar에 `directionTargets: []`, `directionJumps: []`. 재생 언롤러는 반복+엔딩+디렉션을 통합한 상태 머신(패스 카운터, 점프 이력 플래그, Al-Coda 활성화 플래그) 필요. 오디오 트랙(백킹 오디오) 재생 시에는 디렉션 미적용(선형 재생)이라는 예외도 처리.

### 1.10 옥타브 클레프 (Ottava: 8va / 8vb / 15ma / 15mb)
- **동작**: 4개 버튼.
  - **8va**: 기보보다 1옥타브 위로 연주. 버튼 클릭 시 **악보상 음표가 1옥타브 아래로 이동**하고 "8va" 표기가 붙음(실제 음높이는 불변).
  - **8vb**: 반대(1옥타브 아래로 연주, 음표는 위로 이동).
  - **15ma / 15mb**: 동일 동작을 **2옥타브**로 확장.
- **단축키**: 커맨드 팔레트 prefix `octave-sign` (8va, 8vb, 15ma, 15mb). Expression Text로 이름 직접 입력 가능.
- **웹앱 시사점**: beat/구간 속성 `ottava: none|8va|8vb|15ma|15mb`. 렌더러는 기보 위치를 시프트 + 점선 브래킷 표시, 오디오는 원래 피치 유지.

### 1.11 못갖춘마디 (Anacrusis)
- **동작**: `Bar > Anacrusis` 체크 시 **첫 마디와 마지막 마디**는 불완전해도 빨간색 오류 표시를 하지 않음(픽업 바 지원). Da Capo 점프 규칙과 연동(§1.9).
- **단축키**: 커맨드 팔레트 prefix `anacrusis`.

### 1.12 멀티레스트 (Multirest)
- **동작**: 연속된 쉼표 마디를 하나의 멀티레스트로 축약 표시.
- **단축키**: `Ctrl+R` (macOS `⌘R`). 커맨드 팔레트 prefix: `multirest`.

## 2. 노트(Note) 레벨 심볼

팔레트 "Note symbols" 그룹 전체 목록(순서대로): Whole/Half/Quarter/8th/16th/32th/64th note, Rest, Dot, Double-dot, Triplet, Custom n-tuplet, Nested tuplet, Tie, Multiple tie, Fermata, Double flat, Flat, Natural, Sharp, Double sharp, Change accidental, Shift one semitone down/up, ppp~fff (8단계), Crescendo, Decrescendo.

### 2.1 음표 길이 (Note Duration)
- **동작**: **온음표(whole)~64분음표**까지 **7단계**. 길이는 그 노트가 놓이는 비트(grid) 배치를 결정. 같은 비트에 서로 다른 길이의 음을 두려면 **멀티보이스 편집 모드** 사용(트랙당 4보이스).
- **UI**: 팔레트 길이 버튼 클릭, 또는 키보드.
- **단축키**: `+` = 길이 증가(4분→2분), `-` = 감소. **Preferences > Interface에서 +/- 동작 반전 가능**. (부록 표 기준: Increase `-`/`_`, Decrease `+`/`=` — 표기 혼재, 기본 동작은 위와 같고 반전 옵션 존재.)
- **웹앱 시사점**: beat에 `duration: 1|2|4|8|16|32|64`. 비트 그리드 재계산 및 마디 완결성 검증(빨간색 표시)과 연동.

### 2.2 점음표 / 겹점음표 (Dot / Double-dot)
- **동작**: 점 = 원 길이의 **+1/2**(점4분음표 = 1.5박). 겹점 = **+3/4**.
- **단축키**: Dot `Shift+.` (macOS `.`/`⇧.`), Double dot `Ctrl+.` (macOS `⌘.`).

### 2.3 쉼표 (Rests)
- **동작**: 각 음가에 대응하는 쉼표 7종(whole~64th rest / semibreve~hemidemisemiquaver).
- **단축키**: `R` (쉼표 삽입). 커맨드 팔레트 prefix: `rest`.
- **관련 도구**: `Tools > Complete/Reduce Bars with Rests` — 빈/짧은 마디에 쉼표 자동 보충, 초과 쉼표 삭제.
- **웹앱 시사점**: 쉼표는 노트 없는 beat로 모델링. 스타일시트에 타브 전용 "4분쉼표를 `-`로 표시" 옵션 존재(§8.5.2).

### 2.4 튜플렛 (Triplet / Custom N-tuplet / Nested tuplet)
- **동작**:
  - **Triplet**: 같은 음가 2개 시간에 3개 연주(3분할).
  - **Custom n-tuplet**: 좌클릭 = 모드 토글(활성 시 아이콘 **파란색**), **우클릭 = 분할 수 선택 드롭다운**. 분할 수 **2~13**: Duplet(2), Triplet(3), Quadruplet(4), Quintuplet(5), Sextuplet(6), Septuplet(7), Nonuplet(9), Decuplet(10), Undecuplet(11), Dodecuplet(12), Tredecuplet(13) + **`n:m` 커스텀 비율**(X음표를 Y음표 길이에 — X·Y 모두 커스텀, 전용 Custom tuplet 다이얼로그).
  - **Nested tuplet**(GP8 신기능): 튜플렛 안에 튜플렛 중첩. 팔레트에서 표준 튜플렛 버튼 **아래**에 위치. 우클릭 메뉴/설정은 Custom tuplet과 동일. **내부(안쪽) 튜플렛 = Nested tuplet 아이콘으로, 외부(바깥) 튜플렛 = Custom n-tuplet 아이콘으로 편집**. 생성 순서는 무관(외부/내부 어느 쪽 먼저든 가능). 예: 트리플렛 안에 퀸튜플렛 3개.
- **두 가지 편집 워크플로(모두 지원 필수)**: ① 노트 입력 후 마우스 선택 → 버튼 클릭으로 적용, ② 버튼 먼저 활성화(파란색) → 이후 입력되는 노트가 모두 해당 튜플렛으로 생성.
- **렌더링**: 숫자(또는 n:m 비율)와 **브래킷**으로 묶어 표시. 스타일시트 "Show tuplet bracket" 토글 존재.
- **단축키**: Triplet `Shift+/` (macOS `/`/`⇧/`). 커맨드 팔레트: `n:m`(Set custom tuplet), `force-tuplet-bracket`.
- **웹앱 시사점**: beat에 `tuplet: {n, m}` + 중첩 지원을 위한 트리 구조(beat가 tupletGroup에 속하고 tupletGroup이 상위 tupletGroup에 속함). 시간 계산은 재귀적 비율 곱.

### 2.5 붙임줄 (Tie / Multiple Tie)
- **동작**: **Tie** — 노트를 **직전 노트에 연결**해 직전 노트의 길이를 연장. **Multiple tie** — **비트 전체(코드의 모든 노트)** 를 직전 비트에 연결.
- **단축키**: Tie Note `L`, Tie Beat `Shift+L`. 커맨드 팔레트: `tie-note`, `tie-beat`.
- **연동**: 벤드가 걸린 노트가 tie로 연결되면 벤드가 전체 길이로 확장됨(§4.11). 스타일시트 "Always display tied notes on tablature" 옵션(§8.5.2).
- **웹앱 시사점**: note에 `tieOrigin/tieDestination` 참조. 재생 시 tie 체인을 단일 지속음으로 병합.

### 2.6 페르마타 (Fermata)
- **동작**: 특정 비트를 지휘자 재량으로 늘임. **멀티트랙 심볼**(모든 트랙에 동시 적용). 클릭 시 **Fermata 다이얼로그**: ① **심볼(글리프) 종류** 선택, ② **템포 조정으로 멈춤 길이** 설정.
- **UI**: 팔레트 버튼, 악보의 페르마타 직접 클릭.
- **단축키**: `F`. 커맨드 팔레트 prefix: `fermata`.
- **웹앱 시사점**: masterBar 수준 이벤트 `{position, glyphType, tempoScale}`. 재생 스케줄러가 해당 비트 구간의 로컬 템포를 스케일.

### 2.7 임시표 (Accidentals)
- **동작**: **5종** — sharp, double sharp, flat, double flat, natural. 효력 범위: **해당 마디 끝까지**.
- **주의 임시표(Cautionary/Courtesy Accidental)** (GP8 신기능): 이론상 불필요하지만 독자에게 변화를 상기시키는 임시표. 메뉴 `Note > Accidentals > Cautionary accidentals`로 적용/제거.
- **이명동음 변환(Change accidental)**: **음높이를 유지한 채** 임시표 표기만 변경(F# ↔ Gb). `Note > Accidentals` 메뉴에서도 가능.
- **반음 올림/내림**: 노트 또는 **다중 선택 전체**를 반음 이동(Edit > Select all과 조합해 트랙 전체 이조 가능).
- **단축키**: Sharp `Ctrl+9`, Flat `Ctrl+7`, Natural `Ctrl+8`, Double Sharp `Ctrl+Alt+9`, Double Flat `Ctrl+Alt+7`, Change Accidental `Ctrl+Alt+8`, 반음 위 `Alt+Shift+Up`, 반음 아래 `Alt+Shift+Down`.
- **웹앱 시사점**: note는 (string, fret) + 표기 스펠링(accidental preference)을 별도 저장. 마디 내 임시표 전파/캐리 규칙과 주의 임시표 강제표시 플래그(`forceAccidental`) 필요.

### 2.8 다이내믹스 (Nuances / Dynamics)
- **동작**: **노트 단위**로 연주 강도 지정. **8단계**: ppp, pp, p, mp, mf, f, ff, fff. 같은 비트에 서로 다른 다이내믹 → 서로 다른 보이스 사용.
- **UI**: 팔레트 버튼. 커맨드 팔레트 prefix `dynamic`(예: `ff` 입력으로 검색). Expression Text(`>`)로 직접 적용 가능.
- **스타일시트 연동**: "Hide dynamics" 토글(§8.5.2).
- **웹앱 시사점**: note에 `dynamic: 0..7`. Web Audio에서 velocity/gain 매핑 테이블로 변환.

### 2.9 크레셴도 / 데크레셴도 (Crescendo / Decrescendo)
- **동작**: 비트 또는 다중 선택 구간에 점진적 음량 상승/하강(헤어핀). **재생 반영 조건: 최종 다이내믹(target nuance)을 반드시 지정해야 함**.
- **단축키**: 커맨드 팔레트 prefix `crescendo`, `decrescendo`.
- **웹앱 시사점**: 구간 객체 `{startBeat, endBeat, type}` + 시작/종료 다이내믹 보간으로 per-note velocity 계산.

## 4. 기타 이펙트 상세 사양

### 4.1 고스트 노트 (Ghost Note)
- **동작**: 여린 음. 엔진이 **다이내믹을 자동 감소**.
- **렌더링**: 타브에서 프렛 번호를 **둥근 괄호 `( )`** 로 감쌈.
- **단축키**: `O`.

### 4.2 액센트 / 강한 액센트 (Accented / Heavily Accented)
- **동작**: 다이내믹 자동 증가(Heavily는 더 강하게).
- **단축키**: Accent `;`, Heavy accent `:`.
- **연동**: Track Inspector "Accentuation" 옵션 — 마디 내 특정 박을 자동 액센트.

### 4.3 스타카토 (Staccato)
- **동작**: 기보 음가와 무관하게 매우 짧게 재생. **렌더링**: 노트 아래 점.
- **단축키**: `!`.

### 4.4 레가토 (Legato)
- **동작**: 여러 노트에 적용(프레이징) — 가능한 많은 노트를 연결해 유연하게 연주. **렌더링**: 슬러(slur) 아치.
- **단축키**: `Shift+H`. 커맨드 팔레트 prefix: `legato`.

### 4.5 렛 링 (Let Ring)
- **동작**: 이론 음가를 넘어 현이 계속 울림(아르페지오에 흔용).
- **UI 3계층**: ① 노트 단위 팔레트 토글, ② `Tools > Let Ring Options` 마법사 — **현별(per-string)로 마디 범위에 일괄 적용**하는 전용 다이얼로그, ③ Track Inspector "**Auto let ring**" — 트랙 전체 자동 적용. Auto let ring 시 **쉼표가 나오면 모든 현의 렛 링 정지**.
- **Stringed 모드 상호작용**: Stringed 모드에서는 같은 현의 새 노트가 연주되면 그 현의 let ring 종료; Stringed off(피아노 모드)면 같은 피치 중복 울림 가능.
- **단축키**: `i`. 커맨드 팔레트 prefix: `let-ring`.
- **렌더링**: "let ring" 텍스트 + 점선 연장선. 스타일시트에서 심볼 폰트/문구 변경 및 "쉼표 위로 연장선 지속" 토글 가능(§8.4).
- **웹앱 시사점**: note 플래그 + 트랙 옵션 + 현별 범위 규칙의 3중 소스를 재생 시 병합. Web Audio에서는 노트 release를 지연시키는 방식으로 구현.

### 4.6 서스테인 페달 (Sustain Pedal)
- **동작**: "**Ped.**" 표기를 추가하고 페달을 에뮬레이션. 노트 또는 선택 구간에 적용. **해제 별표(`*`)는 선택 구간 끝 뒤에 자동 배치**.
- **UI**: `Effects > Sustain pedal` 메뉴, 팔레트 버튼.
- **단축키**: 커맨드 팔레트 prefix `pedal`.
- **웹앱 시사점**: 구간 이벤트 `{start, end}`; 재생 시 구간 내 모든 노트의 release 억제(피아노 트랙).

### 4.7 팜 뮤트 (Palm Mute)
- **동작**: 브리지 근처 손바닥 뮤트. **렌더링**: 타브 위 "**PM**" 표기(+점선 연장).
- **UI 3계층**: ① 노트/비트 팔레트 토글, ② `Tools > Palm Mute Options` — 현별·마디 범위 일괄 적용 다이얼로그, ③ Track Inspector "Palm mute" 옵션(특정 주법 스타일에서만 활성) + **Palm mute 강도 슬라이더**(재생 파라미터).
- **단축키**: 노트에 `P`, 비트 전체에 `Shift+P`. 커맨드 팔레트 prefix: `palm-mute`(재적용 시 제거 여부 확인 프롬프트).
- **웹앱 시사점**: note 플래그 + 트랙 레벨 intensity(0..1). 오디오는 저역 감쇠+짧은 디케이 샘플 또는 필터로 근사.

### 4.8 데드 노트 (Dead Note)
- **동작**: 짧고 여린 타격음(리듬 효과). **렌더링**: 프렛 번호 대신 "**x**".
- **단축키**: `X`. 커맨드 팔레트 prefix: `dead-note`.

### 4.9 내추럴 하모닉 (Natural Harmonic)
- **동작**: 프렛 위를 가볍게 짚고 개방현 타격. **일반적으로 5·7·12프렛** 사용(그 외는 얻기 어려움).
- **단축키**: `Y`.

### 4.10 아티피셜 하모닉 (Artificial Harmonic — 4종)
- **종류**: **A.H.**(Artificial Harmonic — 왼손 일반 운지 + 오른손 검지 터치), **T.H.**(Tapped Harmonic — 몇 프렛 위 빠른 탭), **P.H.**(Pinch Harmonic — 픽+엄지 터치, 디스토션에 적합), **S.H.**(Semi Harmonic — 핀치와 유사하되 현의 자연 공명 유지).
- **단축키**: `Ctrl+Alt+Y` (macOS `⌥Y`).
- **스타일시트 연동**: "타브에 터치 지점 표시", "옥타브 인공 하모닉을 점(.)으로 대체 표기"(§8.5.2).
- **웹앱 시사점**: note에 `harmonic: {type: natural|artificial|tapped|pinch|semi, touchFret}`. 오디오는 하모닉 배음 샘플/피치 시프트.

### 4.11 벤드 (Bend) — Bend 다이얼로그
- **동작**: 현을 밀어 피치 변경. **Bend 다이얼로그**에서 **커브의 포인트를 드래그**하여 정밀 편집.
  - 왼쪽: **베이스 타입 버튼** — "Bend", "Bend/Release" 등(타입에 따라 **타브에 그려지는 심볼**도 변경).
  - **최대 레벨: 1/4음 ~ 3음(tones)**, "**Full**" = 온음(1 tone). 선택 레벨은 타브 표시("full", "1/2" 라벨)에 반영.
  - **해상도: 1/4음(quarter tone)** 단위 조정 가능.
- **Tie 연동**: 두 노트가 tie로 묶이면 벤드가 **전체 길이로 확장**; tie된 노트 위에서도 벤드 편집 가능(예: **hold** 모티프 배치 — 기본에는 없음). 복잡한 벤드는 오디오 일관성을 위해 tie 필요.
- **벤드 패밀리(커맨드 팔레트 prefix)**: `bend`, `bend-release`, `prebend`, `prebend-bend`, `prebend-release`, `hold-bend`, `release-bend` — 7가지 벤드 형태 모두 지원 필요. 팔레트에서 `bend 2` 식으로 값 직접 입력 또는 ↑↓로 값 스크롤 후 Enter.
- **단축키**: `B`.
- **스타일시트 연동**: "벤드를 표준 기보에도 표시"(기본은 타브 전용), "벤드 이펙트의 실제 들리는 음을 표준 기보에 표시"(§8.5.2).
- **웹앱 시사점**: note에 `bend: {type, points: [{time%, pitchOffset(1/4음 단위, 0~12)}]}`. 렌더러는 화살표+라벨, 오디오는 커브를 Web Audio `detune` 램프 오토메이션으로 스케줄.

### 4.12 페이드 인 / 페이드 아웃 / 볼륨 스웰
- **동작**: Fade-in — 볼륨 노브를 올려 픽 어택이 들리지 않게(바이올린 주법, 역재생 느낌). Fade-out — 어택 후 점차 볼륨 감소. Volume swell — 페이드인 직후 곧바로 페이드아웃.
- **단축키**: Fade In `<`, Fade Out `>`, Volume Swell `Alt+<`/`Alt+>`. 커맨드 팔레트: `fade-in`, `fade-out`, `volume-sweel`(원문 철자).

### 4.13 그레이스 노트 (Grace Notes)
- **동작**: 매우 짧은 꾸밈음. 2종 — **before the beat**(비트 전에 연주), **on the beat**(정확히 비트에서 시작, 다음 비트를 살짝 지연).
- **제약**: **현당 최대 1개**(같은 현에 복수 불가, 현마다는 가능). **마디 길이 계산 및 리듬 표시에서 제외**. Audio Note Settings 적용 불가(다음 노트를 자동 추종).
- **단축키**: before `G`, on-beat `Ctrl+Alt+G` (macOS `⌥G`).
- **웹앱 시사점**: beat에 종속된 `graceNotes[]`(bar-duration 검증기에서 제외). 재생 시 고정 짧은 길이로 스케줄.

### 4.14 해머온 / 풀오프 (Hammer-On / Pull-Off, HoPo)
- **동작**: 같은 현 위 두 노트 전환 — 첫 노트만 어택. 두 번째 노트가 높으면 H, 낮으면 P — **피치 비교로 H/P 자동 판별**(사용자는 단일 HoPo 토글만 적용). 체인 HoPo는 슬러로 표시.
- **단축키**: `H`. 커맨드 팔레트 prefix: `hammer-on-pull-off`.
- **스타일시트 연동**: "H/P 문자 표기 추가" 토글, 슬러 충돌 회피/외곽 노트만 슬러/그룹 단일 슬러 옵션(§8.5.2).
- **웹앱 시사점**: note 플래그 `hopo: boolean`(방향은 파생값). 오디오는 두 번째 노트 어택 트랜지언트 제거.

### 4.15 트레몰로 피킹 (Tremolo Picking)
- **동작**: 한 개 이상 노트를 최대 속도로 반복. **아이콘 우클릭으로 반복 음가(subdivision) 선택**.
- **단축키**: `"` / `⇧"` (32분음표 기준). 커맨드 팔레트 prefix: `tremolo-picking`.
- **웹앱 시사점**: note에 `tremoloPicking: 8|16|32`. 재생 시 음가 단위로 노트 반복 전개.

### 4.16 모르덴트 / 턴 (Ornaments)
- **동작**: Upper mordent — 조표 기준 **위** 이웃음과 빠른 교대. Lower mordent — **아래** 이웃음. Turn(gruppetto) — 본음 주위 4음 선율(위/아래 이웃음 노출). Inverted turn — 아래 음부터 시작.
- **단축키**: 커맨드 팔레트 prefix `ornament`.
- **웹앱 시사점**: note에 `ornament: upperMordent|lowerMordent|turn|invertedTurn`. 재생 전개는 조표(key signature) 기준 이웃음 계산.

### 4.17 왼손 태핑 (Left-Hand Tapping)
- **동작**: 어택 없이 왼손 손가락으로 프렛 타격.
- **단축키**: `(` / `Shift+(`. 커맨드 팔레트 prefix: `left-hand-tapping`.

### 4.18 슬라이드 (Slides — 6종)

| 종류 | 동작 | 렌더링 |
|---|---|---|
| Legato slide | 첫 음 연주 후 둘째 음까지 슬라이드, 재어택 없음 | 사선 + **슬러** |
| Shift slide | 슬라이드 후 둘째 음 **재어택** | 사선만 |
| Slide in from below | 낮은 미지정 프렛에서 올라오며 어택 | 노트 앞 상승 사선 |
| Slide in from above | 높은 미지정 프렛에서 내려오며 어택 | 노트 앞 하강 사선 |
| Slide out downwards | 연주 후 낮은 미지정 프렛으로 | 노트 뒤 하강 사선 |
| Slide out upwards | 연주 후 높은 미지정 프렛으로 | 노트 뒤 상승 사선 |

- **단축키**: Legato Slide `S`.
- **스타일시트 연동**: "sl." 텍스트 표기 추가 토글(§8.5.2).
- **웹앱 시사점**: note에 `slide: {type}` — in/out형은 목적지 프렛 없음. 오디오는 짧은 글리산도(피치 램프).

### 4.19 트레몰로 바 / 와미 바 (Tremolo Bar)
- **동작**: 비트의 **모든 노트** 피치를 브리지 회전으로 변경(다이브 등). **Tremolo Bar 다이얼로그**는 **Bend 다이얼로그와 동일 방식**(프리셋 형태 + 드래그 포인트 커브). **1/4음(quarter tone) 해상도**(GP8 신기능).
- **UI**: `Effects > Vibrato bar` 메뉴, 팔레트 버튼.
- **단축키**: `Ctrl+Alt+V` (macOS `⌥V`).
- **렌더링**: "w/bar" 계열 표기 — 스타일시트 "Whammy Bar Indication" 심볼 문구/폰트 커스텀 가능(§8.4).
- **웹앱 시사점**: beat에 `whammy: {points[]}` (음수 dive 포함, 1/4음 단위). 비트 전체 detune 오토메이션.

### 4.20 트릴 (Trill)
- **동작**: 같은 현에서 HoPo 연속으로 두 음을 빠르게 교대. **Trill 다이얼로그**: ① **둘째 음의 프렛**(첫 음은 기보된 음), ② **속도** 선택.
- **렌더링**: 교대 음은 개별 표시하지 않고 트릴 심볼만 — 악보 경량화. 스타일시트 "Display second note for trill effect" 토글로 둘째 음 표시 가능.
- **단축키**: `N`.
- **웹앱 시사점**: note에 `trill: {secondFret, speed}`. 재생 시 속도 음가로 두 음 교대 전개.

### 4.21 비브라토 (Vibrato — 4종)
- **왼손 비브라토**: 슬라이트/와이드 **2진폭**. **렌더링**: 타브 위 **작은 물결**, **노트가 끝날 때까지** 지속. 팔레트 아이콘 우클릭으로 진폭 선택.
- **트레몰로 바 비브라토**: 슬라이트/와이드 2진폭. **비트(코드) 전체**에 적용. **렌더링**: **큰 물결**, **새 노트가 연주되면 종료**.
- **단축키**: LH Slight `V`, Bar Slight `W`, Bar Wide `Ctrl+Alt+W`(macOS `⌥W`), Vibrato bar `Shift+W`.
- **웹앱 시사점**: note(`vibrato: slight|wide`) / beat(`barVibrato: slight|wide`) 분리 모델. 오디오는 LFO detune.

### 4.22 와우와우 (Wah-Wah)
- **동작**: 이펙트 페달 마커. 옵션: **On/Off**, **Opened/Closed**.
- **재생 조건**: **RSE 사용 + 트랙 이펙트 체인에 Wah-Wah 페달 존재** 시에만 소리에 반영, 아니면 표기 전용.
- **단축키**: Open `Ctrl+Alt+O`(macOS `⌥O`), Close `Ctrl+Alt+C`(macOS `⌥C`). 커맨드 팔레트 패턴: `wah` — `o`=open, `c`=closed, Space=없음.

### 4.23 브러시 (Brush)
- **동작**: 에너지 있게 현을 쓸어내림(스트럼). **Brush 다이얼로그**: **속도(Speed)** + **딜레이(Delay)** 조정. **제약: 브러시 길이 < 비트 길이**(모든 음이 울릴 시간 확보). 방향: down(낮은현→높은현)/up(높은현→낮은현).
- **Auto Brush**: Track Inspector 토글 — 트랙의 **모든 코드에 자동 브러시**(코드 내 음들을 미세 시차로 재생).
- **단축키**: Brush Down `Ctrl+D`, Brush Up `Ctrl+U`. 커맨드 팔레트 패턴: `brush` — `d`/`u`/Space.

### 4.24 아르페지오 (Arpeggio)
- **동작**: 코드 음을 하나씩 연주. **Arpeggio 다이얼로그**: 속도 + 딜레이(브러시와 동일 파라미터). **Down = 최저음현→최고음현, Up = 최고음현→최저음현**.
- **단축키**: Down `Ctrl+Shift+D`, Up `Ctrl+Shift+U`. 커맨드 팔레트 패턴: `arpeggio`.
- **스타일시트 연동**: "Display arpeggio arrow"(화살표 표시) 토글.
- **주의**: 아르페지오에는 Audio Note Settings(오프셋/길이/벨로시티) 적용 불가.

### 4.25 라스게아도 (Rasgueado)
- **동작**: 플라멩코 오른손 스트럼 기법. **18개 모티프** 제공, **전부 오디오 엔진이 재생 해석**. **Rasgueado 다이얼로그**에서 모티프 선택.
- **렌더링**: 코드 위 "Rasgueado" + 선택 모티프명 표시.
- **단축키**: `Shift+R`. 커맨드 팔레트 prefix: `rasguedo`(원문 철자).

### 4.26 운지 표기 (Left/Right-Hand Fingering)
- **동작**: 전용 **좌/우 운지 다이얼로그** 2개. 각 창 하단에 **자체 단축키 목록** 표시 — **해당 창이 열려 있고 포커스를 가질 때만 단축키 활성**.
  - **왼손(프렛팅), 현악기 트랙**: `T`=엄지, `0`=검지, `1`=중지 … (라벨 방식은 스타일시트에서 변경 가능).
  - **오른손(피킹), 현악기 트랙**: `P`=엄지, `i`=검지, `m`=중지 … (PIMA 관례, 스타일시트에서 변경 가능).
  - **피아노/오케스트라 트랙**: 양손 모두 **1~5** 번호(트랙 "stringed" 옵션 off 시 피아노식 운지 자동 제안 — GP8 신기능).
- **렌더링 위치**: 표준 기보에서는 **노트 앞**, 표준 기보 없으면 **타브 아래**. 스타일시트로 위치 변경(§8.5.3).
- **커맨드 팔레트 패턴**: `left hand` — `0 1 2 3 4 T`, `-`=노트 건너뜀, Space=비트 건너뜀; `right hand` — `p i m a c`, 동일 규칙. prefix `left-hand-fingering`, `right-hand-fingering`.
- **웹앱 시사점**: note에 `leftFinger`, `rightFinger`. 표기 관례는 스타일시트 설정에서 렌더링 시 변환.

### 4.27 현 번호 표시 (String Number)
- **동작**: 특정 노트의 현 번호를 **오선(표준 기보)에** 원 안 숫자로 표시(타브 아님).
- **단축키**: 커맨드 팔레트 prefix `show-string-number`.

### 4.28 픽스트로크 (Pickstroke)
- **동작**: 피킹 방향 지정. **'v' 심볼 = 업스트로크**(최고음현→최저음현 방향), 다운은 ㄷ자형 심볼.
- **단축키**: Down `Shift+D`, Up `Shift+U`. 커맨드 팔레트 패턴: `pickstroke` — `d`=down, `u`=up, Space=없음(예: `dduddud`). `picking` prefix — 모노포닉 선택에 `Alternate`(교대)/`Economy`(동일 방향) 자동 피킹 패턴.

### 4.29 태핑 (Tapping)
- **동작**: 오른손/왼손 손가락으로 프렛 타격(양손 병용 흔함). **비트(코드) 전체 단위** 적용. 오디오 엔진 해석됨.
- **렌더링**: 타브 위 "**T**".
- **단축키**: `)`. 커맨드 팔레트 prefix: `tapping`.

### 4.30 슬랩 / 팝 (Slap / Pop — 베이스)
- **동작**: Slap — 오른손 엄지로 현 타격(손목 회전, 퍼커시브). Pop — **최고음 2현 중 하나**를 검지로 강하게 튕김. 보통 슬랩→팝 연쇄. **슬랩 베이스 샘플이 베이스 사운드뱅크에 포함**되어 사실적 재생.
- **단축키**: Slap `$`. 커맨드 팔레트 패턴: `slap pop pattern` — `s`=slap, `p`=pop, Space=없음.

### 4.31 골페 (Golpe)
- **동작**: 손가락(finger) 또는 엄지(thumb)로 **기타 바디 타격**. 2종 변형(Golpe finger / Golpe thumb).
- **단축키**: 커맨드 팔레트 패턴 `golpe` — `f`=finger, `t`=thumb, Space=없음.

### 4.32 픽 스크레이프 (Pickscrape up/down)
- **동작**: 픽을 현에 긁으며 넥 위/아래로 이동. 2종. **오디오 엔진 미해석(표기 전용)**.

### 4.33 데드 슬랩 (Dead Slapped)
- **동작**: 왼손으로 현을 타격하는 베이스 퍼커시브 기법. **오디오 엔진 미해석(표기 전용)**.
- **단축키**: 커맨드 팔레트 prefix `dead-slapped`.

### 4.34 이펙트 재생 해석 여부 요약 (클론 오디오 엔진 체크리스트)

| 재생 반영(오디오 해석) | 표기 전용 |
|---|---|
| Ghost/Accent/Staccato/Legato, Let ring, Sustain pedal, Palm mute(강도 슬라이더), Dead note, 하모닉스 전종, 벤드 패밀리, Fade/Swell, Grace notes, HoPo, Tremolo picking, Ornaments, LH tapping, 슬라이드 전종, Tremolo bar, Trill, 비브라토 4종, Brush, Arpeggio, Rasgueado(18모티프 전부), Tapping, Slap, Pop | **Pickscrape**, **Dead slapped**, Wah-wah(RSE+체인 내 페달 없을 때), String number, Fingering, Barre |

## 5. 표기/디자인 심볼 (Notation & Design Symbols)

팔레트 "Notation symbols" 그룹:

| 심볼 | 동작 |
|---|---|
| Chord | 코드 추가 — Chord 다이얼로그 열림 (§7) |
| Slash | 선택 노트를 슬래시 기보로 변환 |
| Barre | 바레/하프바레 표기 |
| Timer | 경과 시간(min:sec) 표시 |
| Text | 커서 위치에 자유 텍스트 |
| Section | 커서 위치에 섹션 추가/편집 |
| Prevent breakline / Force breakline | 줄바꿈 방지/강제 |
| Automatic / Invert stems orientation | 스템 방향 자동/반전 |
| Automatic / Force / Break / Break secondary beaming / Force beaming group | 빔 그룹 제어 |

### 5.1 슬래시 (Slash)
- **동작**: 선택 노트를 슬래시 기보로 변환하되 **노트 정보를 모두 유지**(비파괴 — 전환 노트가 있는 리듬 모티프에 유용). GP8부터 **표준 기보·드럼 기보에서도 사용 가능**(기존엔 리듬 기보/타브 전용). 트랙 전체를 슬래시 전용 표시로 하려면 Track Properties에서 설정. 모든 악기에 사용 가능.
- **UI**: `Note > Slash` 메뉴 또는 팔레트.
- **스타일시트 연동**: 슬래시 기보에서 오선/바라인 표시 토글, "단일 노트의 프렛·현 표시"(§8.2, §8.5.2).
- **웹앱 시사점**: beat에 `slash: boolean` 렌더 플래그(데이터 보존). 렌더러 분기만 추가.

### 5.2 바레 (Barre)
- **동작**: 특정 프렛에서 **바레 또는 하프바레**를 악보 구간(span)에 걸쳐 표시. 타브 없는 클래식 기타 악보에서 상용. **Barre 다이얼로그**: 타입(full/half), 프렛 번호, 범위.
- **렌더링**: **오선 위**에 표시.
- **단축키**: `Shift+I`. 커맨드 팔레트: `barre`(Set barre fret), `barre-half`.

### 5.3 타이머 (Timer)
- **동작**: 악보 시작부터 해당 지점까지의 경과 시간을 `min:sec`로 표시. **값은 자동 계산**. 반복 구간에서는 **첫 패스 기준 시간** 표시.
- **단축키**: `@`. 커맨드 팔레트 prefix: `timer`.

### 5.4 자유 텍스트 (Text)
- **동작**: 커서 위치(비트)에 자유 텍스트 부착. 악보에서 클릭해 편집 가능.
- **단축키**: `T`. 커맨드 팔레트 prefix: `free-text`.

### 5.5 줄바꿈 제어 (Prevent/Force Breakline)
- **동작**: 해당 마디에서 시스템(줄) 바꿈 강제/방지. 마지막 마디 우측 정렬에도 사용(`Bar > Force break line`).
- **단축키**: Force `Ctrl+Return`(macOS `⌘↵`), Prevent `Shift+&`. 커맨드 팔레트: `force-break-line`, `prevent-break-line`.

### 5.6 스템 방향 (Stem Orientation)
- **동작**: 선택 노트의 스템 방향을 자동/반전. 커맨드 팔레트 prefix: `stem-direction`.

### 5.7 빔 그룹 (Beaming)
- **동작**: 자동 빔 / 강제 빔 / 빔 분리 / **2차 빔만 분리**(1차 빔 유지) / 동일 튜플렛 빔 그룹 강제 — 5가지 제어. Time Signature 다이얼로그에서 박자표별 **기본 빔 연결 방식**도 설정 가능.
- **웹앱 시사점**: beat에 `beamMode: auto|force|break|breakSecondary|forceGroup`. 렌더러의 빔 그루핑 알고리즘이 박자표 기본값 + 노트별 오버라이드를 병합.

## 6. 가사 (Lyrics)

- **접근**: `Track > Show lyrics editor` 메뉴 또는 팔레트 가사 아이콘 → **Lyrics editor 창**.
- **트랙 연결**: 가사는 선택 트랙에 부착되어 그 트랙의 비트에 자동 분배. 보통 보컬 트랙에 부착; 악기 트랙도 가능(음절-리듬 불일치 주의). **"Display on Track" 체크 해제 시 트랙에 표시하지 않고 정보용으로만 보관** 가능.
- **줄 수**: 최대 **5줄**(줄들은 선택 트랙 아래에 상하로 표시). 줄마다 **시작 마디(first bar)** 지정 가능(선행 공백 입력 회피).
- **자동 분배 구문(syntax)**:

| 토큰 | 효과 |
|---|---|
| 공백 | 음절 전환(다음 비트로) |
| `-` (하이픈) | 음절 전환 — 단어를 음절로 분할 |
| `+` | 두 단어를 **한 비트**에 결합(공백 대체) |
| 연속 하이픈/공백 | 여분 구분자 1개당 빈 비트 1개 |
| 줄바꿈 | 공백으로 처리; **연속 줄바꿈은 공백 1개로 축약**(텍스트를 자유롭게 시각적으로 배치 가능) |
| `[…]` | 괄호 안 텍스트는 악보에 미표시(주석/섹션 헤딩용, 예: `[INTRO]`, `[REFRAIN]`) |

- **줄별 Visible 체크박스**: 해제 시 텍스트는 유지하되 악보에서 숨김.
- **제한**: 가사는 트랙의 **첫 스태프에만** 표시 가능.
- **스타일시트 연동**: 가사 위치 — **오선 아래 / 오선 위 / 숨김** 3택(§8.4).
- **커맨드 팔레트**: prefix `lyrics`.
- **웹앱 시사점**: track에 `lyricsLines[5]: {text, firstBar, visible}`. 렌더 시 토크나이저가 구문 규칙으로 음절→비트 매핑 스트림 생성; 원본 텍스트는 항상 보존.

## 7. 코드 이름 / 다이어그램 배치

(코드 다이어그램 편집기 자체 상세는 도구 도메인과 공유 — 여기서는 악보 표기/배치 관점.)

- **입력**: `Note > Chord` 메뉴 또는 단축키 **`A`**, 팔레트 Chord 버튼. 악보의 기존 코드 클릭 → 동일 다이얼로그가 프리필되어 열림.
- **표시 토글(코드별)**: 악보에서 **Name / Diagram / Fingering** 각각 표시·숨김 선택 가능.
- **코드명 형식**: **Classic / Jazz / Rock** 3종 네이밍 관례. 이름 앞 **아포스트로피(`'`)** 를 붙이면 입력 그대로 고정(자동 리네이밍 방지, 예: `'MyChord`).
- **배치(스타일시트, §8.1)**: 다이어그램을 **악보 상단(top of score)** / **악보 내(inline, 비트 위)** / **둘 다** 표시. 다이어그램 스타일 Classic/Jazz/Rock, **크기·간격은 소수점 단위 조정**.
- **헤더 다이어그램 편집 규칙**: 악보 상단 헤더의 다이어그램을 편집하면 **악보 내 해당 코드의 모든 출현에 반영**.
- **트랙 코드 라이브러리**: 생성된 모든 다이어그램은 트랙별 라이브러리("Track chords", 팔레트 Chords 버튼)에 축적. **이름만 표시 중인 코드는 라이브러리에서 다이어그램이 회색 처리**. `+` 버튼으로 신규 생성.
- **다이어그램 글리프**: 너트 위 **○ = 개방현**, **× = 미연주 현**; 운지 번호는 노트 원 안(1=검지, 2=중지 …) + 다이어그램 아래에도 표시. 바레는 배치된 노트로부터 자동 제안. "Highlight root note" 옵션은 다이어그램에서만 루트 색상 변경(악보 렌더에는 미반영).
- **Show Diagram 자동 동작**: 노트가 없는 비트에서 코드 창을 닫으면 메인 다이어그램의 노트가 악보에 자동 입력; 노트가 이미 있으면 다이어그램만 비트 위에 표시(노트 불변).
- **이조 연동 주의**: 코드 창 좌측(빌더)으로 만든 다이어그램만 이조(transpose) 가능. 이름만 입력한 코드는 이조 미적용. 이조 불가 다이어그램은 **삭제되고 이름만 남음**.
- **복사/붙여넣기**: Preferences > Interface "Include chords and scale diagrams in copy paste" 체크 시 단순 붙여넣기에 포함; 아니면 `Edit > Special Paste`.
- **왼손잡이 제한**: 좌수용 다이어그램 표시 불가(타브와 동일) — 대안: Fretboard View의 좌수 모드.
- **스케일 다이어그램 배치(참고)**: `Note > Scale diagram`(`Shift+S`)로 만든 다이어그램은 **선택 비트 위, 오선 위에 표시**; 그리드 간격은 스타일시트 Symbols 섹션에서 조정.
- **웹앱 시사점**: chord 객체 `{name, customNameLocked, diagram{baseFret, frets[], fingers[], barres[]}, show{name,diagram,fingering}}`를 트랙 라이브러리에 저장하고 beat는 chordId 참조. 헤더 렌더는 라이브러리를 순회, 인라인 렌더는 beat 참조 — 편집은 항상 라이브러리 객체를 갱신.

## 8. 스타일시트 (Stylesheet) 시스템

- **접근**: `File > Stylesheet` 메뉴 또는 **`F7`**. 문서별로 저장되는 렌더링 설정 — 음악 데이터는 불변, 렌더링만 변경. 문서 비율/페이지 설정을 언제든 변경 가능.
- **구조: 5개 탭** — ① Page & Score Format, ② Systems & Staves, ③ Header & Footer, ④ Texts & Styles, ⑤ Notation.

### 8.1 Page & Score Format 탭
- **Page 섹션**: 커스텀 페이지 크기 또는 **12개 사전 정의 용지 형식**, **가로/세로(orientation)**, **여백(margins)**.
- **Sizes 섹션**: 악보 전역 비율. **Rhythm proportion 값 — 소수점 단위 조정**. 크기 변경이 **폰트·코드 크기에 영향 여부** 토글.
- **Tuning 섹션**: 트랙 튜닝 표시/숨김. 위치: **악보 상단** 또는 **각 트랙 앞**. 모드: **Name only / Name + notes / 비표준 튜닝 현만 표시(Unusually tuned strings)**. 상단 표시 시 추가 옵션: **2열 분할 / 가로 배치 / 테두리 박스(frame)**. (비표준 튜닝은 기본적으로 악보 상단에 표시됨.)
- **Chord diagrams 섹션**: 스타일 **Classic/Jazz/Rock**, 표시/숨김, 배치(상단/악보 내/둘 다), **크기·간격 소수점 조정** (§7 참조).

### 8.2 Systems & Staves 탭
- **Staves 섹션**: **오선 굵기(staff line thickness)** 수치, **시스템 간 간격(spacing between systems)** 수치 + 토글 8종:
  1. 첫 시스템 들여쓰기(Adjust the first system indentation, 양 조절 가능)
  2. 슬래시 기보에서 오선 표시(Display stave lines when using slashes notation)
  3. 슬래시 기보에서 바라인 표시
  4. 시스템 내 확장 바라인(Display extended barlines in system — 전 스태프 관통 연결)
  5. 각 시스템 시작에 음자리표 반복 표시
  6. 각 시스템 시작에 박자표 반복 표시
  7. 각 섹션 시작에 박자표 표시
  8. **Capo·변칙 튜닝이 표준 기보에 영향(Capo and shifted tunings impact standard notation)** — 체크: 실제 들리는 음 표기(조표 통일) / 해제: 표준 튜닝 읽기 습관 유지를 위한 자동 이조 표기(트랙별 올바른 조표)
- **Brackets 섹션**: 3택 — 브래킷 없음 / **타브+표준 기보 묶음(Group tablature and standard notation)** / **유사 악기 묶음(Group similar instruments)**(오케스트라식). `//` 기호로 시스템 구분 가능.
- **Track name 섹션**: 표시 빈도 — 첫 시스템만 / 각 페이지 첫 시스템 / 모든 시스템 / 숨김. 이름 형식 — 풀네임/약어(약어는 Track Inspector에서 편집). 방향 — 세로/가로. 단일/멀티트랙 뷰 모두 적용.

### 8.3 Header & Footer 탭
- **토큰 시스템**: `%`로 감싼 플레이스홀더 + 자유 리터럴 텍스트 조합(예: `Music by %MUSIC%` → "Music by " + 작곡가 필드값).

| 토큰 | 의미 |
|---|---|
| `%TITLE%` `%SUBTITLE%` `%ARTIST%` `%ALBUM%` | 곡 제목/부제/아티스트/앨범 |
| `%WORDS%` `%MUSIC%` `%TABBER%` `%COPYRIGHT%` | 작사/작곡/채보자/저작권 |
| `%PAGE%` `%PAGES%` | 현재 페이지 번호 / 전체 페이지 수 |

- 각 항목: **표시 체크박스**, **항목별 폰트/크기/스타일**, **정렬(좌/우/중앙)**.
- **첫 페이지 vs 나머지 페이지 별도 구성**: 창 상단 "Other Pages" 셀렉터로 전환.
- 토큰 값 출처: Song Inspector의 곡 정보 필드.

### 8.4 Texts & Styles 탭
- **Texts 섹션**: 악보 내 각 텍스트 요소별 **폰트(OS 설치 폰트) / 크기 / 스타일(regular·bold·italic)** — 타브 프렛 번호도 커스텀 대상. **템포 소수점 표시** 토글. **가사 위치**(오선 아래/위/숨김). **마디 번호 표시**(모든 마디 / 각 스태프(시스템) 첫 마디 / 숨김).
- **Musical symbols 섹션**: 전역 심볼 스타일 **Classic / Jazz**(손글씨풍) 2택. 개별 심볼의 **폰트 변경 + 문구 리네임** 가능 — 대상 10종: Let Ring, Palm mute, Slide, Hammer-On, Pull-Off, Slap, Pop, Pick Scrape, Rasgueado, Whammy Bar Indication. **"Extend symbol lines over rests"** 토글 — 심볼 연장 점선(let ring, PM 등)이 쉼표 위로 지속/정지.

### 8.5 Notation 탭
#### 8.5.1 Rhythmic 섹션 (타브 리듬 표기)
토글 6종:
1. 표준 기보 병기 시 타브의 리듬 스템/빔 숨김(Hide in tablature when using standard notation)
2. 리듬 스템을 타브 내부로 연장(Extend rhythmic inside tablature)
3. 타브 리듬 표기에서 쉼표 위 빔 표시
4. 타브에서 2분·온음표 프렛 번호에 원 표시
5. 표준 기보 빔 수평 강제
6. 드럼 기보 빔 수평 강제

**Rhythmic position in tablature**: **4개 보이스 각각** 리듬 표기 위치를 **위/아래/숨김** 중 독립 선택.

#### 8.5.2 Symbols 섹션 (심볼·주석 세부 토글 — 19종 + 수치 1종)

| 옵션 | 효과 |
|---|---|
| Indicate where the string is touched for artificial harmonics in tablature | 인공 하모닉 터치 지점 타브 표기 |
| Alternate representation "." for artificial octave harmonics in tablature | 옥타브 인공 하모닉 점 표기 |
| Alternate representation "C" for 4/4 and 2/2 signatures | common/cut time 기호 표기 |
| Display real pitch of heard notes for bend effects in standard notation | 벤드 실제 음 표기 |
| Display arpeggio arrow | 아르페지오 화살표 |
| Display second note for trill effect | 트릴 보조음 표기 |
| Always display tied notes on tablature | 타이 노트 항상 표시 |
| Add and adjust spacing between bars for Coda | Coda 앞 간격(수치 조절) |
| Add "H/P" symbols for HoPo | H/P 문자 표기 |
| Avoid notes collisions for HoPo slurs in tablature | 슬러-프렛번호 충돌 회피 |
| Display HoPo slur only on outer notes in tablature | 외곽 노트만 슬러 |
| Display a single HoPo slur over a group of notes in tablature | 그룹 단일 슬러 |
| Add "sl." symbol for slide effect | 슬라이드 "sl." 표기 |
| Colorize fret numbers in tablature for voices 2, 3 and 4 | 멀티보이스 프렛번호 색상화 |
| Hide useless rests on standard notation (기본 off) | 불필요 쉼표 숨김 |
| Alternate representation "-" for quarter rests (tab. only) | 4분쉼표 "-" 표기 |
| Hide dynamics (기본 off) | 다이내믹 숨김 |
| Display fret and string of single notes in slash notation | 슬래시 단일노트 상세 |
| Show tuplet bracket | 튜플렛 브래킷 표시 |
| + Scale diagram grid spacing (수치) | 스케일 다이어그램 그리드 간격 |

#### 8.5.3 Fingerings 섹션
- 왼손 표기: **T1234** 또는 **P1234** 2택.
- 오른손 표기: **pimac / pimax / pimae / timao** 4택.
- 표준 기보 내 위치: **위 / 노트 앞 / 아래** 3택. 타브 내 위치: **위 / 아래** 2택.
- 운지 **크기(scale)** 조정.

### 8.6 스타일 저장/로드/강제
- 스타일시트 창 **좌하단 Options 탭**: **Save style**(이름 지정 저장 — 여러 파일에서 재사용), 내장 스타일 로드 — **Rock / Jazz / Classic** + 사용자 저장 스타일.
- **기본 스타일 강제**: `Preferences > General > "Force stylesheet (when opening or creating a file)"` + 스타일 선택 — 문서 자체 스타일시트를 무시하고 전역 강제.
- **웹앱 시사점(§8 전체)**: 스타일시트는 문서에 직렬화되는 단일 JSON 설정 객체(약 60+ 키). 렌더러는 모든 키를 구독하는 순수 함수형 레이아웃 엔진이어야 하며, 프리셋(Rock/Jazz/Classic)은 키-값 오버레이. 변경 시 전체 리플로우 트리거. PDF/PNG/SVG 내보내기와 인쇄도 동일 스타일시트를 사용.

## 9. 커맨드 팔레트 연동 (이펙트 패턴 일괄 입력)

이펙트를 패턴 문자열로 **비트 단위 일괄 적용**하는 고급 커맨드(선택 구간에 패턴 반복 적용; 선택 없으면 커서부터 악보 끝까지 적용):

| prefix | 패턴 구문 |
|---|---|
| `arpeggio` / `brush` / `pickstroke` | `d`=down, `u`=up, Space=없음 (예: `pickstroke ud` → 업다운 교대 체인) |
| `golpe` | `f`=finger, `t`=thumb, Space |
| `wah` | `o`=open, `c`=closed, Space |
| `slap pop pattern` | `s`=slap, `p`=pop, Space |
| `left hand` | `0 1 2 3 4 T`, `-`=노트 스킵, Space=비트 스킵 |
| `right hand` | `p i m a c`, `-`, Space |
| `picking` | `Alternate`(교대) / `Economy`(동일 방향) |

- 패턴 입력 중 **비트가 순차적으로 라이브 편집**됨(실시간 프리뷰). `unset <Effect>`로 선택 구간에서 특정 이펙트 일괄 제거(예: `unset Tie`). Expression Text(`>` / `Shift+Cmd+E`)로 코드/다이내믹/조표/옥타브 사인/음자리표/디렉션을 prefix 없이 직접 입력.
- 이미 적용된 토글 이펙트(quick command)를 재적용하면 **제거 여부 확인 프롬프트** 표시.
- **웹앱 시사점**: 이펙트 적용 API를 (beatRange, effectType, valueSequence) 형태의 배치 커맨드로 설계하면 팔레트 버튼과 커맨드 팔레트가 동일 경로 공유 가능.

## 11. 웹앱 구현 시사점 종합

1. **데이터 모델 계층**: `masterBar(반복 open/close·엔딩 비트마스크·디렉션·박자·free time·더블바·anacrusis·페르마타)` → `bar(트랙별)` → `voice(×4)` → `beat(duration, dot, tuplet 트리, slash, whammy 커브, brush/arpeggio, tapping, barVibrato, 텍스트/타이머/코드 참조, 다이내믹 헤어핀 구간)` → `note(tie 참조, 임시표 스펠링+forceAccidental, dynamic 0..7, 이펙트 플래그 집합, bend 커브, trill, harmonic, fingering, ornament, slide type, grace 서브리스트)`. 심볼별 부착 레벨(note/beat/bar/master)을 정확히 구분하는 것이 클론 정합성의 핵심.
2. **렌더링**: 스타일시트 60+ 옵션을 구독하는 SVG/Canvas 엔그레이빙 엔진. 표준 기보·타브·슬래시(·드럼) 병렬 렌더 + 심볼 텍스트/폰트 오버라이드(Classic/Jazz), 소수점 단위 크기 값, 보이스별 리듬 표기 위치, 슬러 충돌 회피 로직 지원.
3. **재생(Web Audio)**: ① 언롤링 패스(반복/엔딩/디렉션 상태 머신 — §1.7~1.9 규칙 전부), ② 이펙트 해석 계층(벤드/와미 = detune 램프(1/4음 단위), 비브라토 = LFO, 트릴/트레몰로 피킹/라스게아도/장식음/브러시/아르페지오 = 노트 시퀀스 전개, 다이내믹/액센트/고스트 = velocity, 헤어핀 = 보간, 페르마타/트리플렛 필 = 타이밍 변형), ③ 표기 전용 심볼 무시 목록(§4.34).
4. **입력 UX**: 팔레트 버튼(토글 + 파란 활성 표시 + 우클릭 추가 설정) / 단일 키 단축키 / 악보 요소 직접 클릭 편집(파란 하이라이트) / 커맨드 팔레트 패턴 — 4개 진입점이 모두 동일한 커맨드 레이어를 호출하도록 설계.
5. **검증 엔진**: 마디 완결성(빨간색, anacrusis 예외), simile 위치 제약(1/1~2마디 금지), 브러시 길이 < 비트 길이, 그레이스 현당 1개, 크레셴도 목표 다이내믹 필수, 첫 마디 repeat open 암묵 규칙 등 심볼별 제약 검사기.
