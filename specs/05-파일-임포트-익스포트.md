# Guitar Pro 8 클론 웹앱 — 도메인 사양서: 파일 임포트/익스포트 (File Import/Export)

> 출처: Guitar Pro 8 공식 매뉴얼(334p) 전체 추출본 통합. 본 문서는 **파일 포맷, 임포트(ASCII/MIDI/MusicXML/PowerTab/TablEdit), 배치 컨버터, 익스포트(오디오/ASCII/MIDI/MusicXML/PDF/PNG/SVG), 인쇄, 파일 보호(비밀번호), mySongBook 통합** 도메인의 단일 기준 사양(Single Source of Truth)이다.

## 1. 파일 포맷 및 파일 열기/저장 기반

### 1.1 Guitar Pro 네이티브 파일 포맷 — 버전별 확장자 매핑

| 확장자 | Guitar Pro 버전 |
|---|---|
| `.gp` | 8.x 및 7.x |
| `.gpx` | 6.x |
| `.gp5` | 5.x |
| `.gp4` | 4.x |
| `.gp3` | 3.x |
| `.gtp` | 1.x 및 2.x |

- **정확한 동작**: Guitar Pro 8은 **버전 1 이상 모든 세대의 Guitar Pro 파일을 열 수 있다** (어느 버전으로 만들었든 무관). 저장은 항상 `.gp`(GP8) 포맷.
- **스코어 1개 = 파일 1개** 모델. 여러 파일을 동시에 열 수 있고, 열린 파일은 각각 **탭 바(Tab bar)** 의 탭으로 표시된다(개수 제한 없음, 하드웨어 한계까지).
- 파일 자체에 문서별 표시/재생 정보가 내장된다: **스타일시트(stylesheet), 페이지 레이아웃, 줌(zoom), 노테이션, 재생 엔진(RSE/MIDI)**. 열 때 이 내장 정보로 표시·재생한다 (환경설정 General 탭의 "Force options"로 전역 강제 덮어쓰기 가능 — 예: 항상 150% 줌, 항상 MIDI 재생).
- **웹앱 구현 시사점**: `.gp`(ZIP 컨테이너 기반) 파서 + 레거시 바이너리 포맷(.gp3/.gp4/.gp5/.gpx) 리더가 필요. 내부 데이터 모델은 문서 메타(스타일시트·줌·엔진 설정 포함)를 파일에 직렬화해야 한다.

### 1.2 파일 열기 방법 (4가지 + 최근 파일)

| 방법 | 동작 |
|---|---|
| 메뉴 `File > Open` | 파일 탐색 다이얼로그. 단축키 **Ctrl+O** (Win) / **⌘O** (macOS) |
| **드래그 앤 드롭** | 지원되는 모든 포맷의 파일을 Guitar Pro 창에 끌어다 놓으면 열림 |
| **더블클릭** | OS 파일 연결(확장자 연결)로 아이콘 더블클릭 시 열림 |
| 메뉴 `File > Open recent…` | 최근 열었던 파일 서브메뉴 |
| 홈페이지(Homepage) | `Open File` 버튼, Recent files / Local files / Templates / Examples 카테고리에서 열기 |

- 열 수 있는(임포트 가능한) 포맷 전체 목록: **Guitar Pro(전 버전), MIDI, MusicXML, TablEdit, ASCII, PowerTab**.
- **웹앱 구현 시사점**: File System Access API + 드래그앤드롭 핸들러(dataTransfer)로 다중 포맷 진입점 구현. MIME/확장자 스니핑으로 포맷 라우팅.

### 1.3 저장/닫기 및 파일 관련 단축키

| 동작 | Windows | macOS |
|---|---|---|
| New File | Ctrl+N | ⌘N |
| Open File | Ctrl+O | ⌘O |
| Save | Ctrl+S | ⌘S |
| Save As | Ctrl+Shift+S | ⇧⌘S |
| Close File | Ctrl+W | ⌘W |
| Close All Files | Ctrl+Shift+W | ⇧⌘W |
| Print | Ctrl+P | ⌘P |
| Next / Previous Tab | Ctrl+Tab / Ctrl+Shift+Tab | ⌃⇥ / ⌃⇧⇤ |

- **탭 바의 파일 상태 표시**:
  - **수정된(dirty) 파일**: 탭 **오른쪽에 작은 원(circle)** 표시.
  - **잠긴(비밀번호 보호) 파일**: 탭 **왼쪽에 작은 자물쇠(lock)** 표시 — **자물쇠 클릭 시 비밀번호를 알면 잠금 해제** 가능.
  - 탭에 마우스오버 시 **Save / Close 버튼** 노출.
  - 탭 우클릭(macOS ⌃클릭) 컨텍스트 메뉴: **Close all tabs**, **파일이 있는 폴더 열기**(탐색기/Finder에서 표시).
- **환경설정 연동**: `Preferences > General`의 **"Open last session documents"** — 종료 시 열려 있던 문서를 다음 실행 시 자동 재오픈(옵트인).
- **웹앱 구현 시사점**: 문서별 dirty 플래그 + IndexedDB 세션 복원. 자물쇠 상태는 문서 암호화 메타데이터에서 파생.

### 1.4 홈페이지의 파일 접근 기능 (파일 I/O 관련 부분)

- **검색 바**: Open File 아래 위치. 파일명 전체/일부 입력으로 **실시간 증분 검색**, 카테고리별 결과 개수 표시. 검색 대상 5개 카테고리: **Recent files, Local files, mySongBook, Templates, Examples**.
- **Recent files**: 최근 연 파일 목록. **Delete recent files**로 전체 비우기만 가능(개별 삭제 불가). 행에 마우스오버 시 **왼쪽에 핀(pin) 아이콘** 표시 → 클릭 시 목록 **최상단 고정**, 재클릭 시 해제(고정 상태 영속).
- **Local files**: **폴더 1개를 선택(Choose a folder)해 인덱싱** → 해당 폴더의 Guitar Pro 파일들이 라이브러리로 표시.
- **파일 미리보기**: 홈페이지의 각 파일에 대해 **트랙별 악보 미리보기(visual preview) + 트랙별 오디오 프리리스닝(pre-listening)** 제공(열기 전 확인).
- **웹앱 구현 시사점**: 폴더 인덱싱은 File System Access API의 디렉터리 핸들 영속화로 구현. 미리보기는 파일 헤더/1페이지 경량 파싱 + 렌더 캐시.

## 2. 임포트 (Import)

임포트 진입점: 메뉴 **`File > Import`** (하위에 ASCII, MIDI, MusicXML, PowerTab, TablEdit). 커맨드 팔레트(`Ctrl+E`)에서 **`import` prefix**("File import")로도 접근 가능.

### 2.1 ASCII 탭 임포트

- **메뉴**: `File > Import > ASCII`.
- **정확한 동작**: ASCII 텍스트 파일을 분석해 음악 정보를 추출, 재생/수정/인쇄 가능한 Guitar Pro 스코어를 생성. 임포트되는 정보: **음표(notes), 리듬(rhythm), 이펙트(effects), 마디 변화(bar changes)**.
- **임포트 대상 트랙**: 항상 **현재 활성 트랙(active track)** 에 임포트된다 → 악기별 ASCII 파일을 트랙을 바꿔가며 **연속 임포트**해 멀티트랙 구성 가능.
- 파일 안의 주석/잡텍스트가 있어도 타블라처 부분만 추출한다. 단, **파싱 규칙**(반드시 준수):
  1. 현(string) 라인은 **하이픈 `-` 연속**으로 표현되어야 함.
  2. **마디 변경은 모든 현 라인에 세로 막대 `|`** 로 표시되어야 함.
  3. 같은 시스템(타블라처 라인 묶음)의 현들 사이에 **빈 줄 금지**.
  4. **타블라처 라인 내부에 주석 금지**.
- **후처리**: 임포트 후 마디가 불완전한 경우가 많음 → **`Tools > Check bar duration`**(F4) 도구로 검증 권장. 불완전한 마디는 리듬이 틀리고, **마디 길이를 초과하는 음은 재생되지 않는다**.
- **웹앱 구현 시사점**: 라인 그루핑(현 수 자동 감지) → `|` 기준 마디 분할 → 프렛 숫자/이펙트 문자 토크나이저. 리듬 정보가 없으므로 균등 분배 후 사용자 보정 UX 필요.

### 2.2 MIDI 임포트

- **메뉴**: `File > Import > MIDI`. **지원 포맷: MIDI format 0 및 format 1**.
- **임포트 다이얼로그 구성(5개 영역)**: ① Destination score ② Track selection ③ Global parameters ④ Score options ⑤ **Real-time preview**.

#### ① Destination score (대상 문서)
- **새 파일(new file)** 또는 **현재 파일(current file)** 선택.
- 현재 파일로 임포트 시 MIDI 트랙들은 **활성 파일의 마지막 트랙 뒤에 추가**된다.

#### ② Track selection (트랙 선택/매핑)
- 트랙별 행: 1) **체크박스**(체크 시 임포트) 2) **Destination track** 3) **Instrument** 4) **개별 파라미터**.
- **Destination track**: 기본값은 MIDI 트랙 1개 → 대상 트랙 1개. **그랜드 스태프(grand staff) 감지 시 두 MIDI 트랙이 같은 대상 트랙으로 자동 연결**(수동으로 분리 가능). **여러 MIDI 트랙에 같은 대상 트랙을 지정하면 병합(merge)되어 한 트랙으로 임포트**.
- **Instrument**: MIDI 파일의 **'Program change'** 정보로 결정. **정보 없으면 기본값 piano**. 악기 버튼 클릭 시 악기 선택 창이 열리며 선택 항목: **악기(MIDI 또는 RSE), 튜닝(tuning), 스태프 수(staff count), 노테이션(notations), 트랙 색상·아이콘, 사운드(sound)**.
- 알려진 문제와 해결: Program change가 없으면 드럼 트랙이 피아노로 감지되거나 기타에 그랜드 스태프가 붙을 수 있음 → 악기 버튼으로 변경하거나 staff count로 단일/그랜드 스태프 강제.

#### ③ Global parameters (전체 트랙 공통, 트랙별 개별 설정도 가능)

| 옵션 | 동작 |
|---|---|
| **Quantization** | 임포트 음표의 리듬 정밀도(최소 음길이) 결정. 예: 4분음표 퀀타이즈 → 8분음표 4개가 박자 위의 4분음표 코드가 됨. 권장: **라이브 연주 녹음 MIDI는 낮은 퀀타이즈**, 정확하지만 복잡한 파일은 높은 퀀타이즈 |
| **Increase the note value** | 음표와 다음 음표 사이에 쉼이 있으면 음길이를 늘려 리듬을 단순화 |
| **Import of dotted notes** | 전체 음가가 허용하면 **붙임줄(tie)로 이어진 음을 점음표(dotted note)로 변환** |
| **Authorize multivoice** | 음 길이가 겹치면 한 음을 **다른 보이스(voice)** 에 배치 |
| **Authorize triplets** | 음 길이가 셋잇단(triplet) 길이에 가까우면 셋잇단으로 처리. **라이브 녹음 MIDI에는 비권장**(오검출) |
| **Two channels per track** | 원본이 트랙당 2 MIDI 채널로 녹음된 경우 체크 |

#### ④ Score options (스코어 최적화 옵션)

| 옵션 | 동작 / 기본값 |
|---|---|
| **Time signatures import** | MIDI 파일의 박자표 임포트. 미활성 + 새 파일 선택 시 기본 **4/4** (음표는 4/4 기준 분할) |
| **Key signatures import** | MIDI 파일의 조표 임포트. 미활성 + 새 파일 선택 시 기본 **C major** |
| **Tempo import** | MIDI 파일의 템포 임포트. 미활성 + 새 파일 선택 시 기본 **120 bpm** |
| **Set sound tracks in MIDI / RSE** | 모든 트랙을 MIDI 사운드 또는 RSE 사운드로 일괄 설정 |

(참고: 매뉴얼 원문은 4/4와 C major 두 기본값을 문단 간 뒤바꿔 인쇄 — 합리적 해석은 박자표=4/4, 조표=C major.)

#### ⑤ Real-time preview
- **현재 선택된 트랙**을 미리 렌더링해 표시. **파라미터를 바꿀 때마다 자동 갱신**(실시간 미리보기).

- **웹앱 구현 시사점**: SMF 파서(format 0/1) → 퀀타이즈/보이스 분리/셋잇단 추론 파이프라인은 순수 함수로 구성해 파라미터 변경 시 미리보기 재계산(Web Worker 권장). Program change → GM 악기 → 내부 악기 템플릿 매핑 테이블 필요. 퍼커션은 채널 10 규약.

### 2.3 MusicXML / PowerTab / TablEdit 임포트

- **메뉴**: `File > Import` (각 포맷 항목).
- **MusicXML**: **표준(standard) 및 압축(compressed) MusicXML 모두 임포트** 지원. MusicXML은 Recordare®가 만든 XML 악보 기술 포맷으로 대부분의 음악 소프트웨어가 지원하는 범용 교환 포맷.
- **PowerTab®**, **TablEdit®** 파일도 동일한 `File > Import` 메뉴에서 임포트.
- **웹앱 구현 시사점**: `.mxl`(ZIP) 압축 해제 + XML DOM 파싱. PowerTab(.ptb)/TablEdit(.tef)은 리버스 엔지니어링된 바이너리 스펙 기반 리더 필요(우선순위 낮게 단계적 지원 가능).

### 2.4 배치 컨버터 (Batch converter)

- **메뉴**: `File > Batch converter`.
- **목적**: 구버전 Guitar Pro 파일(**GP1~GP6: `.gtp`, `.gp3`, `.gp4`, `.gp5`, `.gpx`**)을 **GP8 `.gp` 포맷으로 일괄 변환**.
- **워크플로(다이얼로그)**:
  1. **Source Directory**: 변환할 파일이 들어있는 폴더 선택 — **하위 폴더(subfolders) 포함**.
  2. **파일 타입 필터**: `.gpx / .gp5 / .gp4 / .gp3 / .gtp` 중 포함할 확장자 선택 → 해당 위치에서 **발견된 파일 개수를 표시**.
  3. **Destination**: **원본 폴더에 출력** 또는 **다른 위치로 출력** 선택.
  4. **Convert** 버튼 클릭.
- 변환 중 **진행률(progress) 창** 표시, 완료 후 **변환 리포트(conversion report)** 표시.
- **원본 파일은 삭제되지 않는다** (원위치에 그대로 유지).
- **웹앱 구현 시사점**: 디렉터리 핸들 재귀 순회 + Web Worker 풀 병렬 변환 + 진행률 스트림 UI. 결과 리포트(성공/실패 목록) 모델링.

### 2.5 오디오 파일 임포트 (오디오 트랙 — 파일 I/O 경계 기능)

- **임포트 가능 오디오 포맷(6종)**: `mpeg / m4a / wav / flac / x-aiff / ogg`.
- 임포트 방법: 오디오 트랙 창에 **드래그 앤 드롭**, 창에서 **탐색(browse)**, 또는 메뉴 **`Track > Add Audio File…`**, 툴바 버튼. 임포트 성공 시 **파형(waveform)** 표시.
- (동기화·재생 상세는 오디오 도메인 담당. 파일 저장 관련 사항은 §3.9 참조.)

## 3. 익스포트 (Export)

익스포트 진입점: 메뉴 **`File > Export`** (하위에 Audio, ASCII Tab, MIDI, MusicXML, PDF, PNG, SVG). 커맨드 팔레트에서 **`export` prefix**("Export score")로 접근 가능 — 예: 팔레트에 `xml` 입력 시 XML import/export prefix가 제안됨.

### 3.1 오디오 익스포트 (Audio)

- **메뉴**: `File > Export > Audio`.
- **지원 포맷(5종)**: **MP3, WAV, OGG, FLAC, AIFF**.
- **목적**: Guitar Pro 없이 일반 오디오 플레이어에서 재생 가능한 오디오 사본 저장.
- **다이얼로그 옵션**:
  - **Export only selection**: 익스포트 전 마우스로 **다중 선택(multiple selection)** 을 만들고 체크 → 선택 구간만 익스포트.
  - **트랙별 개별 파일 익스포트** (스템(stems) 분리 출력).
  - **메트로놈 포함/제외** 선택.
  - **카운트다운(count-in) 포함/제외** 선택.
- **웹앱 구현 시사점**: OfflineAudioContext로 논리 스코어를 오프라인 렌더 → WAV 인코딩 후 lamejs(MP3)/libflac.js/ogg-vorbis 등 WASM 인코더로 변환. 스템 출력은 트랙별 개별 오프라인 렌더 패스.

### 3.2 ASCII 익스포트 (ASCII Tab)

- **메뉴**: `File > Export > ASCII Tab`.
- **정확한 동작**: **활성 트랙(active track)만** ASCII 텍스트 타블라처로 익스포트. 기타/베이스/드럼용, 문자·숫자·기본 ASCII 기호만 사용.
- **이펙트 표기 심볼 범례(22종, 익스포트 시 사용)**:

| 심볼 | 의미 | 심볼 | 의미 |
|---|---|---|---|
| `L` | tied note (붙임줄) | `.` | staccato |
| `x` | dead note | `P` | popping (bass) |
| `h` | hammer-on / pull-off | `S` | slapping (bass) |
| `b` | bend | `+` | tapping |
| `s` | slide | `W` | wide vibrato |
| `~` | vibrato | `<` | fade in |
| `g` | ghost note | `w` | whammy bar |
| `>` | accentuated note | `=` | tremolo picking |
| `t` | trill | `v` | brush up |
| `M` | palm mute | `^` | brush down |
|  |  | `V` | pick stroke up |
|  |  | `n` | pick stroke down |

- **웹앱 구현 시사점**: 트랙 → 현별 문자 버퍼 직렬화기. 임포트 파서(§2.1)와 심볼 테이블을 공유해 왕복(round-trip) 일관성 확보.

### 3.3 MIDI 익스포트

- **메뉴**: `File > Export > MIDI`.
- **정확한 동작**: 스코어를 **MIDI format 1**, 확장자 **`.mid`** 파일로 익스포트.
- **트랙 제외 방법**: 별도 체크박스 없음 — **글로벌 뷰(Global View) 왼쪽 트랙 목록에서 해당 트랙을 뮤트(mute)** 하면 익스포트에서 제외된다(뮤트가 익스포트 필터 역할).
- **채널 정책**: Guitar Pro는 사운드 표현력 향상(이펙트 분리)을 위해 **트랙당 2 MIDI 채널**을 쓸 수 있음. 일부 외부 소프트웨어는 트랙당 2채널을 지원하지 않음 → 익스포트 파일이 다른 곳에서 안 열리면 **트랙당 1채널로 전환**해야 함(트랙별 채널 설정 노출 필요). 퍼커션 트랙은 항상 **채널 10**.
- **웹앱 구현 시사점**: 재생 엔진의 이벤트 스트림(반복/디렉션 언롤 전의 선형 트랙 데이터)을 SMF 인코더로 직렬화. 트랙별 mute 상태와 채널 매핑을 익스포트 컨텍스트에 반영.

### 3.4 MusicXML 익스포트

- **메뉴**: `File > Export > MusicXML`.
- 스코어를 **MusicXML**(Recordare® XML 악보 포맷, 범용 교환 포맷)로 익스포트.
- **웹앱 구현 시사점**: 내부 모델 → MusicXML 매핑 계층(음표/타브 string·fret은 `<technical>` 요소로). 임포트와 스키마 매핑 공유.

### 3.5 PDF 익스포트

- **메뉴**: `File > Export > PDF`.
- **정확한 동작**: **스코어 전체 또는 선택 구간만** Adobe® PDF®로 익스포트. 출력은 **벡터 패스(vector paths)** (해상도 독립).
- 출력 레이아웃은 **현재 View 모드(Page 또는 Screen, View 메뉴에서 선택)** 와 **`File > Stylesheet > Page & Score format`의 레이아웃 옵션을 그대로 반영**한다.
- **PDF 익스포트 다이얼로그 구성**:
  - **File name 입력 필드** (파일명 지정, §3.6 토큰 지원).
  - **대상 폴더(destination folder)** 선택.
  - **해상도(resolution)** 선택.
  - **Pages 범위 선택**: **한 페이지 / 페이지 선택 / 전체 페이지**.
  - **Selection 범위**: 마우스로 미리 만든 **스코어 선택 구간만** 익스포트.
  - **Export** 버튼으로 실행.
  - 체크박스: **익스포트 완료 시 대상 디렉터리 자동 열기**.
- **웹앱 구현 시사점**: 렌더러가 Canvas가 아닌 벡터 씬그래프(SVG 경로)를 출력하도록 설계하면 pdf-lib/jsPDF로 벡터 PDF 생성 가능. 페이지네이션은 스타일시트 페이지 설정과 동일 코드 경로 사용.

### 3.6 파일명 자동 명명 토큰 시스템 (PDF/PNG/SVG 익스포트 공통)

- File name 입력란 옆의 **"Information" 신택스 버블(툴팁)** 이 사용 가능한 토큰을 안내하고, 입력란 **아래에 결과 파일명 실시간 미리보기** 표시.

| 토큰 | 이름 | 기능 | 예시 출력 |
|---|---|---|---|
| `%T` | Score title | 스코어 제목으로 명명 | `Anatole.png` |
| `%t` | Track name | 현재 트랙 이름으로 명명 | `E.guitar.png` |
| `%n` | Track number | 현재 트랙 번호로 명명 | `3.png` |
| `%N` | Tracks count | 전체 트랙 수로 명명 | `5.png` |
| `%d` | Date | 익스포트 날짜(YYYY-MM-DD) | `2022-03-30.png` |
| `%h` | Time | 익스포트 시각(HH-MM) | `13-03.png` |

- **웹앱 구현 시사점**: 토큰 치환기 유틸 1개를 세 익스포트 다이얼로그에서 공유. 미리보기는 입력 이벤트마다 즉시 갱신.

### 3.7 PNG 익스포트

- **메뉴**: `File > Export > PNG`.
- **정확한 동작**: 스코어를 **PNG 이미지**로 저장(웹 게시/화면용). **WYSIWYG 보장 — 익스포트 이미지는 Guitar Pro 창에 보이는 화면과 정확히 동일**한 렌더링.
- 다이얼로그는 **PDF 익스포트와 거의 동일**: 파일명+토큰, 대상 폴더, **해상도 선택**, 페이지/선택 범위, 완료 시 폴더 열기.
- **PNG 전용 옵션**: **Transparent background(투명 배경)** 체크박스 — 색이 있는 이미지/배경 위에 얹을 때 유용.
- (GP8 개선사항으로 명시된 옵션: 자동 파일명 신택스, 해상도 선택, **선택 구간만 익스포트**, 투명 배경.)
- **웹앱 구현 시사점**: 화면 렌더러와 동일한 씬을 devicePixelRatio 독립 스케일로 오프스크린 Canvas에 재렌더 → `toBlob('image/png')`. 투명 배경은 배경 레이어 스킵.

### 3.8 SVG 익스포트

- **메뉴**: `File > Export > SVG`.
- **정확한 동작**: **SVG (Scalable Vector Graphic)** 벡터 포맷으로 익스포트 — 확대/축소 시 해상도 손실 없음, 웹사이트/2D 그래픽 용도.
- 다이얼로그는 **PDF 익스포트와 거의 동일**하며, PNG처럼 **Transparent background 옵션** 보유.
- **웹앱 구현 시사점**: 렌더러가 SVG 씬을 직접 출력하면 사실상 무비용. 폰트는 outline path 변환 또는 폰트 임베드 정책 결정 필요.

### 3.9 오디오 트랙 임베드/추출 (Guitar Pro 파일 내 오디오 저장)

- 위치: 오디오 트랙 창의 **"Perform tasks" 메뉴(기어 아이콘)**.
- **"Embedded Audio File"**: Guitar Pro 파일에 **오디오 트랙을 포함해 저장**. **기본값: 자동 임베드 활성** — 임베드를 원치 않으면 체크 해제. 임베드된 파일을 열면 오디오 트랙 창이 바로 나타남.
- **"Extract Audio File"**: 내장 오디오 파일을 **오직 .MP3 포맷으로만** 추출 익스포트.
- **환경설정 연동**: `Preferences > General`의 **"Embed audio files by default"** — 임베드 기본값을 전역으로 설정(오디오 트랙 창의 Perform Tasks 메뉴에서도 동일 설정 접근 가능).
- 툴바의 오디오 트랙 버튼에 **파란 외곽선** = 현재 파일에 오디오 트랙이 포함되어 있다는 표시.
- **웹앱 구현 시사점**: `.gp` 컨테이너에 오디오 바이너리 청크 + 싱크포인트 메타를 함께 직렬화. 추출은 원본 보존 또는 MP3 재인코딩(WASM) 중 정책 결정.

## 4. 인쇄 (Printing)

- **메뉴**: `File > Print`. **단축키: Ctrl+P (Windows) / ⌘P (macOS)**.
- **툴바 인쇄 버튼**도 존재하나, **창이 충분히 넓을 때만 표시**된다(반응형 툴바 — 좁으면 버튼 숨김, 메뉴로는 항상 접근 가능).
- **인쇄 옵션은 프린터 드라이버에 의존**한다(용지·부수 등은 OS 인쇄 다이얼로그).
- **페이지 커스터마이징(폰트, 페이지 설정, 여백 등 모든 페이지 설정)은 스타일시트에서 수행**: `File > Stylesheet` (단축키 **F7**) → **Page & Score Format 탭** — **12종 사전 정의 용지 포맷** 또는 커스텀, **가로/세로(orientation)**, **여백(margins)**.
- 관련 팁: 인쇄 지면 작업 시 **Page/Parchment 또는 Page/Grid 뷰 모드** 사용 권장. 반복 기호/디렉션 사용은 인쇄 페이지 수 절감 수단으로 매뉴얼에 명시.
- mySongBook 구매 스코어는 **크레딧 구매 시 인쇄 권한 포함**(§6.5).
- 커맨드 팔레트 quick command **`print`**("Print score") = 인쇄 아이콘 클릭과 동일.
- **웹앱 구현 시사점**: 인쇄 전용 페이지 레이아웃(스타일시트 기준)을 print CSS 또는 PDF 경유로 `window.print()`에 전달. PDF 익스포트와 렌더 경로 공유.

## 5. 파일 보호 (File Protection / Lock & Unlock)

- **메뉴**: `File > Lock/Unlock` → 잠금/해제 창.
- **3가지 보호 옵션**:

| 옵션 | 동작 | 비밀번호 | 대표 용도 |
|---|---|---|---|
| **Finalize** | 파일을 확정(finalize)하여 원치 않는 수정 방지. **해제에 비밀번호 불필요** — 단순 잠금 해제로 다시 수정 가능 | 없음 | 실수 방지용 읽기 전용 표시 |
| **Lock editing** | **모든 수정 금지**, 비밀번호로 보호 | 필요 | 교사가 학생에게 배포 시 편집 방지 |
| **Lock opening** | **비밀번호 없이는 파일을 열 수조차 없음** | 필요 | 온라인 레슨 판매 사이트 |

- **경고(매뉴얼 명시)**:
  1. **비밀번호 분실 시 복구/열기 방법 없음** — 반드시 별도 기록 필요.
  2. 비밀번호는 복잡한 알고리즘이 없는 **소프트 보호(soft protection)** — 결연한 공격자에 대한 절대 보증 아님.
- **UI 연동**: 잠긴 파일은 탭 바에서 **탭 왼쪽 자물쇠 아이콘**으로 표시되며, **자물쇠 클릭 → 비밀번호 입력으로 잠금 해제** 가능(§1.3).
- **웹앱 구현 시사점**: 파일 메타에 `protection: none | finalized | lockEdit | lockOpen` + 비밀번호 해시(lockOpen은 콘텐츠 암호화 권장, 예: AES-GCM + PBKDF2). 에디터는 protection 상태에 따라 명령 디스패처를 읽기 전용으로 전환.

## 6. mySongBook 통합 (공식 악보 라이브러리)

### 6.1 개요
- **mySongBook** = Arobas Music이 발행하는 합법(저작권 준수) 고품질 악보 라이브러리. 수천 곡의 전문 편집 악보(기타/베이스/현악기 미공개 편곡 + 원곡의 모든 악기를 담은 풀 스코어: 기타, 베이스, 드럼, 키보드, 가사).
- 각 탭의 **오디오(사운드) 설정은 전문가가 제작** — 원곡과 최대한 동일한 사운드. **Line In 기능과 최적의 짝**(원곡 기타리스트와 같은 사운드로 합주 가능)으로 소개됨.
- Guitar Pro 자체에는 법적 이유로 무료 탭 카탈로그가 내장되지 않음(예제 파일 23개 제외). **Guitar Pro 8.1부터 일부 mySongBook 타이틀 무료 제공**.

### 6.2 접근 및 계정
- **진입점**: Guitar Pro **홈페이지의 mySongBook 섹션**. 3개 하위 섹션: **Free Tab of The Day / Last Added / All Tabs**(구매 항목 포함).
- **"Sign in to mySongBook"** 버튼 → **이메일 + 비밀번호** 로그인 창. 계정 생성은 무료이나 **웹사이트에서 진행**(인앱 아님).
- **타이틀 열기**: 목록에서 선택 → **활성 구독** 보유 또는 **크레딧으로 기 구매**한 경우 Guitar Pro에서 자동으로 열림. 아니면 구독/크레딧 구매 안내 표시.
- 전체 카탈로그를 이 창에서 브라우징 가능. **구매한 타이틀은 목록 행 오른쪽에 체크 마크** 표시 + **My Purchases** 카테고리에 수록.
- **계정/잔액 UI**: 표시된 **사용자 이메일 클릭** → 계정, **현재 크레딧 잔액**, 크레딧/구독 구매로 직행.

### 6.3 검색과 필터
- **검색 바**: **타이틀 또는 아티스트**로 라이브러리 검색.
- **트랙별 미리보기**: 카탈로그의 **모든 타이틀에 대해 각 트랙별 미리보기** 제공.
- **필터(GP8 신기능, 조합 가능)**: **Genre + Instrument + Score Type** 3축 조합. (예: Folk + Guitar(beginner) + Guitar Single.)

### 6.4 카테고리 메뉴 및 정렬
드롭다운 메뉴 항목(표시 대상 + 정렬):
- **All the tabs** — 전체 라이브러리 표시

- **My Favorites** — 즐겨찾기만 표시
- **My Purchases** — 크레딧 구매 항목만 표시
- **Sort by Added Date** (최신 탭 우선 — **신규 타이틀은 매주 발행**)
- **Sort by Artist**
- **Sort by Title**
- **Hide Free Tab of the Day**

### 6.5 구매(크레딧) 모델
- 타이틀 선택 → 옆의 **"+" 기호** 클릭 → **"Buy for 1 credit(s)"** 선택으로 구매.
- 크레딧 소진 시 충전 안내, guitar-pro.com에서도 구매 가능.
- 크레딧 구매 = 해당 타이틀 **무제한 접근 + 해당 mySongBook 스코어의 인쇄(print) 권한**.
- 대안: **구독(subscription)** — 전체 카탈로그 무제한 접근.

### 6.6 즐겨찾기 / Free Tab of the Day
- **즐겨찾기**: 목록에서 **곡 제목 왼쪽의 노란 별(star)** 체크/해제로 토글(인앱 관리).
- **Free Tab of the Day**: **매일 다른 무료 탭** 제공(기간 한정). 위치: **홈페이지 mySongBook 섹션 최상단**. 카테고리 메뉴의 "Hide Free Tab of the Day"로 숨김 가능.

### 6.7 환경설정 연동
- `Preferences > Interface` 탭: 홈페이지에서 **"mySongBook" 및 "Examples" 항목 제거** 옵션.

- **웹앱 구현 시사점**: mySongBook은 원격 카탈로그 API(검색/필터/정렬 파라미터, 곡별 트랙 미리보기 스트림, 구매·구독 상태) + 계정 세션으로 모델링. 클론에서는 자체 스코어 스토어/커뮤니티 라이브러리로 치환 가능하되 UI 계약(3축 필터, 카테고리 메뉴, 별 즐겨찾기, 체크 마크, 크레딧 버튼)은 동일하게 유지.

## 7. 커맨드 팔레트의 파일 I/O 명령 (교차 참조)

| Prefix | 종류 | 동작 |
|---|---|---|
| `print` | quick | Print score — 인쇄 아이콘 클릭과 동일 |
| `import` | advanced | File import (임포트 다이얼로그 진입) |

## 9. 웹앱 아키텍처 요약 제언 (본 도메인 한정)

1. **포맷 계층 분리**: `codec/` 모듈에 reader(gp, gp3~gpx, midi, musicxml, ptb, tef, ascii)와 writer(gp, midi, musicxml, ascii)를 스코어 IR(중간 표현) 중심으로 대칭 배치 — 배치 컨버터는 reader→writer 파이프의 반복 실행일 뿐.
2. **렌더 경로 단일화**: 화면 렌더러가 벡터 씬(SVG 경로 트리)을 출력하면 PNG(래스터화), SVG(직렬화), PDF(pdf-lib), 인쇄(print CSS)가 모두 같은 씬을 소비 — WYSIWYG 요구사항(§3.7) 자동 충족.
3. **무거운 작업은 Web Worker**: MIDI 임포트 실시간 미리보기, 배치 변환, 오디오 오프라인 렌더+인코딩(WASM: lame/flac/vorbis)은 워커에서 진행률 이벤트로 스트리밍.
4. **보호 파일**: lockOpen은 실제 콘텐츠 암호화(AES-GCM), lockEdit/finalize는 메타 플래그 + 에디터 커맨드 게이팅. "소프트 보호" 수준이라도 클론에서는 진짜 암호화 권장.
5. **mySongBook 대응물**: 카탈로그 REST API(검색 q, genre/instrument/type 필터, sort, category) + 계정·크레딧 원장 + 곡별 트랙 프리뷰(스트리밍 오디오 + 1페이지 렌더 썸네일)로 설계.
