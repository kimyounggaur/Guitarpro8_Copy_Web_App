# Guitar Pro 8 클론 웹앱 — 도메인 사양서: 도구와 부가 기능 (Tools & Utilities)

본 문서는 Guitar Pro 8 공식 매뉴얼(334p) 전체 추출본에서 **Tools 챕터(매뉴얼 p.235~284) 및 관련 부가 기능 + 전체 키보드 단축키(부록 p.324~334 포함)** 를 통합·중복 제거한 단일 기준(single source of truth) 사양서이다.

**포함 범위**: 코드 다이어그램 엔진/에디터, 스케일 엔진(라이브러리·프렛보드 표시), 스케일 다이어그램, 조옮김(Transposition) 도구, 악기 뷰(가상 프렛보드/키보드/드럼킷), 튜너(Tuner), 파일 보호(File Protection), Fretlight 지원, 커맨드 팔레트(Command Palette)·Expression Text·Action List, Tools 메뉴 기타 유틸리티, **앱 전체 키보드 단축키 총람**.

**제외(타 도메인)**: 악보 작성/기보 심볼, 오디오 엔진·사운드뱅크·페달보드, 오디오 트랙 동기화, 임포트/익스포트, mySongBook, 환경설정 상세. (교차 참조만 기재)

---

## 1. 코드 다이어그램 엔진/에디터 (Chord Diagram Engine/Editor)

### 1.1 접근 및 실행
- **기능**: 코드(화음) 다이어그램을 생성·편집하여 악보에 삽입하는 대화상자형 도구.
- **UI 위치 / 진입 경로**:
  1. 메뉴 `Note > Chord`
  2. 단축키 **`A`**
  3. 악보 위의 기존 코드(이름/다이어그램)를 **클릭** → 같은 대화상자가 해당 코드로 미리 채워진 상태로 열림
  4. 에디션 팔레트(좌측 패널)의 **Chords 버튼** → 트랙 코드 라이브러리 경유
- **핵심 동작**: 코드 다이어그램 생성기는 **활성 트랙의 튜닝에 자동으로 맞춰진다** — 시중 코드집에 없는 특수(익조틱) 튜닝에서도 코드 다이어그램 생성 가능.
- **창 구성**: 여러 인터랙티브 영역(코드 빌딩 / 메인 다이어그램 / 다이어그램 후보 리스트 / 재생)으로 구성.
- **웹앱 시사점**: 코드 산출 알고리즘은 튜닝 배열(string→pitch)을 입력으로 받아 보이싱을 계산해야 하며, 트랙 튜닝 변경 시 즉시 재계산이 필요.

### 1.2 이름 기반 코드 빌딩 (Chord Building)
- 첫 번째 영역에서 **코드 이름(루트 + 타입/퀄리티)** 으로 코드를 조합 — "수천 가지 가능성".
- 코드 토널리티(이름)를 선택하면 해당 코드의 **운지(hand position)** 가 표시됨.
- **동등 코드명(Equivalent chords)**: 제안된 코드명 옆 버튼 → 동등한(이명동음/대체 명명) 다이어그램·이름 목록 열람.
- **전위(Inversion)**: 베이스음이 루트와 다른 전위 정의 가능. 또는 단순히 **최저현에 올 베이스 노트만 지정** 가능.
- **표시 토글(코드별, 악보 렌더링에 반영)**: 악보에서 **Name(이름) / Diagram(다이어그램) / Fingering(핑거링)** 각각 표시·숨김.
- **웹앱 시사점**: 코드 데이터 모델 = `{ root, quality, bassNote?, showName, showDiagram, showFingering, customName? }` + 계산된 보이싱 목록.

### 1.3 메인 다이어그램 (인터랙티브 편집기)
- 토널리티 선택 후 나타나는 **메인 다이어그램**이 실제로 악보에 추가되는 다이어그램(이름·핑거링·이론적 구성 포함).
- **핑거링 표시**: 각 노트 원(circle) **안에 손가락 번호** 표시(1=검지, 2=중지, 3=약지, 4=소지 함의), 다이어그램 **아래에도 핑거링 표기**.
- **인터랙션**:
  | 조작 | 동작 |
  |---|---|
  | 프렛 **클릭** | 노트 추가/제거 (토글) |
  | 손가락 위치 **우클릭** | 핑거링 변경 |
  | **스크롤 바** 드래그 | 루트 프렛(다이어그램 기준 프렛) 변경 |
- **바레(Barre) 자동 제안**: 배치된 노트로부터 Guitar Pro가 바레 코드를 자동 감지·제안.
- **Highlight root note 옵션**: 루트 노트를 다이어그램 상에서만 색상 강조 (**악보 렌더링에는 미반영**).
- **너트 위 기호**: **○ = 개방현**(프렛 누르지 않고 연주), **× = 연주하지 않는 현**(뮤트/생략).
- **코드명 편집**: 기본 제안된 이름 수정 가능, (커스텀) 이름이 악보에 표시됨.
  - **리터럴 이름 이스케이프**: 이름 앞에 **아포스트로피(`'`)** 를 붙이면 자동 리네이밍 방지 — 예: `'MyChord` 입력 시 "MyChord" 그대로 유지.
- **코드명 포맷 3종**: **Classic / Jazz / Rock** (명명 규칙 설정).
- **웹앱 시사점**: SVG/Canvas 기반 프렛 그리드 에디터. 상태 = `{ baseFret, notes: [{string, fret, finger}], barres[], openStrings[], mutedStrings[] }`. 바레 감지는 동일 프렛 연속 노트 휴리스틱.

### 1.4 다이어그램 후보 리스트 (Diagram List)
- 선택된 코드에 대해 **가능한 보이싱 다이어그램 목록을 난이도 순(쉬운 것 우선)으로 정렬**하여 표시.
- 목록의 다이어그램 클릭 → **메인 다이어그램으로 승격**.
- **오디션(재생)**: 창 **우상단의 재생(play) 버튼**으로 현재 선택된 다이어그램을 재생.
- **웹앱 시사점**: 보이싱 열거 + 난이도 스코어링(운지 간격/바레 여부/포지션) 정렬. 재생은 Web Audio로 트랙 사운드(또는 기본 기타 음색) 아르페지오/스트럼 재생.

### 1.5 기타 동작 규칙
- **자동 채움(Auto-population)**: 노트는 있으나 다이어그램이 정의되지 않은 비트에서 코드 창을 열면, 해당 비트의 노트가 **메인 다이어그램에 자동 배치**됨.
- **Show Diagram 활성 시**: 선택된 비트에 노트가 없으면 메인 다이어그램의 노트가 **악보에 자동 추가**. 비트에 이미 노트가 있으면 **다이어그램만 비트 위에 표시**(기존 노트 덮어쓰지 않음).
- **왼손잡이 제한**: 왼손잡이용 코드 다이어그램은 표시 불가(태블러처와 동일). 우회책: `View > Show Fretboard View` → 기어(설정) 아이콘 → **가상 왼손잡이 넥** 사용.
- **헤더 다이어그램 편집 전파**: 악보 헤더(상단)의 코드 다이어그램을 편집하면 **악보 내 해당 코드의 모든 출현에 반영**됨.
- **활용 시나리오(매뉴얼 명시)**: ① 태블러처에 코드 다이어그램 추가 ② 한 코드의 모든 프렛보드 포지션 학습 ③ 악기에서 잡은 코드의 이름 식별 + 다른 포지션/핑거링 학습 ④ 특수 튜닝용 표준 코드 포지션 조회.

### 1.6 트랙별 코드 라이브러리 (Track Chords)
- **생성된 모든 코드 다이어그램은 트랙별 코드 라이브러리에 추가**됨 (per-track 컬렉션).
- **접근**: 에디션 팔레트의 **"Chords" 버튼** 클릭 → "Track chords" 창.
- 라이브러리 창의 **`+` 버튼**으로 새 코드 직접 생성.
- **시각 상태**: 악보에 **이름만 표시되는 코드는 라이브러리에서 다이어그램이 회색 처리(greyed-out)** 됨.
- **웹앱 시사점**: 트랙 모델에 `chordLibrary: ChordDiagram[]` 필드. 악보 헤더 다이어그램 렌더링과 라이브러리는 동일 객체 참조(편집 전파).

### 1.7 스타일시트 연동 (렌더링 설정 — F7)
Stylesheet(`File > Stylesheet`, **F7**) > *Page & Score Format* 탭의 **Chord diagrams 섹션**:
| 옵션 | 값/선택지 |
|---|---|
| 다이어그램 표기 스타일 | **Classic / Jazz / Rock** (3종) |
| 표시/숨김 | 토글 |
| 배치 | **악보 상단(top) / 악보 내(inline, 비트 위치) / 둘 다 동시** |
| 코드 크기 | 숫자, **소수점 단위 조정 가능** |
| 코드 간 간격 | 숫자, **소수점 단위 조정 가능** |
- 코드 다이어그램의 크기·위치는 Stylesheet가 최종 결정.
- **웹앱 시사점**: 렌더러는 문서 스타일시트의 diagram 스타일/배치/크기 파라미터를 구독해 재레이아웃해야 함.

### 1.8 복사/붙여넣기·조옮김 연동
- **Interface 환경설정** "Include chords and scale diagrams in copy paste" 체크 시 **일반 붙여넣기(Ctrl+V)에 코드/스케일 다이어그램 포함**. 미체크 시 `Edit > Special Paste`(Ctrl+Shift+V)에서 "Chords and scales diagrams" 항목 선택 필요.
- **조옮김 시 규칙**(4.5절 참조): 코드 창 좌측 **빌딩 옵션으로 생성된 다이어그램만 조옮김 대상**. 이름만 입력한 코드는 조옮김되지 않음. 조옮김 불가능한 다이어그램은 **삭제되고 이름만 남음**.
- **커맨드 팔레트 연동**: `chord` prefix(Set chord), Expression Text에서 `>Cm` 형식으로 즉시 적용, `barre`/`barre-half` prefix로 바레 프렛 설정.

## 2. 스케일 엔진 (Scale Engine — Tools > Scales)

### 2.1 개요 및 스케일 라이브러리
- **접근**: 메뉴 `Tools > Scales`.
- **목적**: 임의 토널리티의 다수 스케일을 **열람·청음**하고, 악보에 사용된 스케일을 **역분석(analyze)**.
- **라이브러리 규모**: **1000개 이상**의 스케일 내장.
- **검색/필터**: **Tags 체크 필터** + **키워드 검색 엔진** 병용.
- **토널리티 선택기**: 선택한 조(tone)로 스케일 표시.
- **재생 버튼**: 스케일 오디션(청음).
- **스케일 구성 표시**: **음정(interval)과 음 사이 반음 수(semitone counts)** 를 보여줌 (이론적 구성 시각화).
- **튜닝 인지 표시**: 프렛보드 표시 방식은 **활성 트랙의 튜닝을 반영**(GP 도구 공통 원칙 — 코드 엔진·스케일·튜너 모두 현재 트랙 튜닝 기준).
- **표시 표면 2종**: **프렛보드 위 스케일 뷰** + **키보드(피아노) 위 스케일 뷰**.
- **웹앱 시사점**: 스케일 = `{ name, tags[], intervals[] }` 정적 데이터셋(1000+). 렌더링은 (tonic + intervals) → 프렛보드 좌표 변환(튜닝 의존) / 건반 좌표 변환.

### 2.2 악보 스케일 분석 (Find Scales from Selection)
- **워크플로**: 악보에서 **마디 선택** → **"Find scales from selection"** 클릭.
- **출력**: 가능한 **조성(tone)+스케일 목록을 출현율(%) 통계와 함께 내림차순**(최적 일치 우선) 표시.
- **한계/가이드**: 결과가 좋지 않으면 선택 범위에 **전조(조성 변화)** 가 포함된 것 — 귀로 전조 지점을 찾아 **전조 없는 마디 범위로 제한**해 재분석하도록 안내.
- **커맨드 팔레트**: `find-scale` prefix (Find-scale(s)).
- **웹앱 시사점**: 선택 노트들의 피치클래스 히스토그램 vs 스케일 템플릿 매칭(가중 일치율 %) 알고리즘.

### 2.3 스케일 → 악보 입력 워크플로 (직접 배치 불가 + 우회)
- **스케일 도구는 스케일을 태블러처에 직접 배치할 수 없음** (제한 사항 명시).
- **우회 워크플로**: 스케일을 **프렛보드 뷰에 표시**한 뒤, **프렛보드를 클릭하여 태블러처에 노트 배치**.
- **프렛보드 우클릭 = 노트 추가 + 커서를 오른쪽으로 자동 전진** → 고속 노트 입력 가능.
- **웹앱 시사점**: 프렛보드 뷰 클릭 핸들러에 좌클릭(추가)/우클릭(추가+advance) 분기 필요.

### 2.4 악기 뷰 내 스케일 오버레이
- 프렛보드/키보드 **양쪽 모두**에서 스케일 **표시/숨김** 가능.
- 뷰 내에서 스케일 표시 활성화 → 아래 버튼 클릭 → **스케일 도구 창**을 열어 스케일 선택.
- 스케일은 **노트 입력 중에도 계속 표시**됨.
- **루트 노트 강조 옵션**: 루트는 원(circle) 대신 **사각형(square)** 으로 렌더링.
- 프렛보드와 키보드에 **동시에** 스케일 표시 가능.
- **커맨드 팔레트**: `show-scale` prefix (Show scale on virtual fretboard).
- **Fretlight 연동**: Fretlight 모드 "Scale" 선택 시 **Scale 창에서 선택한 스케일의 노트만 기타 LED로 점등**(8장 참조).

## 3. 스케일 다이어그램 (Scale Diagrams — Note > Scale diagram)

> GP8 신기능. 스케일 엔진(2장)과 별개로, 악보 위에 붙는 **커스텀 프렛 그리드 다이어그램** 객체.

### 3.1 접근
- 메뉴 `Note > Scale diagram` (매뉴얼 초반부 표기: `Note > Scale…`)
- 단축키: 부록 기준 **`⇧S`(Shift+S)** — ※ 본문(p.242 부근)에는 `⇧F`로 인쇄되어 있으나 `⇧F`는 부록에서 Audio Note Settings에 배정됨. **클론은 ⇧S 채택 권장** (충돌 주석 11.4 참조).
- 에디션 팔레트의 **스케일 다이어그램 아이콘**.
- **빈 비트에서 생성**(빈 다이어그램) 또는 **악보 노트 선택 상태에서 생성**(선택 노트가 다이어그램에 **미리 채워짐** — pre-filled) 두 가지 모두 지원.

### 3.2 다이어그램 창 옵션
| 옵션 | 범위/기본값 |
|---|---|
| 다이어그램 이름 | 텍스트 필드 |
| 현(string) 수 | **3~10** |
| 프렛 수 | **3~24** |
| 시작 프렛 | 기본 **1**(넥 최상단) |
| 프렛 마크(인레이 점) | 표시/숨김 토글 |
| 방향(orientation) | **가로(horizontal) / 세로(vertical)** |
| 활성 심볼 | 편집 중 배치되는 노트에 적용될 기본 심볼 선택 |

### 3.3 심볼 및 노트별 커스터마이징
- 심볼 타입 다수 제공, **각 노트마다 다른 심볼** 지정 가능(기본 심볼과 독립).
- 요소별 커스터마이징 항목: **심볼 / 심볼 색상 / 심볼 내부 텍스트(최대 3자, 예: 계이름·핑거링·스케일 도수) / 텍스트 색상**.
- 용도 예: **토닉 강조**, 특정 스케일 도수 식별.
- **선택(활성) 노트는 사각형 하이라이트**로 표시.
- 편집 한계: **24프렛 × 10현** 최대 그리드 내에서는 제한 없음.

### 3.4 노트 추가/삭제 (마우스)
| 조작 | 동작 |
|---|---|
| 프렛 **한 번 클릭** | 활성 심볼로 요소 추가 |
| 노트 **더블클릭** | 요소 제거 |

### 3.5 확정·삭제·초기화
- **OK** → 다이어그램을 트랙에 추가, **선택된 비트 위(보표 상단)에 표시**.
- **배치된 다이어그램 삭제**: 악보에서 다이어그램 클릭 → 창의 **Delete** 옵션.
- **Reset**: 다이어그램의 **모든 요소를 지우고 빈 상태로 초기화**.

### 3.6 스타일시트/기타 연동
- Stylesheet > Notation 탭 **Symbols 섹션**에서 **스케일 다이어그램 그리드 간격(grid spacing)** 조정 가능.
- Interface 환경설정의 copy/paste 다이어그램 포함 옵션은 스케일 다이어그램에도 동일 적용(1.8절).
- **웹앱 시사점**: `ScaleDiagram = { name, strings(3-10), frets(3-24), startFret, showInlays, orientation, cells: [{string, fret, symbol, symbolColor, text(≤3), textColor}] }`. 악보 렌더러에서 비트 앵커 위 배치.

## 4. 조옮김 도구 (Transposition — Tools > Transpose)

> 트랙 인스펙터의 조성(transposition tonality) 옵션과 **별개로 존재**하는 노트 조옮김 도구. GP8에서 코드 조옮김·다이어토닉 조옮김이 추가됨.

### 4.1 조옮김 창 — 범위 선택기 (2개 독립 선택)
| 선택 | 옵션 |
|---|---|
| 무엇을 | **선택된 노트들** / **트랙의 전체 마디** |
| 어디에 | **현재 트랙만** / **악보의 모든 트랙** |

### 4.2 조옮김 유형 1 — 반음 조옮김 (Semitones)
- 입력: **반음 수** → OK → 해당 반음만큼 이동.
- 체크박스: **코드 다이어그램 포함 조옮김** 여부. (예시: 3반음 이동 + 코드 다이어그램 함께 이동)

### 4.3 조옮김 유형 2 — 크로매틱 조옮김 (Chromatic)
- 각 노트를 **엄격히 정의된 음정**만큼 이동.
- **음정 번호(interval number)**: unison, second, third, fourth, fifth, sixth, seventh (7종).
- **음정 성질(quality)**: minor / major / perfect / diminished / augmented (선택한 음정 번호에 따라 가용 성질이 달라짐).
- **방향**: up / down.
- **옥타브 초과 적용 옵션**(beyond the current octave 토글).
- **코드 다이어그램 조옮김 가능** (예: 단3도 + 다이어그램).
- **조표(key signature) 조옮김**: (a) key signature 체크박스 체크 + (b) **모든 트랙에 적용** 조건을 만족해야 조표도 함께 조옮김됨. (예: 상행 단3도로 코드+조표 동시 이동)

### 4.4 조옮김 유형 3 — 다이어토닉 조옮김 (Diatonic)
- 정의: **조표(조성)를 바꾸지 않고** 보표 상에서 노트를 지정한 음정 수만큼 평행 이동.
- 입력: **음정(도수) 수**, **방향(up/down)**, 선택적으로 **옥타브 수**.
- (예시: 위로 3도 다이어토닉 이동)

### 4.5 코드 다이어그램 처리 규칙 (반드시 재현할 동작)
1. **코드 창의 빌딩 옵션(좌측)으로 생성한 다이어그램만** 조옮김된다.
2. 코드 **이름만** 타이틀 필드에 입력한 코드(다이어그램 미구축)는 조옮김이 **적용되지 않는다**.
3. 다이어그램이 **조옮김 불가능**하면 조옮김 후 **다이어그램은 삭제되고 이름만 표시**로 남는다.

### 4.6 연관 조옮김·조성 기능 (교차 참조 포함)
| 기능 | 위치 | 동작 |
|---|---|---|
| 반음 올림/내림 버튼 | 에디션 팔레트 (Shift one semitone up/down) | 노트 또는 다중 선택을 반음 이동. `Edit > Select all`과 조합해 트랙 전체 조옮김 가능. 단축키 **Alt+Shift+Up / Alt+Shift+Down** |
| Transposition tonality (이조 악기) | 트랙 인스펙터 > Transposition tonality 섹션(드롭다운) | "기보된 C가 실제로 어떤 음으로 소리 나는가" 선택 — 트럼펫/색소폰 등 이조 악기 표시 자동 관리. **Concert tone이 꺼져 있을 때만 유효** |
| Concert tone | 송 인스펙터 버튼 | 켜면 모든 트랙 조옮김 취소, 표시 음 = 실제 소리 음 |
| Capo / shifted tuning 표기 반영 | Stylesheet(F7) > System and Staves 탭 "Capo and shifted tunings impact standard notation" 체크박스 | 체크: 오선 기보가 실음 반영 / 해제: 튜닝·카포 보정 조옮김 자동 적용(읽기 쉬움) |
| 글로벌 토널리티(relative tonality) | 툴바(그룹 10 "Global tonality") / 커맨드 `relative-tonality` | 재생 상대 조성 변경(파일 비파괴). 오디오 트랙에도 적용됨 |
| 오디오 트랙 피치 시프트 | 오디오 트랙 창 | **반음(semitones) + 센트(cents)** 독립 조옮김 (오디오 도메인 상세) |
| 커맨드 팔레트 | `transpose` prefix | Transpose selection |
- **웹앱 시사점**: 조옮김은 피치 연산 계층(semitone 벡터/다이어토닉 도수 연산)과 표기 재산출(스펠링, 조표, 태블러처 재핑거링), 코드 다이어그램 재계산(실패 시 다이어그램 drop) 3단계 파이프라인으로 설계.

## 5. 악기 뷰 (Instrument Views — 가상 프렛보드/키보드/드럼킷)

### 5.1 개요·접근·호환 규칙
- **정의**: 악기의 일부를 화면에 재현한 뷰. **클릭으로 악보에 노트를 직접 입력**한다.
- **접근 (View 메뉴 3개 항목)**:
  - `View > Show fretboard view`
  - `View > Show keyboard view`
  - `View > Show drumkit view`
- **툴바**: 악기 뷰 아이콘들이 **툴바 우측**에 위치(원클릭 토글). 툴바 그룹 12 "Instrument views".
- **단축키(문맥 의존 단일 단축키)**: **`Ctrl+F6`(Win) / `⌘F6`(macOS)**
  - 현악(stringed) 트랙 → **프렛보드** 표시/숨김
  - 비현악 유음정(pitched) 트랙 → **키보드** 표시/숨김
  - 퍼커션 트랙 → **드럼킷 뷰** 표시/숨김
- **트랙 유형 ↔ 뷰 호환**:
  - 유음정 트랙(현악 여부 무관: 기타·베이스·피아노·신스·관현악·유음정 타악 등) → **프렛보드와 키보드 뷰만** 표시 가능
  - 무음정(순수 타악) 트랙 → **드럼킷 뷰만** 가능
  - **프렛보드와 키보드 뷰는 동시 표시 가능** (같은 악보를 두 악기 위에서 동시에 시각화)

### 5.2 공통 기능 (3개 뷰 전부)
- 악기 위를 **마우스 클릭해 노트 입력**; **기존 노트 클릭 시 삭제**(토글).
- 뷰 내장 **이전/다음(Previous/Next) 버튼**으로 악보 커서 이동(키보드 불필요).
- 뷰 좌측의 **기어(설정) 버튼** → 옵션 팝업.
- **표시 범위(display scope) 3모드**:
  | 모드 | 표시 내용 |
  |---|---|
  | Show Beat | 선택된 비트의 노트만 |
  | Show Beat + Bar | 선택 비트 + 활성 마디의 모든 노트 |
  | Show Beat + Next Beat | 선택 비트 + 다음 비트의 노트 |

### 5.3 프렛보드 뷰 (Fretboard View)
- **용도**: 악보 노트 열람, 노트 입력, 스케일 시각화.
- **호버 피드백**(클릭 없이 2곳 표시):
  1. 마우스 아래 악기 위 직접 표시(옵션에서 구성 가능)
  2. 뷰 **좌측 영역**에 노트명 표시 — **숫자가 옥타브**(예: `E2` 형식)
- **UI 요소**: (1) 커서 좌/우 이동 버튼 (2) 설정(기어).
- **트랙 설정 자동 반영**: **현 수·카포 위치**를 트랙 설정에서 상속; 활성 트랙 변경 시 프렛보드 자동 갱신.
- 노트 추가: 원하는 **프렛+현 클릭** / 삭제: 다시 클릭.
- **기본 방향 = 오른손잡이**; 설정 팝업에서 **왼손잡이 모드** 토글 (왼손잡이 코드 다이어그램 미지원의 공식 우회책).
- **우클릭 = 노트 추가 + 커서 우측 전진** (스케일 기반 고속 입력, 2.3절).
- **웹앱 시사점**: 프렛보드 컴포넌트는 `tuning[], stringCount, capo, leftHanded, scaleOverlay?, displayScope` props를 받고, 클릭 → `(string, fret)` → 악보 커서 비트에 노트 커밋 이벤트를 발행.

### 5.4 키보드 뷰 (Keyboard View)
- **UI 요소**: (1) 커서 좌/우 이동 (2) 설정.
- **현재 트랙의 가용 음역 시각화**: 악기 음역보다 너무 낮거나 높은 건반은 **회색 처리 + 클릭 불가**.
- 건반 클릭으로 노트 추가 / 재클릭으로 삭제.
- 스케일 오버레이 지원(2.4절), 루트=사각형 강조.

### 5.5 드럼킷 뷰 (Drumkit View)
- **접근**: `View > Show Drumkit View`, 퍼커션 트랙 활성 시 **Ctrl+F6 / ⌘F6**.
- **UI 요소**: (1) 커서 좌/우 이동 (2) 드럼킷 설정 버튼.
- **레이아웃**: 상단 = **Staff view(기보 뷰)** / 하단 = **General MIDI 목록(MIDI view)**; 설정에서 두 서브뷰 각각 표시/숨김 선택.
- 요소 클릭 → 트랙에 즉시 입력.
- **Staff view 동작**:
  - 요소 선택 시 해당 요소의 **숫자 키패드 단축키를 표시**(입력 가속).
  - 가용 키패드 단축키는 **커서가 위치한 보표 라인에 따라 달라짐**.
  - 호버 시 클릭 없이 **드럼 요소 이름 + MIDI 값** 표시.
- **MIDI view 동작**:
  - **드럼킷의 모든 연주 가능 요소** 시각화.
  - **표준 General MIDI 팔레트를 넘어서는 확장 요소 팔레트** 제공.
  - 설정(기어) 옵션: **General MIDI 요소만 표시** 필터.
- **노트명 표시 옵션 3종**: ① 요소 이름 숨김 ② **호버 시에만** 표시 ③ **전체 표시**(이름+MIDI 값 모두).
- **MIDI 요소 텍스트 크기** 설정 팝업에서 조정 가능.
- **관련 입력 방식**: 드럼 라인별 다중 아티큘레이션(예: 하이햇 open/semi-open/closed)은 **숫자 키패드 1/2/3**으로 순환 선택; 태블러처에 **MIDI 번호 직접 입력** 가능; MIDI 모드 퍼커션 = 채널 10 고정. (드럼 MIDI 27~87 매핑표는 악보 작성 도메인 참조)
- **웹앱 시사점**: 드럼킷 뷰는 SVG 킷 그래픽 + GM(+확장) 요소 맵. 요소별 `{ name, midi, numpadKey(라인 문맥 의존), articulations[] }` 데이터.

## 6. 튜너 (Tuner)

- **정의**: 내장 **프로페셔널 폴리포닉 튜너**. **툴바 아이콘**(그룹 13)으로 팝업 열기.
- **레이아웃**: 팝업 **하단 = 폴리포닉 튜너**, **상단 = 모노포닉 튜너**.
- **폴리포닉 동작**: **모든 현을 한 번 스트럼(brush)** 하면 **전체 현의 튜닝 상태를 동시에 체크**.
- **트랙 연동**: 튜너는 **현재 트랙의 튜닝으로 자동 설정**됨 (튜닝 창/트랙 인스펙터의 튜닝이 기준).
- **표시 규칙**:
  - 튜닝 노트는 **왼쪽→오른쪽으로 가장 굵은(낮은) 현→가장 가는 현** 순으로 배치.
  - 편차는 목표값 대비 계산·표시, **단위는 톤(tone)**.
  - 현마다 **빨간 바(red bar)** — 중앙에 가까울수록 목표 피치에 근접, **정튜닝 도달 시 초록색(green)** 으로 변함.
- **모노포닉 모드**: 단일 노트 연주 → 상단부에 감지·표시.
- **선택 연동**: 폴리포닉 표시부의 특정 노트 **클릭 → 해당 노트가 상단(모노포닉) 부에 선택 표시**.
- **권한**: GP8에서 **마이크 접근이 필요한 기능은 튜너와 Line-In 딱 2개** (녹음 기능 없음 — 아무것도 저장되지 않음). Windows 마이크 개인정보 설정 허용 필요.
- **관련**: 튜닝 창·트랙 인스펙터의 튜닝에는 **재생(청음) 버튼**이 별도로 존재(튜닝을 소리로 확인).
- **웹앱 시사점**: `getUserMedia` + AnalyserNode/자기상관(또는 다중 피치 추정) 기반. 폴리포닉은 현별 목표 주파수 근방 밴드 검출로 근사 구현 가능. 상태 모델 = `{ targetTuning[] (트랙에서 파생), perString: { deviation(tone 단위), inTune } }`.

## 7. 파일 보호 (File Protection — File > Lock/Unlock)

- **접근**: 메뉴 `File > Lock/Unlock` → 잠금/해제 창.
- **보호 옵션 3종**:
| 옵션 | 동작 | 비밀번호 |
|---|---|---|
| **Finalize** | 파일을 확정하여 원치 않는 수정을 방지. 잠금 해제만 하면 다시 수정 가능 | 불필요 |
| **Lock editing** | 모든 수정 금지(열람·재생은 가능). 용례: 교사가 학생 배포용 파일 수정 방지 | **필요** |
| **Lock opening** | 비밀번호 없이는 **파일 열기 자체 불가**. 용례: 온라인 레슨 판매 | **필요** |
- **경고(매뉴얼 명시)**:
  - 비밀번호 분실 시 **복구/열람 불가** — 반드시 별도 보관.
  - 암호화는 복잡한 알고리즘 없는 **소프트 보호** — 절대적 보안 아님.
- **탭 바 연동**: 잠긴(비밀번호) 파일은 문서 탭 **좌측에 자물쇠 아이콘** 표시; **자물쇠 클릭 시 비밀번호 입력으로 잠금 해제** 가능.
- **웹앱 시사점**: 파일 포맷에 `protection: { mode: 'finalized'|'lockEdit'|'lockOpen', passwordHash? }` 필드. lockOpen은 콘텐츠 대칭 암호화, lockEdit은 읽기전용 플래그+비번 검증으로 구현.

## 8. Fretlight 지원 (Fretlight Guitar Integration)

- **정의**: 프렛보드에 LED가 내장된 Fretlight 기타(fretlight.com)와 **네이티브 호환** — 운지 위치를 기타 LED로 점등. GP8에서는 추가 요금 없음.
- **활성화 절차**:
  1. **Fretlight 버튼은 기본 숨김**.
  2. 환경설정(`File > Preferences` Win / `Guitar Pro 8 > Preferences` macOS) → **Interface 탭 → "Show Fretlight button" 체크**.
  3. **툴바 우상단 모서리**에 아이콘 등장(툴바 그룹 15) → 클릭 시 Fretlight 설정 팝업.
- **설정 팝업 옵션**:
| 옵션 | 상세 |
|---|---|
| Enable/disable (팝업 우상단) | 기타 LED 표시 토글. **비활성화해도 설정은 저장**되어 재활성화 시 복원 |
| Device | 연결 장치 선택 — **USB** 연결 또는 **Bluetooth** 연결 장치 목록 |
| Mode (기타에 표시할 내용) | **Tablature**: 커서 위치를 따라 태블러처 노트만 점등 / **Chords**: 커서 위치에 코드가 있을 때 코드만 점등 / **Scale**: **Scale 창에서 선택한 스케일**의 노트만 점등 |
| Left-handed | 체크박스 — 왼손잡이용 미러 점등 |
| Fret range | 표시 프렛 구간 제한 — LED 점등 허용 **첫/끝 프렛** 선택 |
| USB pedal buttons assignation | Fretlight USB 페달 버튼에 자주 쓰는 액션(제공 목록 중) 할당 → 마우스 없이 GP 제어 |
- **Fretlight Wireless(Bluetooth) 요구사항**: **Windows 10** 또는 **macOS 10.10+**, **Bluetooth 4.0** 이상. 활성화 후 **약 10초 동기화**로 자동 감지, 환경설정 창에 **"Connected"** 표시.
- **문제해결 체크리스트**: OS 최소사양 확인 → BT4.0 존재·활성 확인 → 기타가 다른 기기/프로그램에 이미 연결돼 있지 않은지 확인 → **micro-USB 포트 옆 검정 버튼**을 전체 LED가 깜빡일 때까지 눌러 리셋 → GP와 파일을 열면 자동 연결.
- **웹앱 시사점**: WebUSB/Web Bluetooth로 LED 프로토콜 전송 계층 추상화. 재생 커서 이벤트 스트림 → mode별 필터(tab notes / chord / scale set) → fret range·left-handed 변환 → 디바이스 전송.

## 9. 커맨드 팔레트 (Command Palette) 및 하위 모드

### 9.1 개요·접근·기본 UI
- **정의**: 타이핑으로 편집/액션을 수행하는 커맨드라인형 도구 (GP8 신기능, 편집 가속 목적).
- **접근**: 메뉴 `Tools > Show Command Palette` / 단축키 **`Ctrl+E`(Win) / `⌘E`(macOS)**.
- **위치**: **열린 파일 탭 바 바로 아래, 악보 상단**에 표시.
- **초기 상태**: 빈 입력창 + 플레이스홀더 문구 *"Type ? to show the commands list; up or down to browse the command history"*.
- **기본 인터랙션**:
  | 입력 | 동작 |
  |---|---|
  | 일반 텍스트 | 이름 전체/일부로 GP8의 모든 옵션·액션 검색 (입력할수록 실시간 정제되는 증분 검색) |
  | `?` | 사용 가능한 prefix/커맨드 목록 표시 |
  | `@` | **Action List** 열기 (액션 표시·트리거) |
  | `>` | **Expression Text** 모드 열기 |
  | `↑` / `↓` | 제안 목록 탐색 + **명령 히스토리** 탐색, 옵션 값 스크롤(예: 벤드 값) |
  | `Tab (↹)` | 명령 자동완성 / **다음 옵션 패널로 확장** |
  | `Enter` | prefix 선택 확정 / **최종 명령 적용(실행)** |
  | 파란 제안 클릭 | 확정/적용 (Tab·Enter와 동일) |
- **UI 상세 (충실 클론용)**:
  - 제안 목록은 **알파벳순**, 2열 구성: **좌 = 입력할 prefix / 우 = 명령 설명**.
  - **활성 제안은 파란 하이라이트 바**; 완성 대상 제안은 **파란색** 표시.
  - 미완성 옵션 플레이스홀더(`bar-count`, `pattern` 등)는 **이탤릭** 표시.
  - 현재 빌드 중인 명령의 **사용법 도움말이 파란 바 아래**에 표시.
  - **회색 처리된 항목** = 현재 악보 선택 상태에서 사용 불가(문맥 감응 활성화). 흰색 = 사용 가능.
  - 옵션 내 **현재 상태 표시**(예: clef 명령은 현재 음자리표를 목록에 표시).
  - 추가 옵션 패널이 있는 항목에는 **탭 아이콘(↹)** 이 표시됨 — Tab 반복으로 다단계 패널 진행. (매뉴얼 권장: 옵션 누락 방지를 위해 **Tab 우선 사용**, 최종 실행만 Enter/클릭)
  - 이미 적용된 토글 효과 재적용 시 **제거 여부 확인 프롬프트** (예: Palm Mute).
  - 패턴 입력 시 **비트가 하나씩 실시간으로 편집·프리뷰**됨 (확정 전 라이브 반영).
- **자동 닫힘 / flow 모드**: 기본은 명령 적용 후 팔레트 자동 닫힘. `flow` prefix(**Toggle flow mode**)로 연속 편집 간 팔레트 유지.
- **다국어 동작**: prefix·명령은 앱 언어와 무관하게 **항상 영어로 표시**. 단, 내부 번역으로 **사용자 언어로 검색 가능** (예: 프랑스어 UI에서 "accord" 입력 → `chord` prefix 표출).
- **웹앱 시사점**: 모든 메뉴 액션·편집 명령을 `{ prefix, i18nAliases[], contextPredicate, optionPanels[] }` 레지스트리로 인덱싱한 fuzzy-search 팔레트. 상태 머신: 검색 → prefix 확정 → 옵션 패널 체인 → 실행.

### 9.4 패턴 명령 (Patterns) — 9종
- 일부 고급 명령은 **패턴(문자 시퀀스)** 과 결합 — 문자가 비트 단위로 순차 적용되며 입력 즉시 라이브 프리뷰.
- 선택 범위(여러 마디·트랙 전체)에 적용 가능: **팔레트 열기 전 선택** → 패턴 입력 → 선택 전체에 즉시 표시 → Enter/클릭 확정.
- 선택이 없으면 커서부터 악보 끝까지 적용.
- 패턴 사용법 안내는 항상 **파란 바 아래**에 표시.
- 예: `pickstroke` + `dduddud` = down, down, up, down, down, up, down; `pickstroke ud`를 선택에 적용하면 업/다운 교대 피킹이 선택 전체에 체인됨.

| Prefix | 설명 | 패턴 문법 |
|---|---|---|
| `arpeggio` | Set arpeggio pattern | `u`=up, `d`=down, Space=없음 |
| `brush` | Set brush pattern | `u`=up, `d`=down, Space=없음 |
| `golpe` | Set golpe pattern | `f`=finger, `t`=thumb, Space=없음 |
| `left hand` | Set left hand pattern | `0 1 2 3 4 T` ; `-`=노트 건너뜀; Space=비트 건너뜀 |
| `picking` | Set automatic picking pattern | `Alternate`(업/다운 교대) 또는 `Economy`(동일 방향) |
| `pickstroke` | Set pickstroke pattern | `d`=down, `u`=up, Space=없음 |
| `right hand` | Set right hand pattern | `p i m a c` ; `-`=노트 건너뜀; Space=비트 건너뜀 |
| `slap pop pattern` | Set slap/pop pattern | `s`=slap, `p`=pop, Space=없음 |
| `wah` | Set wah wah pattern | `o`=open, `c`=closed, Space=없음 |

> ※ 매뉴얼 표에는 arpeggio/brush/pickstroke 행이 "u = ↓, d = ↑"로 인쇄되어 있으나, 본문 서술은 명시적으로 **d=downwards, u=upwards** — 조판 오류로 판단, 클론은 **u=up, d=down** 매핑 채택.

### 9.5 Expression Text (하위 모드)
- **접근**: 메뉴 `Tools > Expression Text` / 단축키 **`⇧⌘E`**(macOS; Windows는 Shift+Ctrl+E 상당) / 팔레트에서 **`>`** 입력.
- **동작**: 팔레트와 동일하되 **prefix 불필요** — 항목 이름을 직접 입력해 즉시 적용. 예: `>Cm` → C minor 코드 적용.
- **편집 가능 요소(전체)**: **Chords / Dynamics(뉘앙스) / Key signatures / Octave signs / Clefs / Directions**.

### 9.6 Action List (하위 모드)
- **접근**: 메뉴 `Tools > Action List` / 단축키 **`⌥⌘E`**(macOS; Windows는 Alt+Ctrl+E 상당) / 팔레트에서 **`@`** prefix.
- **동작**: **탐색기(explorer)처럼** 앱의 모든 메뉴·서브메뉴를 키보드로 탐색·트리거. 이름 일부 타이핑 = 필터 (예: "fretboard" → `View > Show Fretboard View` 액션 실행, "multitrack" → 멀티트랙 뷰 토글).
- **노출되는 최상위 메뉴 12개(= 앱 전체 메뉴 바)**: **File, Edit, Track, Bar, Note, Effects, Section, Tools, Sound, View, Window, Help**.
- **웹앱 시사점**: 메뉴 트리와 액션 레지스트리를 동일 소스로 관리하면 Action List는 자동 파생됨.

## 11. 키보드 단축키 전체 레퍼런스 (Appendix + 본문 산재분 통합)

### 11.1 단축키 정책 (매뉴얼 명시)
- 상당수 기능이 단축키로 접근 가능하며, 일부 기능은 **복수의 전용 단축키**를 가짐.
- **단축키는 커스터마이즈 불가** — 단, 예외 2건:
  1. 노트 길이 변경 `+`/`-` 키는 **Preferences에서 동작 반전 가능** (Interface 탭 "Plus/minus keys behavior": 기본 `+`=음가 길게(4분→2분), `-`=짧게; 반전 옵션 제공. ※ 부록 표는 mac `-`=Increase/`+`=Decrease로 인쇄 — 11.4 참조).
  2. Fretlight **USB 페달 버튼**에는 액션 할당 가능(8장).
- **디스커버리 UI**: 단축키 보유 버튼에 호버 시 **툴팁으로 단축키 표시**; 메뉴 항목 옆에도 단축키 병기. → 클론에서도 툴팁/메뉴 병기 필수.

### 11.4 알려진 표기 충돌·주의사항 (클론 구현 결정)
1. **Scale Diagram 단축키**: 본문 일부에 `⇧F`로 인쇄, 부록은 `⇧S`(Scale Diagram)·`⇧F`(Audio Note Settings). → **⇧S = Scale Diagram, ⇧F = Audio Note Settings 채택**.
2. **`+`/`-` 음가 키**: 본문(노트 입력) 설명은 "`+`/`-`로 음가 변경, Preferences에서 커스터마이즈(반전) 가능"; Interface 환경설정 기본값 서술은 `+`=음가 길게. 부록 표는 mac `-`=Increase / `+`=Decrease로 인쇄됨. → **Interface 환경설정 서술(+=길게, 반전 옵션 제공)을 정본으로 채택**.
3. **Move Voice의 Windows 열**(Alt+4↔voice1, Alt+1↔voice4)은 인쇄 순서 오류로 추정 — 본문 규칙 **`Alt + <보이스 번호>`** 채택.
4. **패턴 문자 방향**(u/d): 9.4절 주석 참조 — u=up, d=down 채택.
5. 팔레트 계열 단축키(⌘E/⌥⌘E/⇧⌘E)는 부록에 mac 표기만 존재 — Windows는 Ctrl 대응으로 구현(본문에서 Ctrl+E 확인됨).
6. Full Screen: 부록은 양 플랫폼 F11, 본문은 macOS `⌃⌘F` 병기 → **둘 다 등록**.
- **웹앱 시사점**: 브라우저 예약 단축키(Ctrl+N/W/T/P, F11 등)와 충돌 — PWA/Electron 래핑 또는 대체 바인딩 레이어 + "단축키는 고정, 툴팁·메뉴 병기" 정책 유지. 키맵은 `{ action, mac[], win[], context }` 테이블 단일 소스로 관리하고 문맥(팔레트/오디오트랙/핑거링 창 포커스/드럼 트랙)별 스코프 처리 필수.

## 12. 도구 공통 설계 원칙 (매뉴얼 명시 사항 요약)

1. **모든 도구는 활성 트랙 속성에 종속**: 코드 엔진·스케일 도구(scale analyst)·튜너의 결과는 항상 **현재 트랙에 선택된 튜닝과 일치**. → 전역 상태 `activeTrack.tuning`(현 수 3~10, 카포 포함)을 모든 도구가 구독하는 반응형 구조 필요.
2. **악기 뷰 언급 4종**(기능 개요: guitar, bass, piano, drums) — 실제 구현 뷰는 3종(프렛보드/키보드/드럼킷)이며 기타·베이스는 프렛보드 뷰가 트랙 튜닝·현 수로 커버.
3. **마이크 소비자는 튜너·Line-In 2개뿐**, 녹음 기능 없음 (권한 요청 UI 설계 근거).
4. **툴바 반응형**: 창 폭에 따라 도구 버튼(튜너, 악기 뷰, Fretlight 등) 숨김 — 숨겨진 액션은 항상 메뉴(및 Action List)로 접근 가능해야 함.
5. **도구 간 상호 배제/의존 규칙**: 오디오 트랙 존재 시 Fixed BPM 불가; Transposition tonality는 Concert tone OFF에서만 유효; Fretlight Scale 모드는 Scale 창 선택에 의존 — 상태 머신으로 명문화할 것.
