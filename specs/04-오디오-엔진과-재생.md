# 도메인 사양서: 오디오 엔진과 재생 (Guitar Pro 8 클론 웹앱)

> 근거: Guitar Pro 8 공식 매뉴얼(334p) 중 Chapter 5 "Audio in Guitar Pro"(pp.169–216), Score playback(pp.170–181), Preferences > Audio/MIDI(pp.307–316), Appendix Signature Sounds/Effects(pp.312–323), 및 관련 신기능 소개(pp.19–40) 통합.
> 범위: 스코어 재생(트랜스포트/루프/속도/카운트인/메트로놈/템포), 사운드보드(믹서), 트랙 사운드(사운드뱅크/시그니처 사운드/이펙트 체인/프리셋/MIDI 채널), 스코어 사운드(마스터링), MIDI 설정, 라인인, 오디오 트랙, RSE vs MIDI, 부록 시그니처 사운드·이펙트 목록.

## 1. 오디오 엔진 개요 — RSE vs MIDI

### 1.1 RSE (Realistic Sound Engine)
- **동작**: 전문 스튜디오에서 녹음된 샘플 + 모델링된 앰프/이펙트 유닛 사운드로 기타를 포함한 약 100여 종 악기(베이스, 스트링, 피아노, 드럼 등)를 사실적으로 재생.
- **수치**: 라이선스된 사운드뱅크 **200개 이상**, 사운드 템플릿(사운드뱅크+이펙트 체인 조합 프리셋) **1000개** 제공.
- **웹앱 구현 시사점**: Web Audio API 기반 샘플러(사운드폰트/샘플 세트) + 이펙트 그래프(AudioNode 체인)로 구성. 사운드 템플릿은 `{bankId, effectChain[]}` JSON 프리셋으로 모델링.

### 1.2 RSE·MIDI 혼합 재생
- **동작**: 한 스코어 안에서 일부 트랙은 RSE, 일부 트랙은 MIDI로 **동시에 혼합 재생 가능**. 트랙별 토글은 **Track Inspector**(우측 패널)의 사운드 헤더에서 수행.
- **전체 일괄 전환 메뉴**:
  - `Sound > Switch all tracks to MIDI`
  - `Sound > Switch all tracks to RSE`
- **MIDI 레이턴시 보정**: 하드웨어에 따라 MIDI가 RSE와 어긋날 수 있으므로 **양수/음수 레이턴시 값**을 Preferences(Windows: `File > Preferences`, macOS: `Guitar Pro 8 > Preferences`)의 **Audio/MIDI 탭**에서 조정. 메트로놈은 항상 MIDI 출력을 사용하므로 레이턴시 튜닝의 기준으로 삼을 수 있음.
- **환경설정 강제 옵션**: `Preferences > General > "Force playback with MIDI"` — 모든 스코어를 열거나 만들 때 전 트랙을 MIDI 모드로 강제.
- **웹앱 구현 시사점**: 트랙 모델에 `engine: "RSE" | "MIDI"` 필드. 웹에서는 MIDI 출력을 Web MIDI API로, RSE를 Web Audio로 구현하고 두 경로 간 지연 오프셋(ms) 설정을 둠.

### 1.3 MIDI 모드의 특성
- MIDI 재생 시 Guitar Pro는 **음높이(pitch)와 길이(duration) 정보만** 사운드 디바이스로 전송 — 음질은 사용자 장비(사운드카드/신디사이저)에 의존.
- MIDI 모드 악기 목록은 **General MIDI(GM) 표준**으로 고정(확장 불가).
- **페달보드, 드럼 믹스 등 RSE 전용 기능은 MIDI 트랙에서 비활성화**됨.

## 2. 스코어 재생 (Score Playback / Transport)

### 2.1 재생 시작·정지
- **시작 방법**: ① `Sound` 메뉴의 재생 옵션 ② **Space(스페이스바)** ③ 툴바 재생 버튼.
- **재생은 항상 편집 커서 위치에서 시작**.
- **선택 영역 재생**: 마우스로 구간을 먼저 선택한 뒤 재생하면 해당 부분만 재생됨.
- **재생 중 클릭 이동**: 재생 중 스코어를 클릭하면 **재생을 멈추지 않고** 클릭 지점부터 이어서 재생.
- **웹앱 구현 시사점**: 재생 스케줄러는 커서 tick 위치에서 시작하는 lookahead 방식(Web Audio clock)으로 구현하고, seek(클릭) 시 재스케줄만 수행.

### 2.2 툴바 트랜스포트 버튼 (좌→우 순서)
1. 첫 마디로 이동 (Go to first bar)
2. 이전 마디로 이동
3. **재생/정지 토글** (Launch/stop playback)
4. 다음 마디로 이동
5. 마지막 마디로 이동
- 툴바 중앙의 **LCD 디스플레이**: 현재 트랙명 표시(클릭 시 트랙 전환 팝업), 곡 위치/템포 판독, 마디의 실제 길이 vs 이론 길이 표시.

### 2.3 재생 단축키 (부록 확정판)

| 동작 | Windows | macOS |
|---|---|---|
| 재생/일시정지 (Play/Pause) | `Space` | `Space` |
| 처음부터 재생 (Play From The Beginning) | `Ctrl+Space` | `⇧Space` |
| 빨리감기 (Fast Forward) | `Ctrl+Right` | `⌘→` |
| 되감기 (Rewind) | `Ctrl+Left` | `⌘←` |
| 한 스텝 앞으로 (Forward) | `Alt+Right` | `⌥→` |
| 한 스텝 뒤로 (Step Backward) | `Alt+Left` | `⌥←` |
| 루프 토글 (Toggle Loop) | `F9` | `F9` |
| 상대 속도 (Relative Speed) | `Ctrl+F9` | `⌘F9` |
| 자동화 편집 (Edit Automations) | `F10` | `F10` |
| 오디오 노트 설정 (Audio Note Settings) | `⇧F` | `⇧F` |
| 재생 중 속도 증가 | `+` | `+` |
| 재생 중 속도 감소 | `-` | `-` |

### 2.4 재생 커서와 뷰 추적
- **재생 커서 스타일** (`Preferences > Interface > Score` 섹션, 4옵션):

| 옵션 | 동작 |
|---|---|
| None | 재생 중 커서 표시 없음 |
| Smooth | 비트 간 부드럽게(연속적으로) 이동 |
| Each note | 비트마다 점프 이동 |
| Metronome | 메트로놈 틱마다 이동 |

- **현재 재생 마디 하이라이트**: 재생 중인 마디를 **연노랑(pale yellow)** 으로 강조(체크박스로 해제 가능).
- **자동 스크롤(auto-follow)**: 재생 시 커서를 따라 뷰가 자동 스크롤. 재생 중 사용자가 수동 스크롤하면 auto-follow **일시 중단**되고, 워크스페이스 **좌상단에 "Resume auto-scroll" 버튼**이 나타남 → 클릭 시 커서 위치로 복귀.
- **웹앱 구현 시사점**: requestAnimationFrame 기반 커서 렌더링 + 스크롤 이벤트로 auto-follow 상태 관리. 커서 스타일은 보간(interpolation) 모드 enum으로 처리.

### 2.5 루프 재생
- **동작**: 재생 전 마우스로 구간을 선택하고 **툴바의 루프 아이콘을 활성화**하면 해당 구간이 반복 재생됨. 루프 옵션은 툴바 또는 `Sound` 메뉴에 존재.
- **단축키**: `F9`.
- **웹앱 구현 시사점**: 루프 구간은 선택 범위(startTick~endTick)로부터 파생; 스케줄러가 endTick 도달 시 startTick으로 재스케줄. 카운트인 "루프 간 반복" 옵션(§3)과 연동.

### 2.6 재생 속도 도구 (Speed Settings)
- **접근**: 툴바의 **Speed settings 아이콘**. 활성 시 툴바 컨트롤이 **파란색**으로 표시; 같은 버튼 재클릭으로 해제(원래 템포 복귀). **파일을 수정하지 않는 비파괴적** 재생 오버레이.
- **3가지 모드**:

| 모드 | 동작 | 수치/옵션 |
|---|---|---|
| **Relative speed (상대 속도)** | 현재 템포에 속도 계수를 곱해 재생 | 기본 프리셋 **7개 (25%~100%)**; Speed Settings 창에서 커스텀 **10%~300%** |
| **Fixed BPM (고정 템포)** | 스코어의 템포 자동화를 무시하고 **전곡을 단일 고정 속도**로 재생 | 기본 프리셋 **5개(현재 템포 포함)**; 커스텀 **10~300 bpm**; 드롭다운의 "custom values" 항목으로 임의값 입력. **오디오 트랙이 있는 파일에서는 사용 불가** |
| **Progressive speed (Speed Trainer, 점진 속도)** | 루프된 구간을 **반복할 때마다 속도를 증가** | **시작값/종료값 각각 10~300%**, **반복 횟수**, **스텝 계수(step coefficient)** |

- **재생 중 실시간 속도 변경**: 재생 중 `+`/`-` 키 →
  - 상대 속도 모드: **5% 단위** 증감
  - 고정 템포 모드: **5 bpm 단위** 증감
- **웹앱 구현 시사점**: 속도는 스케줄러의 시간 배율(playbackRate 계수)로 적용하며 파일 데이터와 분리된 세션 상태로 유지. Speed Trainer는 루프 pass 카운터에 따라 계수를 선형 증가.

### 2.7 템포와 템포 자동화
- **초기 템포 설정/템포 변경 프로그래밍**: `Edit > Automations > Edit Automations`(단축키 **F10**) → 자동화 창. **툴바 LCD의 템포 값을 클릭**해도 동일 창이 열림.
- **LCD는 재생 중 실제 현재 템포를 실시간 표시**.
- **수치**: 템포는 **소수점 값 허용**(예: 100.5 bpm). GP8부터 **30 bpm 미만도 허용**(구버전 하한 30 폐지). 스타일시트에서 템포 소수점 표시 여부 토글 가능.
- 각 템포 자동화 포인트에는 스코어에 표시되는 **라벨**을 붙일 수 있음. 템포 자동화는 **마스터(전 트랙) 레벨 전용**.
- 스코어 헤더의 템포 텍스트도 클릭 편집 가능(클릭 가능 스코어 요소).
- 커맨드 팔레트 연동: `tempo`(템포 자동화 추가), `hide-tempo-automation`/`show-tempo automation`(표시 토글), `relative-speed`.
- **웹앱 구현 시사점**: 템포 맵(tick→bpm, float)을 재생 스케줄러의 시간 변환 함수로 사용. 오디오 트랙 활성 시 템포 맵 대신 싱크 포인트가 우선(§12).

## 3. 메트로놈과 카운트인 (Count-in)

### 3.1 메트로놈
- **활성화**: `Sound > Metronome` 메뉴, 또는 툴바 중앙 **LCD의 메트로놈 토글 버튼**.
- **메트로놈 사운드는 항상 MIDI 출력을 통해 재생**됨(RSE/MIDI 레이턴시 동기화의 기준으로 활용 가능).

### 3.2 카운트인 (Countdown)
- **활성화**: `Sound > Countdown` 메뉴, 또는 LCD의 카운트인 토글 버튼.
- **동작**: 재생 시작 전에 **1마디 이상**의 카운트다운을 재생(악기를 잡고 템포를 체득할 시간 제공).

### 3.3 LCD의 메트로놈/카운트인 컨트롤 (3개)
1. 카운트인 on/off
2. 메트로놈 on/off
3. **설정(기어 아이콘)** — 설정 팝오버에서 구성 가능한 항목:
   - 메트로놈/카운트인 **볼륨**
   - **카운트다운 길이**(마디 수)
   - **루프 간 카운트다운 활성화**(루프 pass마다 카운트인 재생)
   - **비주얼 메트로놈 활성화**

### 3.4 비주얼 메트로놈 / 비주얼 카운트다운 (GP8 신기능)
- **동작**: 상단 패널(top panel)에서 **박마다 플래시**로 표시. **카운트다운 중에도 표시 가능**.
- **비주얼 전용 메트로놈**: 메트로놈 **볼륨을 0**으로 설정하면 소리 없이 플래시만 동작.
- **웹앱 구현 시사점**: 메트로놈 상태 = `{enabled, volume(0=시각전용), countInEnabled, countInBars, countInBetweenLoops, visualEnabled}`. 플래시는 오디오 클럭 이벤트에 동기화된 CSS 애니메이션 트리거.

## 4. 사운드보드 (Soundboard / 믹서)

### 4.1 위치와 목적
- **위치**: 화면 하단 **Global View의 좌측 부분**.
- **목적**: 트랙별 오디오 파라미터(악기, 볼륨, 팬 등)의 **초기값(스코어 초기 설정)** 을 설정. 곡 중간의 변화는 자동화로 처리.

### 4.2 트랙 스트립 구성 요소 (매뉴얼 번호 순)
1. 트랙 아이콘(악기 아이콘)
2. 트랙명
3. 멀티트랙 모드에서의 **트랙 표시/숨김**(eye)
4. **Mute** 버튼
5. **Solo** 버튼
6. **트랙 볼륨**(페이더)
7. **트랙 볼륨 자동화** 활성/비활성 버튼
8. **트랙 팬**(노브)
9. **트랙 팬 자동화** 활성/비활성 버튼
10. **트랙 이퀄라이저** 접근 버튼

### 4.3 Solo / Mute 시맨틱
- **Solo** = **헤드폰 아이콘**. 지정한 트랙(들)만 단독 재생. 복수 트랙 동시 솔로 가능.
- **Mute** = **취소선 스피커 아이콘**. 지정 트랙(들)을 무음 처리. 복수 트랙 동시 뮤트 가능.
- 매뉴얼 가이드: 한두 트랙만 듣고 싶으면 Solo, 한두 트랙만 빼고 듣고 싶으면(직접 연주할 파트 등) Mute.
- 부가 역할: Global View의 Mute는 **MIDI 내보내기의 트랙 제외 필터**로도 동작(타 도메인).

### 4.4 볼륨/팬 자동화 배지
- 트랙에 볼륨 자동화가 있으면 볼륨 슬라이더 옆에 **문자 "A"** 배지 표시. 이 배지는 **자동화 활성/비활성 토글**로 동작. 팬 노브 옆에도 동일한 팬 자동화 인디케이터 존재.

### 4.5 트랙 이퀄라이저
- **접근**: 사운드보드 트랙 스트립(항목 10)에서 직접.
- **동작**: 트랙 사운드를 자유롭게 셰이핑, **악기 유형별 EQ 프리셋** 로드 가능. 프리셋 선택 시 **즉시 적용**.

### 4.6 Focus / Unfocus 현재 트랙 (GP8 신기능)
- **위치**: 사운드보드 하단 **마스터 트랙 스트립**, Mute/Solo 버튼 아래의 **노브**.
- **동작**: 현재 선택된 트랙을 다른 트랙 대비 상대적으로 부각/약화:
  - 오른쪽으로 돌림(Focus) → **다른 모든 트랙의 볼륨 감소** (선택 트랙 부각)
  - 왼쪽으로 돌림(Unfocus) → **선택 트랙의 볼륨 감소** (배경화)
- 커맨드 팔레트: `focus`/`unfocus` — **퍼센트 값**으로 설정 가능.
- 오디오 트랙에도 focus/unfocus 적용됨(§12.3).
- **웹앱 구현 시사점**: 마스터 스트립의 bipolar 노브. GainNode 그래프에서 선택 트랙/기타 트랙에 상보적 게인 적용. 선택 트랙 변경 시 즉시 재계산.
- **웹앱 구현(믹서 전반)**: 트랙 상태 모델 `{visible, mute, solo, volume, volumeAutomationEnabled, pan, panAutomationEnabled, eq}` + 마스터 스트립 `{focusPercent}`. 각 트랙 = GainNode + StereoPannerNode + EQ 체인.

## 5. 트랙 사운드 (Track Sound)

### 5.1 사운드의 구조
- **트랙 사운드 = 사운드뱅크(Sound Bank, 악기의 raw "unplugged" 소리) + 이펙트 체인(Effect Chain, 보통 페달 몇 개 + 앰프 1개)**. 이 조합이 완성된 사운드를 생성.
- **트랙 생성 시**: 트랙 생성 마법사에서 악기 템플릿 선택 → 기본 사운드 자동 적용; 같은 창에서 다른 사운드 선택 가능(재생 버튼으로 미리듣기). 생성 후에는 **Track Inspector의 Sound 섹션**에서 변경.
- **하나의 트랙에 복수의 사운드** 보유 가능 → 곡 중간 사운드 체인지(예: 클린 → 디스토션) 구현.

### 5.2 Sounds 헤더 (Track Inspector, 3버튼)
1. **RSE ↔ MIDI 엔진 전환**(트랙별 토글)
2. **사운드 추가 또는 복제**
3. **"A" 버튼** — 사운드 체인지 자동화 창 열기

### 5.3 개별 사운드 항목 UI (4요소)
1. **사운드 라이브러리 접근**(사운드명 클릭)
2. **사운드 옵션 버튼**(복사/붙여넣기 등)
3. **사운드 잠금(Lock)** — 자물쇠/스피커 아이콘. 모든 자동화를 무시하고 이 사운드를 강제 사용
4. **상세 표시/숨김 토글**

상세(Show details) 확장 시 표시 항목:
1. **라벨(Label)** — 사운드 체인지 지점에 스코어에 표시될 텍스트. 기본값 = 사운드명, 자유 편집 가능(권장 예: "w/disto")
2. **사운드뱅크 선택기**
3. **이펙트 체인 목록**
4. **"Show Pedalboard…"**(페달보드 열기)

### 5.4 사운드 라이브러리 / 시그니처 사운드
- 사운드명 클릭 → **수백 개의 사운드** 브라우징. 카테고리별 **표준(Standard) 사운드** 목록 + **"Signatures" 서브메뉴**(유명곡/아티스트 톤에서 영감을 받은 수백 개 추가 프리셋, §14).
- **재생 중 호버 미리듣기(GP8 신기능)**: 재생이 진행되는 동안 사운드 목록의 항목에 **마우스를 올리면 실시간으로 그 사운드가 시청(audition)** 되고, 클릭하면 트랙에 적용. (예: "Nylon Spinosa" 사운드 실시간 프리뷰)
- **웹앱 구현 시사점**: 호버 시 해당 트랙의 신스 체인을 임시 사운드로 hot-swap하고 마우스 아웃 시 원복하는 프리뷰 모드 필요.

### 5.5 뱅크(Bank) 설정
- **뱅크명 클릭** → 다른 뱅크 선택.
- **뱅크 아이콘 클릭**(이름 왼쪽) → 뱅크 파라미터: **Mono/Stereo**, **마이크(mics) 구성**.
- 사운드뱅크 설정 창 구성: ① Instrument settings ② Instrument library ③ Bank options.

### 5.6 이펙트 체인 (Effect Chain)
- 트랙 사운드당 **이펙트 슬롯 6개**. 빈/채워진 슬롯을 선택해 이펙트 추가.
- 슬롯 UI (4요소):
  1. **이펙트 설정**(파라미터 설정 창 열기)
  2. **이펙트 라이브러리**(이펙트 선택)
  3. **바이패스(on/off)** — 슬롯 우측의 전원 버튼
  4. **옵션 버튼** — **Move up / Move down(슬롯 재배치), Copy, Paste, Delete**
- 이펙트 라이브러리는 **앰프 + 페달 + 마스터링 DSP**로 구성(전체 목록 §13).
- 이펙트명 좌측 **아이콘 클릭** 시 해당 이펙트의 **설정 팝업** 열림.
- **웹앱 구현 시사점**: `sound.effectChain: EffectSlot[6]`, 각 슬롯 `{effectType, params, bypassed}`. 각 이펙트는 AudioNode 서브그래프(WaveShaper=디스토션, Convolver=리버브/캐비닛, Delay, BiquadFilter 등)로 팩토리 생성.

### 5.7 페달보드 (Pedalboard, GP8 신기능)
- **개념**: 이펙트 체인의 시각적(스큐어모픽) 복제 — 실제 노브/스위치가 달린 페달·앰프 그래픽을 드래그 조작.
- **접근**: Track/Song Inspector의 이펙트 체인 섹션 아래 **"Show Pedalboard…"**.
- **용량 및 제약**:
  - **현재 트랙: 최대 6개**(페달+앰프)
  - **마스터링: 최대 3개**
  - **RSE 트랙 전용**(MIDI 불가; 필요 시 Sound 메뉴로 엔진 전환)
  - 페달보드는 **선택된 사운드에 종속** — 사운드가 여러 개인 트랙은 사운드마다 별도 페달보드.
- **창 툴바 12개 기능**: ① Mastering/현재 트랙 전환 ② 사운드 변경(현재 트랙 전용, 추가 사운드 존재 시) ③ 페달 삽입/추가 ④ 선택 페달 좌/우 이동 ⑤ 복사 ⑥ 붙여넣기 ⑦ 복제 ⑧ 삭제 ⑨ 폭 맞춤(Fit width) ⑩ 페달 브라우저 표시/숨김 ⑪ 이펙트 카테고리 변경(브라우저 필터) ⑫ 브라우저 줌 인/아웃.
- **페달 추가**: 툴바 `+` 버튼, 인스펙터 이펙트 체인 섹션, 또는 **페달 브라우저에서 드래그&드롭**.
- **페달 삭제**: 창 밖 또는 브라우저 위로 드래그, 혹은 선택 후 **Delete 키**.
- **페달 편집**: **우클릭 → 교체 목록**; 드래그로 순서 변경; 각 페달의 노브/컨트롤/온오프 스위치 개별 조작; 페달마다 **Presets 목록** 보유.
- **실시간성**: 모든 페달/앰프 변경은 **재생 중 실시간 반영**.
- **웹앱 구현 시사점**: Canvas/SVG 기반 보드 + 드래그&드롭, 파라미터 변경 → AudioParam 즉시 갱신. 이펙트 체인 데이터와 페달보드는 동일 모델의 두 가지 뷰.

### 5.8 사운드 체인지 (사운드 자동화)
- **워크플로**:
  1. Sounds 헤더에서 트랙에 사운드 추가/복제
  2. **Sound automations 창** 열기: `Edit > Sound > Sounds automation` 또는 인스펙터의 **"A" 버튼**
  3. "Add an automation at the cursor" 선택 후 적용할 사운드 선택; 위치는 이후 이동 가능
- **빠른 방법**: `Edit > Sounds > <사운드명>` — 커서 위치에 즉시 사운드 체인지 삽입.
- **표시**: 사운드의 **라벨 텍스트가 체인지 지점의 스코어 위에 렌더링**됨.
- **강제(Lock)**: 사운드 항목의 자물쇠 아이콘으로 자동화를 전부 무시하고 단일 사운드 강제.
- **라인인 연동**: 라인인 사용 중에도 커서가 사운드 체인지 지점을 지나면 **자동으로 라이브 입력에 적용**(§11).
- **웹앱 구현 시사점**: 사운드 체인지 = `{tick, soundId}` 이벤트 배열. 재생 스케줄러가 지점 통과 시 트랙의 액티브 신스 체인을 교체(클릭 노이즈 방지용 짧은 크로스페이드 권장).

### 5.9 사운드 프리셋 저장/관리
- **저장**: Sound 헤더의 Options 버튼 → **`Save as user sound preset`** → 이름 지정 + **패밀리(family)** + **악기 유형** 지정 → 해당 카테고리의 사운드 목록에 나타남.
- **관리**: Sound library 버튼 → **`Manage user sound presets…`** → OS 파일 탐색기가 프리셋 폴더를 엶.
- **저장 구조**: 프리셋은 **디스크의 일반 파일**이며 **카테고리/패밀리/악기** 폴더 계층으로 분류. 폴더를 직접 편집하면 앱의 프리셋 목록에 그대로 반영(파일 삭제=프리셋 삭제, 파일 복사=가져오기). 기본 폴더 계층은 존재하나 사용자 프리셋 생성 전까지 비어 있음.
- **경로**(매뉴얼 원문이 "guitarpro7"으로 표기):
  - Windows: `%APPDATA%/Arobas Music/guitarpro7/My Sound Presets`
  - macOS: `~/Library/Application Support/Arobas Music/GuitarPro7/My Sound Presets`
- **웹앱 구현 시사점**: IndexedDB/서버에 카테고리 트리 구조의 프리셋 저장 + JSON 파일로 내보내기/가져오기 지원.

### 5.10 드럼 믹스 (Drum Mix, GP8 신기능)
- **접근**: 드럼 트랙의 **뱅크 설정 아이콘(드럼셋 사운드뱅크 아이콘)** 클릭 → **Drum mix 창**(팝업).
- **기능**: 드럼 킷 **각 구성 요소(킥/스네어/심벌 등)별로**:
  - **Solo(isolate)** / **Mute**
  - **볼륨** 개별 설정
  - **팬** 개별 설정
- 트랙 전체의 **Mono/Stereo 모드** 선택 토글.
- **제약**: **RSE 트랙 전용** — MIDI 모드에서는 사용 불가(숨김/비활성).
- **웹앱 구현 시사점**: 드럼 뱅크를 킷피스별 버스로 분리(`kitPiece → {gain, pan, mute, solo}`), 마스터에 합산 전 개별 처리.

### 5.11 해석 옵션 (Interpretation — 오디오 엔진 관점 요약)
- Track Inspector 4번째 섹션. 재생 렌더링에 직접 영향:
  - **Playing style**(택1): `Pick` / `Finger` / `Picking` / `Bass Slap`
  - **Palm mute** 토글 + **강도 슬라이더**(Track Inspector) — 특정 연주 스타일에서만 가용
  - **Accentuation**: 마디 내 특정 박 자동 강세
  - **Auto let ring**: 기보된 길이와 무관하게 현 공명 유지(트랙 전체); 쉼표가 나오면 모든 현의 let-ring 중단
  - **Auto brush**: 코드 구성음에 미세 시간차(스트럼 시뮬레이션)
  - **Stringed**: 현악기 모드(같은 음높이 동시 공명 허용, 이펙트를 현 단위로 정렬). 끄면 피아노 모드(음높이 단위)
- **드럼 스틱 타입**: Track Inspector에서 스틱 종류 선택 — RSE가 스트로크 타입별로 해석.
- **웹앱 구현 시사점**: 노트 렌더러(스코어→오디오 이벤트 변환기)의 트랙별 해석 파라미터. 대부분의 악보 기호(다이내믹, 벤드, 슬라이드, 트릴, 와우[체인에 Wah 페달 필요], 라스게아도 18모티프, 태핑, 슬랩/팝 등)는 오디오 엔진이 해석 — 단 Pickscrape, Dead Slapped는 표기 전용(비가청).

## 6. 오디오 노트 설정 (Audio Note Settings, GP8 신기능)

- **접근**: `Note > Audio Note Settings` 메뉴 또는 단축키 **`⇧F`(Shift+F)**. 노트 1개 또는 선택 영역에 적용.
- **동작**: 기보와 무관하게 **재생 전용** 미세 조정. 각 수정은 다른 이펙트를 고려하지 않은 **노트의 이론값 기준**.
- **파라미터**:

| 파라미터 | 동작 | 범위/기본값 |
|---|---|---|
| **Duration** | 기보 길이 대비 실제 발음 길이 비율 | **25%~1200%**, 기본 **100%** |
| **Offset** | 기준 박 대비 발음 시점을 앞/뒤로 이동(±) | 수치 범위 명시 없음; 해상도: **4분음표 = 480 ticks** |
| **Relative velocity** | 선택 영역을 주변 대비 크게/작게 | 기본 **0 dB**, 조정 **±1~±15 dB** |

- **Reset 버튼**: 기본값(100% / 0 / 0 dB)으로 복원.
- **예외**: **꾸밈음(grace note)** 은 다음 노트를 자동 추종하므로 개별 설정 불가; **아르페지오는 이 창으로 수정 불가**.
- **웹앱 구현 시사점**: 노트 모델에 옵셔널 필드 `{durationPercent, offsetTicks, relativeVelocityDb}`; 오디오 이벤트 생성 시에만 반영, 엔그레이빙 무영향.

## 7. 자동화 (재생 관점: Tempo / Volume / Pan)

- **자동화 3종**: **Tempo**(마스터 전용, 전 트랙 공통), **Volume**(트랙별 또는 마스터), **Pan**(트랙별 또는 마스터). 팔레트에서 삽입 가능한 자동화 레인은 총 5종: tempo / master volume / master pan / track volume / track pan.
- **접근**: `Edit > Automations > Edit Automations`, **F10**, 팔레트의 자동화 아이콘, (템포는) 스코어의 자동화 마커 클릭.
- **에디터 동작**:
  - 창 열림 시 현재 위치에 포인트가 없으면 **자동으로 포인트 추가**
  - 그래프 클릭으로 키포인트 배치; 포인트는 **박/마디에 자동 스냅**
  - 마우스/화살표 키로 좌우(시간)·상하(값) 이동; **Delete 키로 삭제**
  - **전이(transition) 2종**: "Constant until next point"(다음 포인트까지 유지 후 급변) / "Progressive to the next point"(램프)
  - **Remove Automations** 버튼: 전체 자동화 일괄 삭제
- 자동화 표시/숨김 가능. 트랙 볼륨 자동화 활성 시 사운드보드에 "A" 배지(§4.4).
- **오디오 트랙에는 볼륨/팬 자동화가 적용되지 않음**(§12.3).
- **웹앱 구현 시사점**: 자동화 = 레인별 `{tick, value, transition}[]`. 볼륨/팬은 AudioParam.linearRampToValueAtTime으로, 템포는 스케줄러 시간 맵으로 컴파일.

## 8. 스코어 사운드 — 마스터링 (Score Sound / Mastering)

- **접근**: **Song(Score) Inspector**(우측 패널)의 사운드 마스터링 섹션.
- **구조**: 마스터 버스에 **고정 순서의 3단 직렬 이펙트 체인**:
  1. **Compressor / Limiter**
  2. **Reverb**
  3. **Equalizer**
  - 추가로 마스터링 패널에 **Pedalboard 접근** 항목 존재(마스터 페달보드는 최대 3개).
- **슬롯 동작**: 트랙 이펙트 슬롯과 동일한 UI(설정 / 라이브러리 / 우측 전원 버튼 바이패스 / 옵션) — 단, **각 슬롯의 라이브러리는 그 슬롯의 이펙트 유형으로 제한**(1번엔 컴프레서류만, 2번엔 리버브만, 3번엔 EQ만).
- **마스터 볼륨/팬 자동화**: 마스터 트랙에 볼륨·팬 자동화 설정 가능(전 트랙 동시 영향, §7). 커맨드 팔레트: `master-volume`, `master-pan`.
- **웹앱 구현 시사점**: masterBus = DynamicsCompressorNode → ConvolverNode(리버브) → EQ(BiquadFilter 뱅크) → destination. 슬롯별 타입 필터링된 라이브러리 셀렉터.

## 9. MIDI 설정

### 9.1 MIDI 출력 포트
- **동시 사용 가능 MIDI 포트: 최대 4개**; 포트마다 다른 MIDI 디바이스 지정 가능.
- **포트 1이 신규 트랙의 기본 포트** — 최상의 디바이스를 포트 1에 배치 권장.
- Preferences > Audio/MIDI 탭에서 포트별 **Test 버튼**(샘플 사운드 재생으로 디바이스 확인).

### 9.2 포트·채널 할당 (Instrument 패널)
- 트랙별로 사용할 **포트와 채널**을 Instrument 패널에서 선택. 트랙을 MIDI로 전환하면 Track Inspector에 포트/채널 옵션이 나타남.
- **기본값: 트랙당 MIDI 채널 2개** — 이펙트(벤드 등)를 별도 채널에서 재생하여 이펙트 없는 노트에 영향이 가지 않게 함.
- **채널 공유 규칙**: 두 트랙이 같은 포트의 같은 채널을 쓰면 **악기/볼륨/팬이 서로 연동(tie)** 됨. 채널 드롭다운에 어떤 트랙이 어떤 번호를 쓰는지 표시. 트랙이 많은 스코어에서는 **트랙당 1채널** 권장(일부 외부 소프트웨어의 트랙당 2채널 미지원 호환 문제도 동일 해법).
- **퍼커션 규칙**: 퍼커션 트랙은 **항상 채널 10**(MIDI 규격).
- **"Force 11-16" 옵션**: 트랙의 **현마다 채널 1개씩(채널 11~16)** 사용 — 같은 박에서 현별로 서로 다른 벤드가 있을 때 필수.
- **웹앱 구현 시사점**: 트랙 모델에 `{midiPort(1-4), midiChannels[], force11to16}`. 채널 공유 시 파라미터 연동 로직 필요.

### 9.3 MIDI 캡처 (스텝 입력)
- **접근**: `Sound > MIDI Capture`.
- **동작**: MIDI 키보드/기타/컨트롤러로 **한 박(beat)씩** 노트를 입력. **실시간 녹음이 아님**.
- 설정: `Sound > Audio/MIDI settings` — 디바이스 선택/연결 확인, **캡처 감도**(두 노트를 분리하는 지연시간), **채널 감지·현 할당** 설정.

### 9.4 MIDI 입력 상세 (Preferences > Audio/MIDI)
- **MIDI device**: 캡처용 입력 디바이스 선택(드라이버 설치 + 연결 + 선택 필요).
- **Capture sensitivity**: **같은 코드(화음)로 묶을 두 노트 사이의 최대 지연(ms)**. 이 값을 초과해 도착한 노트는 **새 박**에 배치.
- **String affectation(현 할당)**:
  - MIDI 키보드: 현/프렛 정보가 없으므로 GP가 **프렛보드 상 운지가 최적화되도록 현·프렛 자동 배정**.
  - 채널별 현 전송 MIDI 기타: 모드 2종 — **"First channel for the highest string"** / **"First channel for the lowest string"**.
- **웹앱 구현 시사점**: Web MIDI API onmidimessage → 감도 창(ms) 내 노트 병합 → 화음/새 박 판정 → 현 배정 알고리즘.

## 10. 오디오 환경설정 (Preferences > Audio 탭)

> 이 탭의 모든 설정은 **변경 즉시 적용**(OK 버튼 없음).

| 설정 | 동작 / 수치 |
|---|---|
| **Audio Device** | 연결된 오디오 디바이스 선택. Windows: **ASIO 강력 권장**(Line-In에 필수적), macOS: **CoreAudio** |
| **Audio output** | 출력 대상(스피커/헤드폰). 다중 출력 디바이스는 **두 번째 콤보박스**로 서브 출력 선택. **Check 버튼** = 테스트 사운드 재생 |
| **Audio input** | Line-In 입력 소스(마이크/기타). 우측에 **VU 미터**(실시간 입력 레벨 표시) |
| **Buffer size** | 버퍼 샘플 수 선택. 통상 **512 샘플**; 스트림은 **44100 Hz**. 작게 → 저지연·고CPU(과소 시 클릭/팝/스터터), 크게 → 지연 체감(인지 한계 **~10 ms**) |
| **MIDI Output** | 4포트 × 디바이스 드롭다운 + Test 버튼; **MIDI↔RSE 레이턴시** 오프셋(메트로놈=MIDI 출력 기준) |
| **MIDI Input** | 캡처 디바이스, 감도, 채널→현 매핑(§9.4) |

- **마이크 접근**: GP8이 마이크를 쓰는 기능은 **튜너와 Line-In 딱 2가지**. **녹음 기능은 없음**(아무것도 저장하지 않음).
- **웹앱 구현 시사점**: AudioContext({latencyHint, sampleRate:44100}) + MediaDevices.enumerateDevices 기반 디바이스 셀렉터, AnalyserNode 기반 VU 미터.

## 11. 라인인 (Line-In)

### 11.1 개념
- 실제 기타를 오디오 인터페이스에 연결해 **선택된 트랙의 사운드(프리셋+사운드뱅크+이펙트 체인)를 통해 실시간(무지연) 재생**.

### 11.2 플레이얼롱 워크플로 (매뉴얼 명시 절차)
1. 원하는 기타 트랙 선택
2. 사운드보드(Global View 좌측)에서 그 트랙을 **Mute**
3. 기타 연결 후 **Line In 옵션 활성화**
4. 재생 시작 → 뮤트된 트랙은 무음, 사용자가 그 파트를 라이브로 연주
5. 라이브 입력은 **선택 트랙의 정의된 사운드로 즉시 재생**되며, 스코어에 **사운드 체인지가 있으면 재생 커서 위치에 따라 자동 적용**

### 11.3 하드웨어 연결
- 외장(USB/FireWire) 또는 내장(PCI/PCI-e) 오디오 인터페이스 필요; 기타→인터페이스(잭 케이블), 출력은 인터페이스의 스피커/헤드폰.
- **입력과 출력에 같은 사운드카드 사용을 강력 권장**.

### 11.4 Line-In 팝업 (툴바 우측 아이콘)
- **OFF / ON 상태** 토글. **기어 휠** → Preferences의 Audio/MIDI 탭으로 연결(입력 사운드카드 확인).
- 활성화 후 **실제 기타가 꽂힌 오디오 입력**을 선택해야 함. **피드백(하울링) 경고** 및 볼륨 감소 제안 동작 존재.
- 팝업 설정 항목:

| 설정 | 동작 |
|---|---|
| **Input gain** | 사운드카드 입력 게인 증감 |
| **Effects chain** (체크박스) | 입력을 현재 트랙의 이펙트 체인에 통과시킬지 여부(외부 이펙트 시뮬레이터 사용/튜닝 시 해제) |
| **Noise Gate** (노브) | 배경 노이즈가 사라질 때까지 천천히 올림 |
| **Volume** | 전체 믹스 내 Line-In 악기의 출력 볼륨 |
| **Limiter** | 임계값 초과 사운드의 볼륨을 낮춰 피드백 방지 |

- 이펙트 체인은 언제든 Track Inspector에서 변경 가능.
- **사용 후 반드시 OFF** — 켜두면 입력 신호를 계속 처리하여 시스템 사운드에 잡음이 낄 수 있음(매뉴얼 경고).

### 11.5 사운드 잠금 (Line-In Lock, GP8 신기능)
- Line-In 창의 **자물쇠 아이콘**: 현재 사운드+이펙트 체인을 잠금.
- 잠금 시: 아이콘이 **파란색**, 툴바 Line-In 아이콘에도 자물쇠 표시. **트랙 변경, 파일 변경, 파일 닫기에도 유지**. **Effect Chain 체크 해제나 Line-In OFF에도 잠금 유지**. **앱 재시작 시 해제**.
- 잠금 해제 시 현재 트랙의 이펙트 체인이 다시 적용.
- **웹앱 구현 시사점**: getUserMedia → MediaStreamSourceNode → (게이트→게인→트랙 이펙트 체인→리미터) → master. 잠금 상태는 앱 세션 전역 상태(파일 스코프 아님).

## 12. 오디오 트랙 (Audio Track, GP8 신기능)

### 12.1 개념과 용도
- GP 파일에 **오디오 파일 1개**를 통합 트랙으로 첨부. 용도(매뉴얼 명시): 채보(transcription) 보조, 보컬 트랙 추가, 백킹 트랙, (focus/unfocus와 결합한) 믹싱 보조, 실연 레이어링을 통한 렌더링 향상, 교육용 오디오 코멘터리.

### 12.2 추가/표시
- **추가 방법 3가지**: ① 툴바의 오디오 추가 버튼 ② `Track > Add Audio File…` ③ 앱으로 **드래그&드롭**. (오디오 트랙 창에서 Browse로도 가능)
- **표시/숨김**: 툴바 버튼 또는 `View > Show Audio Track`. 파일에 오디오 트랙이 있으면 **툴바 버튼에 파란 외곽선** 표시.
- **창 위치**: 스코어(작업 영역)와 Global View **사이**.
- **가져오기 형식**: `mpeg / m4a / wav / flac / x-aiff / ogg` (6종).
- 가져오기 성공 시 **웨이브폼** 표시.

### 12.3 스코어와의 통합 시맨틱 (재생 엔진 규칙)
- **오디오 트랙에 영향을 주는 것**: 템포, 상대 조성(relative tonality/transposition), 루프 재생, 카운트인, focus/unfocus, **마스터 트랙 볼륨·팬**, 페르마타·트리플렛 필 등 해석 옵션, 스코어 구조 변경(약박(anacrusis), 박자표 변경 등).
- **영향을 주지 않는 것**: 이펙트 적용, **트랙 볼륨/팬 자동화**, **고정 BPM(fixed tempo)** — 고정 템포는 오디오 트랙 존재 시 아예 **사용 불가**.
- **반복/디렉션 미적용**: 오디오 트랙은 **펼쳐진(unfolded) 선형 뷰**로 표시되며 재생은 항상 **선형**(커서 점프 없음).
- **오디오 트랙 활성 시 싱크 포인트가 스코어 재생 속도를 구동하며, 스코어의 템포 자동화는 무시됨**. 토글 버튼으로 오디오 트랙을 일시 비활성화하면 오디오 미재생 + 스코어 템포 자동화로 복귀. 생성 시 **곡의 첫 템포에 해당하는 첫 싱크 포인트(`[`)가 항상 존재**.
- **웹앱 구현 시사점**: AudioBufferSourceNode의 재생과 스코어 스케줄러를 싱크 포인트 매핑(오디오 시간↔스코어 tick의 조각별 선형 함수)으로 결합. 반복 해석기(unroller)는 오디오 트랙 모드에서 비활성.

### 12.4 오디오 트랙 창 UI
- **가져오기 후 툴바 버튼(8개)**: ① 오디오 트랙 활성/비활성 ② Browse(파일 교체) ③ 오디오 트랙 삭제 ④ 줌 인/아웃 ⑤ **Perform tasks 메뉴(기어)** ⑥ 필터 적용 ⑦ Semitones(반음 이조) ⑧ Cents(센트 미세 이조)
- **표시 정보(6)**: ① 템포/동기화 존 ② 마디 ③ 트랙명 ④ 길이(Duration) ⑤ 싱크 포인트 ⑥ 1마디 블록
- **그리드**: 웨이브폼 위에 곡의 마디 구조를 반영한 격자. **1블록=1마디**, **첫 박 = 빨간 실선**, **나머지 박 = 노란 점선**, 첫 박 위에 **마디 번호** 표시. 그리드는 **박자표에 적응**(예: 6/8이면 마디당 6개 라인).
- **선택 연동**: 스코어에서 여러 마디를 선택하면 오디오 트랙에도 동일 선택이 적용됨(**역방향은 불성립**). 여러 마디 선택은 오디오 트랙의 duration bar에서 수행.
- **줌**: 트랙패드/마우스, 창의 줌 버튼, Perform tasks 메뉴.

### 12.5 브래킷 `[ … ]` (입출력 지점)
- 두 브래킷이 오디오의 **입력/출력 싱크 포인트**를 표시; **재생은 두 브래킷 사이 구간으로 엄격히 제한**(첫 브래킷 이전 오디오는 해석 안 됨).
- 스코어에 마디를 추가하면 해석되는 마디와 브래킷 구간이 그만큼 **확장**(예: 1마디 프로젝트에 3마디 추가 → 4마디 재생). 브래킷 밖의 **"+" 아이콘**으로도 스코어에 마디 추가 가능.
- 브래킷은 **마우스 드래그로 이동** 가능(오디오의 실제 시작/끝을 스코어에 정렬). **브래킷 이동은 그리드를 직접 재조정**.

### 12.6 패딩(Padding) 조정 — 동기화 1단계
- 목적: 오디오의 한 시점과 스코어의 한 시점을 일치(보통 시작점끼리, 임의 시점도 가능).
- **주 방법**: **Ctrl(Win)/Cmd(mac)를 누른 채 웨이브폼을 그리드 위에서 드래그**.
- **대안**: 첫 번째 싱크 포인트 `[` 로 그리드를 웨이브폼 위에서 이동.
- **패딩 리셋**: Perform tasks(기어) 메뉴 또는 오디오 트랙 내 **우클릭** → 첫 싱크 포인트가 오디오 시작점으로 복귀.

### 12.7 싱크 포인트 관리 — 동기화 2단계
- 싱크 포인트 3종: ① 입력 싱크 포인트(`[`) ② 중간 싱크 포인트(삼각형 `▼`) ③ 출력 싱크 포인트(`]`).
- **중간 싱크 포인트는 두 브래킷 사이 구간에서만 추가 가능**.
- 싱크 포인트는 **모든 마디의 첫 박 위에 기본 존재하되 비활성** 상태 — 사용자가 조작해야 활성화.
- **추가**: 박 라인을 **더블클릭** → **흰색 삼각형** 표시(예시에서 활성 포인트는 노란색 강조).
- **제거**: 같은 박을 다시 더블클릭 → 사라지거나(중간), 마디 시작 포인트는 **검은색(비활성)** 으로 표시.
- **이동**: 선택 후 좌우 드래그 → **해당 시점의 템포가 웨이브폼 상 위치와 스코어 구조로부터 자동 계산**되어 상/하향 조정됨. **한 포인트의 이동은 다른 포인트 위치에 절대 영향을 주지 않음**.
- **템포 유지 규칙**: 변경된 포인트의 템포 값은 **다음 포인트까지 유지**; 포인트 **삭제 시 좌측 이전 포인트의 값이 이어받음**.
- **전체 삭제**: Perform tasks 메뉴 또는 우클릭.
- **커서 연동**: 스코어의 임의 박에 편집 커서를 놓으면 **오디오 트랙 커서도 같은 지점으로 이동** → 그 자리에서 더블클릭으로 정확한 박에 싱크 포인트 추가 가능.
- **권장 워크플로 2가지**: ① 시작과 끝을 먼저 맞춘 후 중간 보정 ② 첫 마디부터 순차적으로 좌→우 진행.

### 12.8 템포 자동화 상호 임포트 (양방향)
- 기본적으로 **스코어의 템포 자동화는 오디오 트랙 창에 자동 표시되지 않음**. 싱크 포인트가 추가/수정되는 순간부터 **오디오 트랙의 템포 표시가 스코어 템포 자동화를 대체**.
- **스코어 → 오디오 트랙**: Perform tasks 메뉴의 **"Create Sync Points from Score Tempo Automations"** — 기존 싱크 포인트를 덮어쓰거나 새로 생성.
- **오디오 트랙 → 스코어**: **"Create Score Tempo Automations from Sync Points"**(기어 메뉴 또는 웨이브폼 위 우클릭) — 스코어의 기존 템포 자동화를 대체.
- 오디오 트랙 표시 중에는 싱크 포인트 옆 템포 정보가 **스코어가 실제 재생하는 템포**를 반영.

### 12.9 오디오 트랙에서 재생 시작
- 오디오 트랙의 마디(또는 마디 일부)에 호버 → 커서가 **스피커 아이콘**으로 변경.
- **클릭 후 홀드** → 그 마디부터 재생 시작. **Shift를 누른 채**면 선택 구간 **루프 재생**.
- 재생 중 시야를 벗어나면 **Resume auto-scroll** 버튼으로 커서 위치 복귀.

### 12.10 오디오 트랙의 믹서/오디오 설정
- 오디오 트랙 표시 시 Global View에 **"Audio Track"** 이라는 트랙 행이 추가되며 일반 트랙처럼 믹싱 가능: **볼륨 / 팬 / EQ / Solo / Mute**.
- **필터**: **6종의 필터** 제공 — 특정 주파수 대역을 분리 또는 제거해 트랙 내 악기 분간을 도움.
- **피치 시프트**: **반음(semitones) + 센트(cents, 1/100톤)** 단위 독립 이조(스코어와 별개).
- **웹앱 구현 시사점**: AudioBufferSource → (pitch-shift 노드; 예: SoundTouch/phase-vocoder) → BiquadFilter(6프리셋) → gain/pan → master. 이조·필터는 실시간 파라미터.

### 12.11 내장(embed)과 추출
- **"Embedded Audio File"**(Perform tasks 메뉴): GP 파일에 오디오를 **내장한 채 저장** — **기본값: 자동 내장(ON)**; 내장을 원치 않으면 체크 해제. 내장 파일을 열면 오디오 트랙 창이 바로 나타남. 기본 내장 여부는 `Preferences > General > Embed audio files by default`로도 제어.
- **"Extract Audio File"**: 오디오 파일을 내보냄 — **MP3 형식 전용**.

## 13. 재생에 영향을 주는 악보 구조 (오디오 엔진 요구사항 요약)

> 반복/디렉션의 상세 편집은 타 도메인이나, **재생 엔진의 언롤링(unrolling) 규칙**은 본 도메인의 필수 구현 사항이므로 요약 기재.

- 재생 엔진은 다음을 해석해 마디 재생 순서를 전개해야 함:
  - **Repeat open/close**(pass 수 지정, 중첩 반복 지원, 하나의 open에 여러 close 허용)
  - **Alternate endings**(pass 번호 1~8, 후속 마디 암묵 연장 규칙)
  - **Directions**: Da Capo(약박 있으면 2번 마디로 점프, 약박은 1회만 재생), Segno/Segno Segno 타깃, Coda/Double Coda(Al Coda로 활성화, 없으면 마지막 출현에서 자동 활성), Fine(점프가 이미 발생했을 때만 정지), 반복 내 Segno형 점프=**마지막 pass**, Coda형 점프=**첫 pass**, "Al X"는 **다음 pass 강제** vs 미지정 시 **마지막 출현** 해석
- **Triplet feel**: 기보는 스트레이트, 재생만 스윙(모티프 선택형; 대표: triplet feel 8th) — 순수 재생 변환.
- **Fermata**: 멀티트랙 일시 정지; Fermata 창에서 **템포 조정 방식으로 길이** 지정.
- **Free time 마디**: 리듬/템포 자유 연주 구간.
- **Simple/Double repeat sign(%)**: 직전 1/2마디 반복 재생.
- **Grace note on the beat**: 정박에서 시작하며 **다음 박을 살짝 지연**시켜 재생.
- **크레셴도/데크레셴도**: **최종 다이내믹(target nuance)이 지정되어야만 재생에 반영**.
- **다이내믹 8단계**(ppp~fff) 및 대부분의 이펙트 기호가 재생에 반영됨(§5.11 참조).
- **Timer 기호**: 반복 구간에서는 **첫 pass 기준 시간**을 표시 — 재생 시간 계산기가 언롤링과 별개로 필요.
