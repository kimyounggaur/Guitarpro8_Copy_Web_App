# Guitar Pro 8 클론 웹앱 — 도메인 스펙: 악보 데이터 모델과 편집

> 근거: Guitar Pro 8 공식 사용자 매뉴얼(334p) 전체 추출본.
> 범위: 악보 구조(곡 정보/트랙/마디/섹션/보이스/음표), 트랙 생성·유형, 튜닝·카포, 마디 속성(박자표/조표/음자리표), 멀티보이스 편집, 음표 입력(마우스/키보드/프렛보드/MIDI), 복사·붙여넣기·잘라내기, 선택 모델, 실행취소, 자동화(템포/볼륨/팬), 어시스턴트/위저드, 드럼·퍼커션 입력.
> 제외: 이펙트 기호 상세, 스타일시트 렌더링 옵션, 오디오 엔진/사운드뱅크, 반복·디렉션 재생 언롤링, 임포트/익스포트 — 타 도메인 문서 참조.

---

## 1. 악보(Score) 데이터 모델 — 전체 계층

### 1.1 계층 구조
```
File(.gp) = Score (1 파일 = 1 악보)
 ├─ Song Information (메타데이터 10개 필드)
 ├─ Master Track (전 트랙 공유 속성: 박자표, 조표, 반복기호, 섹션, 템포 자동화, 마스터 볼륨/팬 자동화)
 ├─ Tracks [] (개수 사실상 무제한 — 파일이 비대하면 성능 저하 경고)
 │   ├─ 트랙 속성: name, shortName, color, icon
 │   ├─ 노테이션 타입: standard / tablature / slash / numbered (복수 조합 가능)
 │   ├─ 스태프: single / grand staff (분할 기준 C3)
 │   ├─ Tuning (+ capo, partial capo, display label)
 │   ├─ Transposition tonality (이조 악기 설정)
 │   ├─ 사운드 목록(트랙당 여러 사운드 → 사운드 체인지 자동화), RSE/MIDI 엔진 플래그
 │   ├─ 해석 옵션(playing style + 5개 토글)
 │   ├─ 볼륨/팬 자동화 레인
 │   └─ Bars [] (모든 트랙이 항상 동일한 마디 수 — 길이가 다른 트랙 불가)
 │        └─ Voices [4] (1 메인 + 3 추가)
 │             └─ Beats []
 │                  └─ Notes [] (음가, 점/겹점, 튜플렛, 타이, 임시표, 셈여림, 프렛/현, 오디오 노트 설정 등)
 └─ (선택) Audio Track 1개
```

### 1.2 핵심 불변 규칙 (매뉴얼 명시)
- **모든 트랙은 항상 같은 마디 수**를 가진다. 특정 마디 B의 조표·박자표는 모든 트랙(T1, T2, …)에서 동일 — 마스터 트랙 레벨 속성.
- 반복 기호(open/close), 섹션도 마스터 트랙 속성 → 단일 트랙에만 정의 불가.
- 1개 파일에 최소 1개 트랙, 트랙마다 최소 1개 마디.
- 한 음표는 **한 번에 하나의 보이스에만** 속함.
- 악보 오류(불완전/초과 마디 등)는 **빨간색**으로 표시(현재 편집 중인 마디는 예외). Preferences > Score errors에서 검사 항목별 on/off.
- 인그레이빙 규칙은 Elaine Gould의 *Behind Bars* 기반 자동 처리(수직 배치, 바라인 자동 추가, 트랙 간 동기 검증).

**웹앱 구현 시사점**: 마스터 트랙(공유 바 속성 배열)과 트랙별 바 콘텐츠를 분리한 정규화 모델이 필수. 바 삽입/삭제는 항상 전 트랙 트랜잭션으로 처리하고, 바 duration 검증기를 상태 변경마다 돌려 빨간색 렌더 플래그를 갱신.

## 2. 곡 정보 (Song Information)

| 항목 | 내용 |
|---|---|
| 동작 | 곡 메타데이터 편집. 값은 악보 헤더 렌더링과 헤더/푸터 토큰(%TITLE% 등)에 사용됨 |
| UI 위치 | 우측 **Song Inspector** (F5 토글). 악보 위 제목/아티스트 등을 **직접 클릭**해도 해당 인스펙터가 열림 |
| 필드(10) | Title, Artist, Subtitle, Album, Words(작사), Music(작곡), Copyright, Transcriber(Tabber), Notice, Instructions |
| 기본값 | 신규 파일은 전부 빈 값. Preferences > User information에 저장한 기본 작성자/저작권 정보로 자동 프리필 가능 |
| 관련 | Song Inspector에는 **Concert tone 토글 버튼**도 위치 |

**웹앱 시사점**: `score.meta` 객체 + 헤더 렌더러가 토큰 치환. 클릭 가능한 헤더 텍스트 → 인스펙터 포커스 이동 UX.

## 3. 새 악보 생성

| 항목 | 내용 |
|---|---|
| 진입점 | `File > New`, 홈페이지의 New 버튼, 탭바 우측 `+` 버튼(새로 만들기/열기 메뉴), 단축키 **Ctrl+N** (⌘N) |
| 동작 | 기본은 **빈 파일(트랙 0개)** 생성 → 즉시 **트랙 생성 위저드(Add Track 창)** 자동 오픈 |
| 템플릿 | `File > New From Template` = 사전 정의 템플릿에서 생성(홈페이지 Templates 버튼과 동일). 사용자 템플릿 저장 가능 |
| 기본 템플릿 설정 | Preferences > General의 **Default template**: 예) "Jazz Trio" 선택 시 새 파일에 재즈기타+업라이트베이스+드럼 3트랙이 미리 구성됨 |
| 멀티 문서 | 여러 파일 동시 오픈(탭바). 열린 파일 수 제한 없음. 수정된 파일은 탭에 ● 표시. "Open last session documents" 옵션으로 이전 세션 복원 |

**웹앱 시사점**: 빈 스코어 생성 → 트랙 위저드 모달 자동 표시 플로우. 템플릿은 직렬화된 트랙 프리셋 JSON.

## 4. 트랙 (Tracks)

### 4.1 트랙 추가 — 트랙 생성 위저드(Instrument 창)
| 항목 | 내용 |
|---|---|
| 진입점 | 메뉴 `Track > Add`, Global View의 Add track 버튼. 단축키 **Ctrl+Shift+Ins** (⌥⌘N) |
| 생성 플로우(창 내 순서) | ① 악기 타입 선택: **Stringed / Orchestra / Drums / MIDI** → ② 하위 목록에서 구체 악기 선택 → ③ 트랙 정보(name, color, icon) → ④ 노테이션 타입(standard/tablature/slash/numbered) + 스태프 타입(single/grand staff) → ⑤ 튜닝 선택·수정 → ⑥ 사운드 선택(**미리듣기 재생 버튼** 제공) → ⑦ **Create** 클릭 |
| 빠른 생성 | 악기 목록에서 **더블클릭** 시 즉시 트랙 생성 |
| 설정 저장 | 창 좌하단 **Save instrument settings** 버튼: 동일 악기 선택 시 현재 설정 재사용 |
| 지원 악기 | 3~10현의 모든 프렛 악기(기타/베이스/밴조/만돌린/우쿨렐레 등) + 피아노/스트링/드럼 등 약 100종 |

### 4.2 트랙 삭제 / 순서 변경
| 기능 | 동작 | UI/단축키 |
|---|---|---|
| 삭제 | 현재 트랙 제거 | `Track > Delete`; Global View 트랙 우클릭 → Delete. 단축키 **Del** (⌥⌘R) |
| 순서 변경 | 위/아래 이동 | `Track > Move Up / Move Down`; Global View 우클릭 메뉴; Global View 헤더의 ↑↓ 버튼 |
| 트랙 전환 | 다음/이전 트랙 | **Ctrl+Down / Ctrl+Up** (⌘↓/⌘↑); 툴바 LCD의 트랙명 클릭 → 트랙 목록 팝업 |

### 4.3 Track Inspector (우측 패널, F6 토글) — 4개 섹션
1. **Track properties**: name, short name, color, icon. 색상은 Global View와 툴바(현재 편집 트랙 표시)에 반영. short name은 Stylesheet에서 활성화 시 악보 시스템 앞에 표시.
2. **Musical notation**:
   - Notation: standard/tablature/slash/numbered **복수 조합** 가능.
   - Staves: single ↔ grand staff 전환. 기존 노트 존재 시 옵션 제공 — 분할: **Divide on C3** / 전부 1st staff 유지 / 전부 2nd staff 유지; 병합: 두 스태프 merge / 1st만 / 2nd만.
   - Tuning: 변경 + **미리듣기 버튼**. 내장 튜너도 이 튜닝에 맞춰짐.
   - Transposition tonality: 드롭다운으로 "기보된 C가 실제로 어떤 음으로 소리 나는가" 선택(트럼펫·색소폰 등 이조 악기 자동 표기). **Concert tone이 꺼져 있을 때만 유효**.
3. **Track sound**: 트랙당 **복수 사운드** 보유 가능(곡 중간 사운드 체인지용), RSE/MIDI 전환, 사운드 추가/복제, 자동화 창 진입("A" 버튼). *(상세는 사운드 도메인)*
4. **Interpretation options**:
   - **Playing style**(택1): `Pick`, `Finger`, `Picking`, `Bass Slap`
   - 토글 5종: **Palm mute**(특정 주법에서만 가능, 강도 슬라이더 별도), **Accentuation**(마디 내 특정 박 자동 강세), **Auto let ring**(음표 길이와 무관하게 현 공명 유지; 트랙 내 쉼표가 나오면 전 현의 let ring 정지), **Auto brush**(같은 비트의 코드 음들을 미세 시차로 재생 = 스트럼 시뮬레이션), **Stringed**(현악기 모드 — 같은 피치 다수 동시 공명 허용, off 시 피아노 모드. Stringed on: 노트를 현 단위로 정렬, 같은 현의 새 노트가 이전 노트 let-ring 차단; off: 피치 단위 구분)
   - 피아노 트랙에서 **Stringed off** 시 핑거링 창이 피아노식(1 2 3 4 5) 제안 (우측 패널 Track 탭 하단)

**웹앱 시사점**: `Track { props, notationTypes[], staffConfig, tuning, transposition, sounds[], engine, interpretation }`. 그랜드 스태프 분할/병합은 노트 재배치 마이그레이션 함수 필요.

## 5. 튜닝(Tuning) & 카포(Capo)

| 항목 | 내용 |
|---|---|
| 개념 | 피치가 있는 모든 트랙은 튜닝 보유(비현악기도 타브 표시를 위해 튜닝 존재). 비표준 튜닝(E A D G B E가 아닌 경우)은 기본적으로 악보 상단에 표시(Stylesheet로 제어) |
| 진입점 | ① 악보에 표시된 튜닝 텍스트 클릭 ② Track Inspector의 튜닝 클릭 → **Tuning 창** |
| 튜닝 라이브러리 | guitar, bass, banjo, shamisen, cello, contrabass, mandolin, ukulele, viola, violin. 현 수 **3~10** |
| 커스텀 튜닝 | 현별 개별 조정 + **임시표 표기 선택**(♯/♭ 스펠링). 수동 편집 결과가 라이브러리 튜닝과 일치하면 **자동 인식/하이라이트** |
| 미리듣기 | 재생 버튼으로 튜닝 청음 |
| 사용자 프리셋 | 옵션 버튼 → **"Save as Tuning Preset…"** (이름 지정 저장 창). 저장 후 같은 메뉴에서 편집/삭제. **capo, partial capo, display label은 프리셋에 저장되지 않음** |
| Capo | Tuning 창에서 **capo + partial capo 동시 사용 가능**. 프렛 번호는 **절대값**(partial capo가 capo보다 낮은 프렛이면 무효 — 경고 동작). 다이얼로그에 **프렛보드 그림**으로 시각화 |
| Display label | 악보 표시용 튜닝 이름 텍스트 필드(라이브러리 이름은 불변) |
| 커밋 모드(2택1) | **Keep the fingering**: 타브 프렛 번호 유지, 피치 변경 / **Adjust the fingering**: 새 튜닝·카포 하에서 동일 피치가 되도록 핑거링 재계산 |
| 표기 연동 | Stylesheet(F7) > System and Staves > **"Capo and shifted tunings impact standard notation"**: 체크 시 오선보가 실제 소리 피치 표기(전 트랙 조표 통일), 해제 시 튜닝/카포 보정 이조가 자동 적용되어 표준 튜닝처럼 읽기 쉬움(Track Inspector의 Transposition tonality에서 확인/수정) |

**웹앱 시사점**: `tuning = { strings: pitch[], capo, partialCapo, label, accidentalPreference }`. 튜닝 변경 시 keep/adjust 두 가지 변환 함수(프렛 보존 vs 피치 보존) 구현. 프렛→피치 계산은 `pitch = stringPitch + max(capo적용) + fret`.

## 6. 마디 (Bars)

### 6.1 마디 추가/삽입/삭제
| 기능 | 동작 | UI/단축키 |
|---|---|---|
| Insert Bar | 현재 마디 **앞에** 새 마디 삽입 — **전 트랙 동시** | `Bar > Insert Bar`; **Ins** (⌘+) |
| Delete Bar | 해당 마디를 **전 트랙에서** 삭제 | `Bar > Delete Bar`; **Ctrl+Del** (⌃-) |
| 마지막에 추가 | 악보 끝에서 **→(오른쪽 화살표)** 키를 누르면 새 마디 생성(마지막 마디 뒤에 추가하는 유일한 방법) | 커서 이동으로 자동 생성 |
| 대량 추가 | 커맨드 팔레트 `add-bar` + 개수(예: 20) → 한 번에 20마디 추가 | Ctrl+E 팔레트 |

### 6.2 마디 오류 관리 (자동 duration 검증)
- 커서를 오른쪽으로 이동 시: 현재 마디가 박자표 기준 **완전하면** 다음 마디로 이동, **불완전하면 직전 비트와 같은 음가의 새 비트를 자동 생성**.
- **불완전/초과 마디는 빨간색** 렌더(편집 중 마디는 제외).
- 마디가 완전해지면 음표 추가 시 자동으로 다음 마디로 이동.
- 툴바 **LCD**에 현재 마디의 실제 duration vs 이론 duration 표시. 해당 표시에 마우스 오버 시 **어느 보이스가 불완전한지 툴팁**으로 안내.
- **Anacrusis**(못갖춘마디): `Bar > Anacrusis` 체크 시 **첫 마디와 마지막 마디**는 불완전해도 빨간색 표시 제외.
- 재생 동작: 불완전 마디는 리듬이 틀어지고, 마디 duration을 초과한 노트는 재생되지 않음.

### 6.3 박자표 (Time Signature)
| 항목 | 내용 |
|---|---|
| 진입점 | 편집 팔레트의 박자표 버튼, `Bar > Time signature` 메뉴, 악보의 박자표 직접 클릭 → **Time Signature 창**. 단축키 **Ctrl+T** (⌘T) |
| 기본값 | **4/4** |
| 의미 | 아래 숫자 = 온음표 분할 단위(기준 음가), 위 숫자 = 마디당 단위 수 |
| 부가 옵션 | 같은 창에서 해당 박자표의 **기본 빔 연결(beaming) 방식** 선택 |
| 자동 동작 | 박자표 변경 지점마다 **더블 바라인 자동 삽입** |
| 표기 옵션 | Stylesheet: 4/4·2/2를 C(커먼타임) 기호로 대체 표기 가능 |

### 6.4 조표 (Key Signature)
| 항목 | 내용 |
|---|---|
| 진입점 | 편집 팔레트 조표 버튼, 악보의 조표 직접 클릭 → **Key Signature 창**. 단축키 **Ctrl+K** (⌘K) |
| 옵션 | **키 + 모드(major/minor)** 선택. 기본값 **C major** |
| Concert tone 옵션 | 창 내 체크박스: 선택한 조표를 콘서트 톤에 적용할지 이조 톤에 적용할지 — 나머지 한쪽은 자동 계산 |
| 규칙 | 조표는 **전 트랙 공통**(이조 악기가 transposing tone 모드일 때만 자동으로 달라짐) |

### 6.5 음자리표 (Clef)
| 항목 | 내용 |
|---|---|
| 자동 설정 | 트랙 악기에 따라 G 또는 F 클레프 자동 지정 |
| 진입점 | 편집 팔레트 클레프 버튼, 악보의 클레프 직접 클릭 → **Clef 창** |
| 종류 | 피치 클레프 4종: **G, F, C3, C4** + 옥타브 클레프 4종: **8va, 8vb, 15ma, 15mb** |
| Transpose 토글 | 활성: 클레프 변경 후에도 **같은 소리가 나도록 노트 재계산** / 비활성: 기보 위치 유지, 소리 피치 변경 |

### 6.6 마디 레벨 기타 속성 (마스터 트랙 데이터 모델에 포함)
- Triplet feel(스윙 모티프 — 대표: triplet feel 8th, 전용 에디터), Free time bar(점선 바라인 + 괄호 박자표), 더블 바라인(수동 추가 가능), simile 반복 기호(1마디 %: 첫 마디 불가 / 2마디: 첫 두 마디 불가), Repeat open/close(+반복 횟수 대화상자), Alternate endings(패스 1~8), Directions(Coda/Segno 등), 섹션 마커. *(재생 언롤링·상세 기호는 타 도메인)*

### 6.7 시스템 레이아웃 / 디자인 모드 (마디 배치)
| 기능 | 동작 | UI/단축키 |
|---|---|---|
| Force break line | 해당 마디에서 강제 줄바꿈(마지막 마디 우측 정렬에도 사용) | `Bar > Force break line`; **Ctrl+Return** (⌘↵); 팔레트 아이콘 |
| Prevent line break | 줄바꿈 방지 | `Bar > Prevent line break`; **Shift+&**; 팔레트 아이콘 |
| System layout | 시스템(줄)당 기본 마디 수 지정 대화상자. 적용 시 기존 줄바꿈 리셋. **"Start at current system"** 체크: 현재 시스템 이후에만 적용 / 해제: 문서 전체 | `Bar > System layout` (Page/Parchment 뷰 모드에서 사용) |
| Design mode | 시스템 내 마디 폭, 시스템당 마디 수 수동 조정. 룰러 위 **삼각형 드래그**로 마디 폭 조절(수정된 삼각형은 빨간색), **비트 단위 좌우 이동**, 이동 중 **자동 정렬 + 노란 세로 기준선**, `+`/`-` 아이콘으로 시스템 내 마디 추가/제거, 시스템 우측 **X(cross) 아이콘**으로 해당 시스템 간격 리셋(앵커 라인 우측 `x`는 전체 앵커 리셋), View 메뉴에서도 간격 리셋 | 팔레트의 Design mode 아이콘(보이스 셀렉터 우측) 또는 View 메뉴; **Ctrl+Alt+D** (⌃⌥D) |
| 규칙 | 줄바꿈은 **활성 트랙 또는 멀티트랙 뷰에만** 적용 — 트랙마다 다른 마디 배치 가능, 멀티트랙 뷰는 자체 배치 보유 |

**웹앱 시사점**: `masterBar { timeSig, keySig(mode), tripletFeel, freeTime, repeats, section, doubleBar }` + 트랙별 `barLayout { forcedBreak, preventBreak, widthOverrides, beatOffsets }`. 렌더러는 뷰 모드별(페이지/스크린) reflow와 수동 오버라이드 병행.

**웹앱 시사점**: `section { barIndex, letter, name, boxed }` 마스터 트랙 배열. Global View 미니맵과 점프 내비게이션에 공용 사용.

## 8. 보이스 (Voices) / 멀티보이스 편집

| 항목 | 내용 |
|---|---|
| 구조 | 트랙당 **최대 4보이스**(메인 1 + 추가 3). 같은 비트에 서로 다른 음가 배치 가능(베이스라인/멜로디/화음 분리 등). 한 음표는 하나의 보이스에만 소속 |
| 보이스 색상 | 보이스별 고유색 — **yellow, green, pink, purple**(커서 사각형은 yellow/green/blue/red로도 기재 — 커서는 활성 보이스 색을 따름) |
| 편집 규칙 | 비활성 보이스의 노트는 **회색 처리 + 편집 불가**. **멀티보이스 모드** 활성화 시 전 보이스 동시 편집 가능 |
| 진입점 | `Edition > Voice` 메뉴(활성 보이스 선택 + 멀티보이스 토글), 편집 팔레트 상단의 보이스 셀렉터/멀티보이스 버튼 |
| 표기 옵션 | Stylesheet > Notation: "Colorize fret numbers in tablature for voices 2, 3 and 4"; 타브 내 리듬 표기 위치를 **보이스별로 above/below/hidden** 지정 가능 |
| 위저드 | **Move/Copy/Swap Voices**(Tools 메뉴): 보이스 간 이동/복사/교환 대화상자. 커맨드 팔레트: `copy voice`, `move-voice`, `swap-voices`, `voice` |

### 보이스 단축키
| 동작 | Windows | macOS |
|---|---|---|
| 보이스 1~4 편집 전환 | Ctrl+1 ~ Ctrl+4 | ⌘1 ~ ⌘4 |
| 노트를 보이스 n으로 이동 | Alt+1 ~ Alt+4 | ⌥1 ~ ⌥4 |
| 멀티보이스 편집 토글 | Ctrl+M | ⌘M |

**웹앱 시사점**: `bar.voices[4].beats[]` 고정 4슬롯. 렌더러는 활성 보이스 강조 + 비활성 회색, 히트테스트는 멀티보이스 모드에 따라 필터링.

## 9. 음표(Notes) 입력과 편집

### 9.1 일반 원칙
- 노트는 **타브 또는 오선보** 어느 쪽에서 입력해도 **즉시 상대 노테이션에 복제**.
- **Slash 노테이션은 리듬 전용**(입력용 아님) — 코드를 스태프에 넣으면 슬래시가 생기고 그 리듬만 수정 가능.
- **악보 클릭은 노트를 추가하지 않음** — 커서 이동만(탐색 중 오편집 방지).
- 편집 커서 = 활성 보이스 색의 사각형. 커서가 노트 위에 있으면 상대 노테이션의 대응 노트가 **회색 사각형**으로 표시.
- **Tab 키**: 타브 ↔ 오선보 전환. 피아노 그랜드 스태프에서는 좌수/우수 스태프 전환.

### 9.2 키보드 입력 (최속 방식, 주로 숫자패드)
| 동작 | Windows | macOS | 비고 |
|---|---|---|---|
| 스태프 위 이동 | ↑ → ↓ ← | 동일 | ← 이전 비트 / → 다음 비트(끝에서 새 마디 생성) / ↑↓ 라인 이동 |
| 노트 입력 | 0–9 | 동일 | **오선보**: 1–9 = 배치할 현 선택, **0 = 최적 현 자동 배치**. **타브**: 숫자 = 프렛 번호, 짧은 **입력 지연으로 2자리 프렛** 지원 |
| 노트 확정 | Enter/Return | ⌤/↵ | Type Note |
| 음가 변경 | +/- | 동일 | **Preferences > Interface에서 +/- 동작 반전 가능**. 기본: `+` = 음가 길게(4분→2분), `-` = 짧게. 부록 표기: Increase=`-/_`, Decrease=`+/=` |
| 비트 삽입 | Ins (본문) / Ctrl+Ins (부록) | ⌘+ / ⌃+ | `Note > Insert a beat` 메뉴 동일 |
| 노트 삭제 | Backspace | ⌫ | Delete Note. Delete(선택 삭제) = Ctrl+Shift+Del (⌦) |
| 비트 삭제 | Shift+Del | ⌘- | Delete the Beats; `Note > Delete a beat` |
| 쉼표 입력 | R | R | |
| 마지막 비트 복사 | C | C | Copy Last Beat |
| 노트를 이웃 현으로 이동 | Alt+Up / Alt+Down | ⌥↑/⌥↓ | 피치 유지, 현 변경 |
| 스태프에서 노트 상하 이동 | Ctrl+Alt+Up / Ctrl+Alt+Down | ⌥⌘↑/↓ | 피치 변경 |
| 반음 올림/내림 | Alt+Shift+Up / Alt+Shift+Down | ⌥⇧↑/↓ | 선택 영역 전체에도 적용 — Select All과 결합해 트랙 전체 이조 |

### 9.3 마우스 입력
- **Instrument Views**(가상 프렛보드/키보드/드럼킷)를 클릭해 입력. 팔레트의 음가 버튼으로 길이 변경. `Note > Insert a beat / Delete a beat` 메뉴.
- Instrument View 공통: 뷰 내 **이전/다음 커서 버튼** 내장(키보드 불필요), **기존 노트 클릭 = 삭제(토글)**, 좌측 **기어 버튼** = 옵션 팝업. 표시 범위 3모드: **Show Beat / Show Beat + Bar / Show Beat + Next Beat**.
- **프렛보드 뷰**: 활성 트랙의 **현 수·카포 위치 자동 반영**, 트랙 전환 시 자동 갱신. 호버 시 클릭 없이 노트 미리 표시(마우스 아래 + 좌측에 음이름·옥타브, 예: E2). 좌손잡이 모드 설정 가능. 프렛+현 클릭 = 추가, 재클릭 = 삭제. 스케일 오버레이 표시 시: 클릭 = 타브에 노트 배치, **우클릭 = 노트 배치 + 커서 우측 이동**(고속 입력).
- **키보드(피아노) 뷰**: 현재 트랙의 **연주 가능 음역만 활성**(범위 밖 건반은 회색·클릭 불가). 건반 클릭 = 추가/삭제.
- 토글 단축키: **Ctrl+F6** (⌘F6) — 트랙 타입에 따라 프렛보드(현악)/키보드(피치악기)/드럼킷(퍼커션) 컨텍스트 전환. 프렛보드+키보드 동시 표시 가능.

### 9.4 MIDI 캡처 (스텝 입력)
- `Sound > MIDI Capture`: MIDI 키보드/기타/컨트롤러로 **한 비트씩** 노트 입력(실시간 녹음 아님).
- 설정: `Sound > Audio/MIDI settings` — 장치 선택, **캡처 감도**(같은 코드로 묶을 두 노트 간 최대 지연 ms — 초과 시 새 비트로 배치), 채널 감지·현 할당 설정(MIDI 기타의 경우 "First channel for the highest/lowest string" 2모드; 키보드는 프렛보드 상 핑거링 최적화 자동 배치).

### 9.5 음가·리듬 데이터 모델 (노트/비트 속성)
| 속성 | 값/규칙 | 입력 UI |
|---|---|---|
| 음가(duration) | 7단계: whole ~ 64th | 팔레트 버튼 7개, +/- 키 |
| 점/겹점 | dot = +1/2 길이, double-dot = +3/4 길이 | 팔레트; **Shift+.** (./⇧.), 겹점 **Ctrl+.** (⌘.) |
| 쉼표 | 음가별 7종(whole rest ~ 64th rest) | R 키, 팔레트 |
| 튜플렛 | 트리플렛 버튼 + Custom n-tuplet(우클릭으로 **2~13 분할** 선택: Duplet~Tredecuplet, **n:m 커스텀 비율** 대화상자) + **Nested tuplet**(내측 튜플렛 전용 버튼, 우클릭 메뉴 동일. 외측은 Custom n-tuplet으로 — 생성 순서 무관). 두 워크플로: ①노트 입력 후 선택→버튼 적용 ②버튼 선활성(파란색)→이후 입력이 튜플렛 | 팔레트(우클릭/롱프레스로 비율 선택); **Shift+/** (트리플렛) |
| 타이 | Tie(직전 노트에 연결, 길이 연장), Multiple tie(비트 전체 타이) | **L** / **Shift+L**(Tie Beat) |
| 임시표 | 5종: sharp/double sharp/flat/double flat/natural. **효력은 해당 마디 끝까지**. Cautionary accidental(주의 임시표): `Note > Accidentals > Cautionary accidentals`. Change accidental = 피치 유지 이명동음 전환 | 팔레트; Sharp **Ctrl+9**, Flat **Ctrl+7**, Natural **Ctrl+8**, Double Sharp **Ctrl+Alt+9**, Double Flat **Ctrl+Alt+7**, 이명동음 **Ctrl+Alt+8** |
| 셈여림(nuance) | 노트별 8단계: ppp pp p mp mf f ff fff. 같은 비트에 다른 셈여림 = 보이스 분리 필요. Crescendo/Decrescendo는 비트(들)에 적용 + **최종 셈여림 지정 필수**(재생 반영 조건) | 팔레트 |
| 그레이스 노트 | 현당 최대 1개, **마디 duration 계산에서 제외** | 팔레트/G 키 |
| 오디오 노트 설정 | `Note > Audio Note Settings` (**Shift+F**): Duration **25~1200%**(기본 100%), Offset(비트 전/후 ±), Relative velocity(기본 0dB, ±1~15dB). 이론값 기준 적용, **4분음표 = 480 ticks**. 그레이스 노트·아르페지오는 제외. Reset 버튼 | 대화상자 |

### 9.6 슬래시(Slash) 표시
- `Note > Slash` 또는 팔레트: 선택 노트를 슬래시 표기로 전환 — **노트 정보는 보존**(비파괴). standard·drums 노테이션에서도 사용 가능(GP8 신규). 트랙 전체 슬래시 전용 표시는 Track Properties에서.

**웹앱 시사점**: `Note { string, fret, pitch, tie, accidental, dotted, doubleDotted }` + `Beat { duration, tupletRatio(n:m, nested), dynamics, slash, graceNotes[], audioSettings }`. 튜플렛 중첩·보이스 4개·타브/오선 동기화를 지원하는 커스텀 렌더 엔진 필요. tick 해상도 480/quarter로 내부 시간 표현 통일.

## 10. Numbered Notation (Jianpu, 숫자보) — GP 8.1+

| 항목 | 내용 |
|---|---|
| 선택 | Track Inspector에서 4번째 노테이션 타입으로 선택. **퍼커션 제외 전 악기** 사용 가능. 숫자보만 단독 표시하는 렌더 모드 지원 |
| 표기 | 숫자 **1~7 = 이동도(movable-do) 음계 도수**, **0 = 쉼표**. 단선율 표시(슬래시처럼 1노트만). 조표는 `1=C`(장조) / `1=a`(단조) 형식 |
| 음가 | 4분음표보다 길면 우측 대시: `1`=4분, `1 -`=2분, `1 - -`=점2분, `1 - - -`=온, `1 - - - -`=점온. 4분보다 짧으면 **밑줄**(1개=8분, 2=16분, 3=32분, 4=64분) — 밑줄은 빔과 동일 규칙으로 그룹화. 쉼표(0)도 동일 |
| 옥타브 | 숫자 위/아래 **점**(옥타브당 1개) — 건반 7옥타브 표현 |
| 임시표 | 조표 기준 상대 표기: C장조에선 숫자 앞에 임시표; Eb장조에선 Eb=`1`(무기호), E내추럴은 E# 방식으로 표기 |
| 주변 요소 | 현재 지원: 튜플렛, 슬러(HoPo/legato 포함), 액센트, 오너먼트, 가사, 핑거링, 페르마타 |

**웹앱 시사점**: 노트 모델은 공유하고 렌더러 레이어만 분리(피치→도수 변환은 조표 기준).

## 12. 복사 / 붙여넣기 / 잘라내기

### 12.1 단일 트랙 Cut/Copy
- Edit 메뉴 또는 악보 우클릭(macOS Ctrl+Click) 컨텍스트 메뉴.
- 선택 콘텐츠 전체 복사(기본 모드). **바 구조(박자표·조표 등)는 미포함**. **멀티 보이스 포함**.
- 이종 악기 간 붙여넣기 허용(기타→피아노). **퍼커션↔선율 트랙 간에는 리듬만 보존(쉼표로 표현)**.

### 12.2 멀티트랙(All Track) Cut/Copy
- **바 구조 포함**(박자표/조표 등). 섹션 포함 통짜 복사 가능. 멀티트랙 Cut은 선택 마디를 **전 트랙에서 제거**.

### 12.3 Simple Paste
- 클립보드를 현재 선택 위에, 기본은 선택 **앞에** 붙여넣기.
- 선택이 1마디뿐이면: 붙여넣은 마디들이 그 앞에 추가되거나 현재 마디를 클립보드 내용으로 채움.
- 커서가 빈 마디면: 빈 마디부터 채우고 부족하면 마디 자동 생성.
- **Preferences > Interface** 옵션: simple paste에 **코드/스케일 다이어그램 포함** 여부 (GP8 신규).

### 12.4 Special Paste (대화상자)
- 옵션: **Number of pastes**(N회 반복 붙여넣기), **Insert**(현재 마디에 삽입 vs 덮어쓰기).
- 붙여넣을 요소 선택(6종): Chords and scales diagrams / Texts / Timers / **Track automations** / **Master track automations** / **Sections** (GP8에서 track·master 자동화 분리됨).
- 제약: master track automations·sections 붙여넣기는 클립보드가 **All Track Copy**여야 함.

### 12.5 단축키
| 동작 | Windows | macOS |
|---|---|---|
| Copy / Cut / Paste | Ctrl+C / Ctrl+X / Ctrl+V | ⌘C / ⌘X / ⌘V |
| 멀티트랙 Copy / Cut | Ctrl+Shift+C / Ctrl+Shift+X | ⇧⌘C / ⇧⌘X |
| Special Paste | Ctrl+Shift+V | ⇧⌘V |
| 마지막 비트 복사 | C | C |

**웹앱 시사점**: 클립보드 페이로드에 `mode: single|multitrack` 태그와 바 구조 스냅샷을 포함. Special paste는 요소 타입별 필터 적용 후 merge.

## 15. 자동화 (Automations) — 템포/볼륨/팬

### 15.1 개념과 종류
| 타입 | 범위 | 비고 |
|---|---|---|
| **Tempo** | 항상 **전 트랙(마스터) 전용** | 재생 중 자동 가감속. 각 포인트에 악보 표시용 **라벨** 지정 가능. **소수점 템포 허용**(예: 100.5bpm), **30bpm 미만 허용**(GP8 신규) |
| **Volume** | 트랙별 또는 마스터 | 활성 시 Global View 볼륨 슬라이더 옆 **"A" 배지** 표시 — 클릭으로 자동화 enable/disable 토글 |
| **Pan** | 트랙별 또는 마스터 | 활성 시 팬 노브 옆 인디케이터 표시(토글) |

팔레트에는 자동화 삽입 아이콘 5종: Tempo / Master volume / Master pan / Track volume / Track pan — 각각 커서 위치에 자동화 포인트 삽입 대화상자를 오픈. (커맨드 팔레트: `tempo`, `volume`, `pan`, `master-volume`, `master-pan`, `hide-tempo-automation`, `show-tempo automation`)

### 15.2 자동화 에디터
| 항목 | 내용 |
|---|---|
| 진입점 | `Edit > Automations > Edit Automations`, 단축키 **F10**, 팔레트 자동화 아이콘, 악보의 템포 마커 직접 클릭, 툴바 LCD의 템포 값 클릭 |
| 동작 | 창 오픈 시 현재 위치에 포인트가 없으면 **자동 추가**. 그래프 클릭으로 키 포인트 배치. 포인트는 **비트/마디에 자동 스냅**. 마우스 또는 화살표 키로 좌우(시간)/상하(값) 이동. 선택 포인트는 **Delete** 키로 삭제 |
| 전이(transition) 타입 | 좌측 패널에서 포인트별 2택1: **Constant until next point**(다음 포인트까지 유지 후 급변) / **Progressive to the next point**(다음 값까지 램프) |
| 초기 템포 | 같은 창에서 곡의 initial tempo 정의 |
| 일괄 삭제 | **Remove Automations** 버튼 = 전체 자동화 제거 |
| 표시 | 트랙 위 자동화 변경 표시를 숨기기/표시 가능 |
| 오디오 트랙 연동 | 오디오 트랙 활성 시 싱크 포인트가 템포를 대체(악보 템포 자동화 무시). 양방향 변환 메뉴 존재 *(오디오 도메인 참조)* |

**웹앱 시사점**: `automation { type, scope(track|master), points: [{tick, value, transition, label?}] }`. 재생 스케줄러(Web Audio)에서 constant는 setValueAtTime, progressive는 linearRamp로 매핑. 템포 자동화는 tick→초 변환 테이블 재생성 필요.

## 19. 커서 내비게이션 (편집 커서)

- 커서 위치는 악보와 Global View 간 **항상 동기화**. 악보 클릭 = 커서 이동(재생 중에는 재생을 멈추지 않고 그 지점부터 재생 재개).
- Global View: 좌측 트랙명 클릭 = 그 트랙 현재 마디 첫 비트로(재생 중이면 재생 중인 비트로); 우측 바 그리드 클릭 = 트랙+마디 동시 지정; 섹션 첫 마디 클릭 = 섹션 시작으로. Alt+휠 = 수평 스크롤.
- 편집 중 현재 트랙이 Global View에 안 보이면 편집 시점에 **자동 리포지션**.

## 22. 웹앱 아키텍처 종합 시사점

1. **문서 모델**: `Score → MasterBars[] + Tracks[] → Bars[] → Voices[4] → Beats[] → Notes[]` 정규화. 마스터 바 속성(박자/조표/섹션/템포자동화)과 트랙 바 콘텐츠 분리. tick(480/quarter) 기반 내부 시간축.
2. **불변식 엔진**: 바 duration 검증(빨간색 플래그 + LCD의 실제/이론 duration + 불완전 보이스 툴팁), 전 트랙 동일 마디 수, 보이스 단일 소속 — 모든 뮤테이션 후 검증 훅.
3. **편집 커널**: 커서(트랙/마디/보이스/비트/현) + 선택 범위를 단일 상태로. 키보드 우선 입력(숫자=프렛, 화살표=이동, 끝에서 → = 마디 생성)이 UX의 중심.
4. **양방향 렌더 동기화**: 타브·오선·슬래시·숫자보 4개 뷰가 같은 노트 모델을 투영 — 입력은 어느 뷰에서든 즉시 전 뷰 반영. 상대 노테이션의 대응 노트는 회색 커서로 표시.
5. **Undo**: 위저드·바 삽입 등 다중 객체 변경을 원자 트랜잭션으로 묶는 커맨드 스택 (Ctrl+Z / Ctrl+Y).
6. **자동화**: 포인트 배열 + 전이 타입(constant/progressive) → Web Audio 파라미터 램프 및 템포 맵(tick→sec) 재계산.
7. **입력 다양화**: 가상 프렛보드/키보드/드럼킷(Canvas/SVG) 클릭 입력 + Web MIDI 스텝 캡처(감도 ms로 코드 묶음 판정).
8. **클릭-편집 라우팅**: 렌더된 모든 편집 가능 요소(15종)에 히트 메타데이터를 달아 공용 다이얼로그로 연결, 호버 시 파란 하이라이트.
9. **튜닝/이조 파이프라인**: 실음 ↔ 표시 피치 변환을 (tuning, capo, transposition tonality, concert tone, stylesheet 옵션)의 순수 함수로 일원화.
