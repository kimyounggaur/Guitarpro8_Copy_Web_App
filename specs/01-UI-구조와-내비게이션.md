# Guitar Pro 8 클론 웹앱 — 도메인 사양서: UI 구조와 내비게이션

> 출처: Guitar Pro 8 공식 사용자 매뉴얼(334p) 전체 추출본 통합.
> 범위: 홈페이지(런처), 메인 스크린 레이아웃(툴바/탭바/에디션 팔레트/워크스페이스/인스펙터/글로벌 뷰), 악보 내비게이션, 악보 표시 모드, 줌, 멀티트랙 뷰, 전체화면, 환경설정(General/Interface/Score errors/User information 탭), 명령 팔레트(Command Palette), 다크 모드, 창 관리.
> 다른 도메인(악보 편집, 오디오 엔진, 임포트/익스포트 등)의 내용은 인터페이스 접점만 언급한다.

---

## 1. 홈페이지 (Homepage / 런처)

### 1.1 홈페이지 기본 동작
- **동작**: 앱 실행 시 홈페이지가 표시된다. 파일이 열려 있는 상태에서도 **툴바의 Homepage 아이콘**으로 언제든 다시 열 수 있다. 목적은 사용자 파일과 mySongBook 카탈로그에 대한 빠른 접근.
- **UI 위치/인터랙션**: 앱 시작 화면. 상단에 **New File**(새 파일 생성 후 즉시 편집 시작), **Open File**(파일 탐색기 열기) 버튼.
- **단축키**: 새 파일 `Ctrl+N`(Win) / `⌘N`(macOS) — 홈페이지의 New 버튼과 동일 동작.
- **웹앱 시사점**: 라우트 분리(`/home` vs `/editor`)보다는 에디터 위에 오버레이/탭으로 구현해 열린 문서를 유지한 채 홈 복귀 가능해야 함.

### 1.2 검색바 (홈페이지 통합 검색)
- **동작**: Open File 옵션 **아래**에 위치. 파일명의 전체/일부를 입력하면 **타이핑할 때마다 실시간으로 결과가 좁혀지는(incremental) 검색**. **각 카테고리 옆에 검색 결과 개수가 표시**된다.
- **수치/옵션**: 검색 대상 카테고리 5개 — ① Recent files ② Local files ③ mySongBook ④ Templates ⑤ Examples.
- **웹앱 시사점**: 카테고리별 인덱스(IndexedDB + 서버 카탈로그)를 동시 질의하고 카테고리 헤더에 카운트 배지를 렌더링.

### 1.3 Recent files (최근 파일 + 핀 고정)
- **동작**: 최근에 연 파일 목록. **Delete recent files** 옵션은 목록 전체를 비운다(**개별 항목 삭제는 불가** — 매뉴얼 명시 제한).
- **핀(Pin) 고정**: 파일명에 **마우스오버하면 행 왼쪽에 핀 아이콘**이 나타난다. 클릭 시 해당 파일이 목록 **최상단에 고정**되고, 다시 클릭하면 고정 해제.
- **웹앱 시사점**: `recentFiles: {path, pinned, lastOpened}[]`를 localStorage/IndexedDB에 영속화. pinned 항목을 상단 정렬.

### 1.4 Local files (로컬 파일 라이브러리)
- **동작**: **Choose a folder** 옵션으로 컴퓨터의 폴더 하나를 선택해 **자신의 파일 라이브러리로 인덱싱**. 해당 폴더의 Guitar Pro 파일들이 Local files 카테고리에 나타난다.
- **웹앱 시사점**: File System Access API의 `showDirectoryPicker()` + 핸들 영속화(IndexedDB)로 동일 UX 구현 가능.

### 1.5 mySongBook 섹션 (홈페이지 내)
- **동작**: mySongBook 카탈로그 전체 타이틀 목록에 직접 접근. **3개 하위 섹션**: ① Free Tab of The Day(매일 교체되는 무료 탭, 섹션 최상단) ② Last Added ③ All Tabs(구매 항목 포함). 구매/구독 이용은 **"Sign in to mySongBook"**(이메일+비밀번호) 선행 필요. GP 8.1부터 일부 타이틀 무료.
- **웹앱 시사점**: 외부 카탈로그 API 연동 지점. 홈페이지 UI 관점에서는 로그인 상태 배지와 3개 서브리스트만 구현하면 됨(상세는 mySongBook 도메인).

### 1.6 Templates
- **동작**: 기본 제공 템플릿 여러 개 + **사용자가 직접 만들어 저장한 템플릿**에 빠르게 접근. `File > New From Template` 메뉴와 동일 결과.
- **웹앱 시사점**: 템플릿 = 트랙 구성이 미리 설정된 문서 스냅숏(JSON). 사용자 템플릿 저장 스토어 필요.

### 1.7 Examples
- **동작**: 다양한 음악 스타일의 **예제 파일 23개** 제공(편집·렌더링 능력 쇼케이스).
- **웹앱 시사점**: 정적 에셋으로 번들.

### 1.8 파일 미리보기 / 프리리스닝
- **동작**: 홈페이지의 **각 파일**에 대해 **시각적 미리보기(preview)** 와 **트랙별 오디오 미리듣기(pre-listening)** 제공 — 파일을 열기 전에 각 트랙을 들어볼 수 있다.
- **웹앱 시사점**: 파일 카드에 축소 악보 렌더 + 트랙 리스트별 재생 버튼. 서버 사전 렌더 썸네일/오디오 스트림 또는 클라이언트 경량 파서.

### 1.9 직접 접근 링크
- **동작**: 홈페이지에서 웹사이트 **Support** 섹션, **사용자 가이드**, 제품 **소셜 네트워크**로 바로가는 링크 제공.

### 1.10 홈페이지 섹션 숨김 (환경설정 연동)
- **동작**: Preferences > Interface 탭에서 홈페이지의 **mySongBook**과 **Examples** 항목을 **제거(숨김)** 할 수 있다.

## 2. 메인 스크린 레이아웃 — 6개 패널

메인 UI는 다음 **6개 패널**로 구성된다.

| # | 패널 | 위치 | 숨김 가능 | 토글 단축키 |
|---|---|---|---|---|
| 1 | Toolbar(툴바) | 상단 | 불가(항상 표시) | — |
| 2 | Tab bar(탭바) | 툴바 바로 아래 | 불가 | — |
| 3 | Edition palette(에디션 팔레트) | 좌측 | 가능 | `F2` |
| 4 | Workspace(워크스페이스) | 중앙 | 불가(악보 영역) | — |
| 5 | Inspector(인스펙터) | 우측 | 가능 | Song `F5` / Track `F6` |
| 6 | Global view(글로벌 뷰) | 하단 | 가능(수직 리사이즈 가능) | `F8` |

- 패널 표시/숨김 컨트롤: **메인 창 좌측 상단의 표시 옵션 버튼들** + **View 메뉴**의 동일 커맨드.
- 워크스페이스와 툴바를 제외한 **모든 패널은 숨길 수 있다**(특히 재생 시 가시성 향상 목적).
- **웹앱 시사점**: CSS Grid 기반 레이아웃 + 패널별 `visible` 상태를 전역 스토어에 저장(세션 간 영속). 글로벌 뷰는 드래그로 수직 리사이즈.

### 2.1 툴바 (Toolbar)
- **동작**: 창 **최상단**. 재생·뷰 옵션과 핵심 기능에 빠르게 접근. **사이드 패널·글로벌 뷰 없이도 툴바만으로 앱의 필수 기능 사용이 가능**하도록 설계됨.
- **반응형 동작**: 툴바는 **창 너비에 따라 내용이 적응** — 공간이 부족하면 일부 버튼이 숨겨지고, 숨겨진 모든 동작은 애플리케이션 메뉴에서 계속 접근 가능(예: Print 버튼은 창이 충분히 넓을 때만 표시).
- **툴바 구성 15개 그룹(왼쪽→오른쪽 순서)**:
  1. Homepage 접근 아이콘
  2. 인터페이스 패널 표시/숨김 토글(좌측 상단)
  3. Zoom(줌 컨트롤)
  4. Display modes(표시 모드 아이콘)
  5. Undo/Redo
  6. Print
  7. Navigation & playback(트랜스포트 버튼)
  8. LCD(곡 위치/템포 표시부)
  9. Loop & speed settings(루프 + 재생 속도/스피드 트레이너)
  10. Global tonality(전역 조성)
  11. Audio track(오디오 트랙 접근 — 파일에 오디오 트랙이 있으면 버튼에 **파란 외곽선** 표시)
  12. Instrument views(가상 프렛보드/키보드/드럼킷 토글, 우측)
  13. Guitar tuner(튜너)
  14. Line-in(우측)
  15. Fretlight(기본 숨김, Preferences > Interface에서 표시 — 최우측)
- **기타**: 툴바에는 **현재 편집 중인 트랙의 색상**이 표시된다. 단축키가 있는 버튼에 호버하면 **툴팁에 단축키** 표시.
- **웹앱 시사점**: ResizeObserver로 오버플로 우선순위 기반 버튼 축소(우선순위 낮은 그룹부터 숨김). 모든 툴바 액션은 커맨드 레지스트리를 공유해 메뉴/명령 팔레트와 단일 소스로 연결.

### 2.2 LCD 디스플레이 (툴바 중앙 위젯)
- **동작**:
  - **현재 트랙 이름** 표시. 클릭하면 **전체 트랙 목록 팝업 메뉴**가 열려 현재 트랙 전환 가능(글로벌 뷰가 숨겨져 있어도 트랙 전환 수단이 됨).
  - **템포 값** 표시. 클릭하면 **템포 자동화 창**(Edit > Automations > Edit Automations와 동일)이 열린다. 재생 중에는 **실시간 현재 템포**를 표시(소수점 템포 지원, 예: 100.5bpm).
  - **현재 마디의 실제 길이 vs 이론 길이** 표시(마디 불완전 검증). 호버 시 **어느 보이스가 불완전한지 툴팁**으로 안내.
  - **카운트인 on/off**, **메트로놈 on/off**, **설정(기어)** 3개 컨트롤 노출. 설정에서 메트로놈/카운트인 볼륨, 카운트다운 마디 수, 루프 간 카운트다운, **비주얼 메트로놈**(상단 패널에 박자마다 플래시; 볼륨 0이면 시각 전용) 구성.
- **웹앱 시사점**: LCD는 다수 도메인(재생/편집 검증/트랙)의 상태를 구독하는 복합 위젯. 상태 스토어에서 currentTrack, tempo(live), barDurationStatus, metronome 설정을 구독하는 단일 컴포넌트로 설계.

### 2.3 탭바 (Tab bar) — 멀티 문서 인터페이스
- **동작**: 툴바 바로 아래, 워크스페이스 위. **활성 문서와 열린 모든 문서**를 탭으로 표시. 열 수 있는 파일 수 **제한 없음**("컴퓨터 성능이 한계").
- **오버플로**: 열린 문서가 공간을 초과하면 **스크롤 버튼**이 나타나 전체 문서 탐색.
- **상태 표시**:
  - **수정된(dirty) 파일**: 탭 **오른쪽에 작은 원** 표시.
  - **잠긴(비밀번호 보호) 파일**: 탭 **왼쪽에 작은 자물쇠**. **자물쇠 클릭 시 비밀번호를 알면 잠금 해제** 가능.
- **호버**: 탭에 마우스를 올리면 해당 파일의 **Save / Close 버튼**이 나타난다.
- **컨텍스트 메뉴**(우클릭 Win / ⌃클릭 macOS): **Close all tabs**, **파일이 있는 폴더 열기**(탐색기/파인더에서 표시).
- **`+` 버튼**: 탭바 우측 끝. 클릭 시 **생성 또는 열기** 소메뉴 표시.
- **단축키**: 다음 탭 `Ctrl+Tab` / `⌃⇥`, 이전 탭 `Ctrl+Shift+Tab` / `⌃⇧⇤`, 닫기 `Ctrl+W`/`⌘W`, 모두 닫기 `Ctrl+Shift+W`/`⇧⌘W`.
- **웹앱 시사점**: 문서별 독립 상태 트리(악보 데이터+뷰 상태+undo 스택). dirty 플래그는 undo 스택 기준으로 계산. "폴더 열기"는 웹에서 다운로드/핸들 노출로 대체.

### 2.4 에디션 팔레트 (Edition palette)
- **동작**: 창 **좌측**. 표준 악보·타브·슬래시 표기로 악보를 쓰는 데 필요한 **주요 편집 도구와 모든 기호**를 모은 패널. 버튼 클릭 → 해당 요소/속성이 **현재 선택된 노트에 적용**. 추가 설정이 필요한 기호는 클릭 시 **다이얼로그 창**이 열림.
- **아이콘 규약**: 아이콘 **우하단에 작은 삼각형**이 있으면 **우클릭 또는 롱클릭으로 추가 설정** 접근 가능(예: n-tuplet 비율 선택, 트레몰로 피킹 음가, 비브라토 진폭).
- **활성 상태 표시**: 모드형 버튼(튜플렛 등)은 활성 시 **파란색**으로 표시.
- **팔레트 10개 그룹(순서대로)**:
  1. 현재 편집 보이스(voice selector)
  2. Multivoice(멀티보이스 토글)
  3. Design mode(디자인 모드 — 보이스 셀렉터 오른쪽 버튼, `Ctrl+Alt+D`/`⌃⌥D`)
  4. Lyrics(가사)
  5. Chords(코드)
  6. Bar symbols(마디 기호)
  7. Note symbols(노트 기호)
  8. Effect symbols(이펙트 기호)
  9. Notation symbols(표기 기호)
  10. Automation symbols(자동화 기호)
- **단축키**: 표시/숨김 `F2`.
- **웹앱 시사점**: 그룹 접이식 세로 팔레트. 버튼 = 커맨드 디스패치(선택 컨텍스트 인자). 삼각형 규약을 위한 long-press/contextmenu 공통 핸들러.

### 2.5 워크스페이스 (Workspace)
- **동작**: 창 **중앙**의 작업 영역 — 트랙을 쓰고 노트를 추가하며 악보를 편집하는 곳. 수평/수직 **스크롤바** 보유. 클릭 가능한 악보 요소는 **호버 시 파란색 하이라이트**(클릭 가능 확인), 클릭 시 대응 편집 다이얼로그가 열림(메뉴/팔레트와 동일 다이얼로그).
- **웹앱 시사점**: Canvas/SVG 악보 렌더러 + 히트테스트 레이어. 클릭 가능 요소는 렌더 시 바운딩 박스와 요소 타입을 등록해 두는 방식.

### 2.6 인스펙터 (Inspector)
- **동작**: 창 **우측**, **2부 구성**:
  - **Song inspector**: 곡 속성(Title, Artist, Subtitle, Album, Words, Music, Copyright, Transcriber, Notice, Instructions), 표기(notation) 옵션 설정, 사운드 **마스터링** 설정, **Concert tone 버튼**. 악보 위의 제목/아티스트 등을 직접 클릭해도 이 인스펙터가 열림.
  - **Track inspector**: 현재 트랙 속성(이름/짧은 이름/색상/아이콘), 표기·튜닝 옵션, Transposition tonality, 사운드/연주 해석(interpretation) 설정. 하단에 "Stringed" 옵션 등.
- **단축키**: Song Inspector `F5`, Track Inspector `F6`.
- **웹앱 시사점**: 우측 도킹 패널을 Song/Track 2개 아코디언(또는 탭)으로 구성. 악보 요소 클릭 → 인스펙터 해당 필드로 포커스 이동하는 딥링크 필요.

### 2.7 글로벌 뷰 (Global View, 하단 패널)
- **동작**: 사운드보드(믹서)와 결합되어 동작. ① 트랙별 오디오 컨트롤 + ② **재생 중인 악보의 축약(미니맵) 뷰** — 격자를 클릭해 아무 마디로나 즉시 이동. **멀티트랙 선택**도 가능(여러 마디 복사·붙여넣기에 유용).
- **구성 요소 13개(매뉴얼 도판 기준, 좌→우)**:
  1. Add track 버튼
  2. Track names(트랙 목록, 좌측 열 — 트랙 색상/짧은 이름 표시)
  3. 트랙 위/아래 이동 버튼(헤더)
  4. Multitrack 토글 아이콘(가시성 아이콘들 위)
  5. Mute(트랙별)
  6. Focus/Unfocus current track
  7. Solo(트랙별)
  8. Volumes(트랙별 볼륨 슬라이더)
  9. Volume automations(활성 시 슬라이더 옆 **"A" 배지**, 클릭 토글)
  10. Panoramics(트랙별 팬)
  11. Pan automations(팬 노브 옆 배지)
  12. Equalizers(트랙별 EQ 접근)
  13. Global view 격자(마디 그리드 오버뷰)
- **격자(우측부) 구성**: ① 마디 번호(헤더 행) ② 마디 셀 ③ 섹션 마커 ④ 커서(현재 위치 하이라이트).
- **인터랙션**:
  - **트랙 이름(좌측) 클릭** → 그 트랙의 현재 마디 첫 박에 편집 커서 배치(재생 중이면 재생 중인 박).
  - **격자(우측) 클릭** → **트랙과 마디를 동시에** 정확히 선택.
  - 섹션의 첫 마디 클릭 → 해당 섹션 시작점으로 커서 이동. **섹션 더블클릭 → 섹션 전체 선택**.
  - 트랙 우클릭 → 컨텍스트 메뉴(트랙 삭제, 이동, 섹션 추가/수정 등).
  - 선택 영역 가장자리의 **핸들 드래그**로 다중 선택 확장.
  - `Alt` + 마우스 휠 = **수평 스크롤**.
- **자동 재배치**: 현재 작업 트랙이 글로벌 뷰에 보이지 않을 때 악보를 편집하면 글로벌 뷰가 **편집 중인 트랙/마디 위치로 자동 스크롤**.
- **패널 자체**: 수직 리사이즈 가능, `F8` 토글.
- **웹앱 시사점**: 좌측 믹서 스트립(고정) + 우측 가상 스크롤 마디 그리드(트랙 수×마디 수가 커질 수 있어 캔버스/가상화 필수). 커서·섹션·선택은 오버레이 레이어.

### 2.8 메뉴바 (창 관리 컨텍스트)
- **최상위 메뉴 12개**: **File, Edit, Track, Bar, Note, Effects, Section, Tools, Sound, View, Window, Help** (Windows는 창 상단, macOS는 화면 상단; macOS에는 추가로 "Guitar Pro 8" 앱 메뉴가 있어 Preferences/Check for updates/About 포함. Windows에서는 Preferences가 File 메뉴, Check for updates/About이 Help 메뉴).
- 메뉴 항목에는 **연결된 단축키가 라벨 옆에 표시**된다.
- **웹앱 시사점**: 웹 메뉴바 컴포넌트(12개 메뉴)를 커맨드 레지스트리에서 자동 생성. Action List(명령 팔레트 `@`)가 같은 트리를 탐색하므로 메뉴 정의를 데이터로 유지.

## 3. 악보 내비게이션 (Navigate in a Score)

일반 원칙: **편집 커서 위치는 악보와 글로벌 뷰 사이에 항상 동기화**된다.

### 3.1 악보 클릭으로 커서 이동
- **동작**: 악보 클릭 → 편집 커서가 클릭 지점으로 이동. **클릭만으로는 노트가 입력되지 않음**(탐색 중 실수 편집 방지). **재생 중에도 동작**: 클릭하면 **재생을 멈추지 않고** 클릭 지점부터 재생을 재시작.
- **웹앱 시사점**: 재생 중 seek는 오디오 스케줄러 리셋 + 커서 리포지션으로 구현(트랜스포트 정지 없이).

### 3.2 키보드 커서 이동 (전체 표)
| 동작 | Windows | macOS |
|---|---|---|
| 다음/이전 박(beat) | `→` / `←` | `→` / `←` |
| 다음/이전 줄(line) | `↓` / `↑` | `↓` / `↑` |
| 다음 스태프 | `Tab` | `⇥` |
| 이전 스태프 | `Shift+Tab` | `⇤`/`⇧⇥` |
| 마디 시작으로 | `Home` | `↖` |
| 첫 마디로 | `Ctrl+Home` | `⌘↖` |
| 마지막 마디로(현재 줄 끝) | `End` | `↘` |
| 최종 마디로 | `Ctrl+End` | `⌘↘` |
| 다음/이전 트랙 | `Ctrl+Down` / `Ctrl+Up` | `⌘↓` / `⌘↑` |
| 다음/이전 섹션 | `Ctrl+Alt+Right` / `Ctrl+Alt+Left` | `⌥⌘→` / `⌥⌘←` |
| Go To(마디/섹션 이동 다이얼로그) | `Ctrl+G` | `⌘G` |
- 특이 동작: 악보 끝에서 `→`를 누르면 **새 마디가 생성**된다(마지막 마디 뒤에 마디를 추가하는 유일한 방법). 마디가 불완전한 상태에서 `→` 이동 시 직전 박과 같은 길이의 새 박 생성.
- **웹앱 시사점**: 커서 = `{trackId, barIndex, beatIndex, stringOrStaffLine, voice}` 모델. 키 핸들링은 포커스가 워크스페이스일 때만.

### 3.3 글로벌 뷰 클릭 내비게이션
- 2.7 참조(트랙명 클릭 / 격자 클릭 / 섹션 클릭·더블클릭).

### 3.4 트랜스포트 버튼 (툴바)
- **동작**: 좌→우 5버튼 — ① 첫 마디로 ② 이전 마디 ③ 재생/정지 토글 ④ 다음 마디 ⑤ 마지막 마디로.
- **관련 단축키**: 재생/일시정지 `Space`, 처음부터 재생 `Ctrl+Space`(Win)/`⇧Space`(mac), 되감기/빨리감기(마디) `Ctrl+←/→`(Win)/`⌘←/→`(mac), 한 박 스텝 `Alt+←/→`/`⌥←/→`.
- **웹앱 시사점**: 트랜스포트 상태 머신(정지/재생/루프) — 커서·글로벌 뷰·오디오 트랙 창에 브로드캐스트.

### 3.5 섹션 메뉴 내비게이션
- **동작**: 섹션 마커는 악보 구간 이동 수단. **Section 메뉴**가 악보의 전체 섹션 목록을 나열하며 선택 시 해당 위치로 점프. `Edit > Go to...`(`Ctrl+G`)로도 특정 섹션/마디로 이동. 명령 팔레트에서 `$` prefix = Go to section, `:` prefix = Go to bar.
- **웹앱 시사점**: 섹션 목록은 마스터 트랙의 파생 데이터 — 메뉴/글로벌 뷰/명령 팔레트가 동일 셀렉터를 공유.

### 3.6 악보 스크롤과 자동 추적(auto-follow)
- **편집 모드**: 커서를 움직이지 않고 스크롤 가능 — 스크롤바, **마우스 휠·트랙패드** 지원. 글로벌 뷰의 트랙/마디도 스크롤 가능.
- **재생 모드**: 커서가 자동 이동하며 뷰가 커서를 따라 스크롤. 재생 중 수동 스크롤 시 **자동 추적이 일시 중단**되고, **워크스페이스 좌측 상단에 나타나는 "Resume auto-scroll" 버튼**을 클릭하면 즉시 커서 위치로 복귀·재추적.
- **웹앱 시사점**: `autoFollow` 플래그 — 사용자 스크롤 이벤트에서 해제, 버튼 클릭에서 재설정. 스크롤 애니메이션은 rAF로 커서 위치 보간.

### 3.7 트랙 전환 수단 요약
- 글로벌 뷰 트랙 클릭 / **Track 메뉴** / **LCD 트랙명 클릭 팝업** / `Ctrl+↑·↓`. 글로벌 뷰가 숨겨져 있어도 나머지 수단으로 전환 가능해야 함.

## 4. 악보 표시 모드 (Score Display Modes)

- **UI 위치**: **View 메뉴** 또는 **툴바의 Display mode 아이콘**.
- **6개 모드**:

| 모드 | 동작 |
|---|---|
| **Vertical Page** | 페이지들을 수직 스크롤로 배열(인쇄용 페이지 뷰) |
| **Horizontal Page** | 페이지들을 수평 스크롤로 배열 |
| **Grid** | 수평 페이지 개수가 창 크기에 맞게 적응, 스크롤은 수직 |
| **Parchment** | 페이지 구분(페이지 브레이크) 제거 — 더 촘촘한 연속(두루마리) 뷰 |
| **Vertical screen** | 가용 창 공간 전체 사용, 마디 배치 자동 리플로우 |
| **Horizontal screen** | 가용 창 공간 전체 사용, **멀티트랙 뷰에서 유용** |

- 매뉴얼 팁: 인쇄(종이) 뷰 작업 시 **Page/Parchment 또는 Page/Grid** 모드 사용.
- PDF 익스포트는 **현재 View 모드(Page/Screen)를 반영**한다(익스포트 도메인과 접점).
- Page/Parchment 모드에서만 **줄당 마디 수 수동 제어(Bar > System layout, Force/Prevent break line)** 가능.
- 명령 팔레트 `view` prefix(Change view)로도 전환 가능.
- **웹앱 시사점**: 렌더러에 레이아웃 전략 패턴(paged vertical/horizontal/grid, continuous, reflow-vertical/horizontal) 도입. Screen 모드는 컨테이너 크기 의존 → ResizeObserver로 재레이아웃.

## 5. 줌 (Zoom)

- **UI 위치**: 툴바의 Zoom 컨트롤(그룹 3).
- **단축키**: Zoom In `Ctrl++`(또는 `Ctrl+Shift++`) / `⌘>`, Zoom Out `Ctrl+-` / `⌘<`. 명령 팔레트 `zoom` prefix.
- **문서 저장**: 줌 값은 **문서(파일)에 저장되는 표시 정보** 중 하나(스타일시트, 페이지 레이아웃, 표기, 재생 엔진과 함께). Preferences > General의 **Force options**로 문서 설정과 무관하게 **항상 특정 줌(예: 150%)을 강제** 가능.
- **오디오 트랙 창의 줌**은 별도(자체 줌 버튼/트랙패드/Perform tasks 메뉴) — 악보 줌과 독립.
- **웹앱 시사점**: 줌 = 렌더 스케일 팩터(문서별 상태) + 전역 강제 오버라이드 설정. 벡터 렌더링이므로 스케일 시 재래스터라이즈 필요(캔버스) 또는 CSS transform(SVG).

## 6. 멀티트랙 뷰 (Multitrack)

- **동작**: 워크스페이스에 **단일 트랙 / 모든 트랙 / 선택한 트랙 하위집합**을 표시. 멀티트랙 활성화 시 **모든 트랙**이 표시되고, 해제 시 단일 트랙 표시로 복귀.
- **UI 위치/인터랙션**:
  - `View > Multitrack` 메뉴 또는 **단축키 `F3`**.
  - 글로벌 뷰 좌측의 **멀티트랙 아이콘**(트랙별 가시성 아이콘들 위)으로도 토글.
  - 멀티트랙 ON 상태에서 글로벌 뷰의 **트랙별 가시성(눈) 아이콘**으로 개별 트랙 표시/숨김.
- **연관 동작**: 멀티트랙 모드의 복사/잘라내기는 마디 구조(박자/조표 등)를 포함(`Ctrl+Shift+C/X`). 디자인 모드의 줄바꿈은 활성 트랙 또는 멀티트랙 뷰에만 적용되며 **트랙별·멀티트랙별로 서로 다른 마디 배치** 가능. Horizontal screen 모드가 멀티트랙에 유용. 스타일시트의 트랙명 표시 설정은 싱글/멀티트랙 모두에 적용.
- **웹앱 시사점**: 뷰 상태 `{multitrack: bool, visibleTrackIds: Set}` + 트랙별 독립 레이아웃 캐시(싱글용/멀티용 별도 시스템 배치 저장).

## 7. 전체화면과 창 관리 (Full Screen / Window Management)

## 9. 명령 팔레트 (Command Palette)

### 9.1 열기와 배치
- **접근**: `Tools > Show Command Palette` 메뉴 또는 **`Ctrl+E`**(mac 표기 `⌘E`).
- **배치**: **악보 상단, 열린 파일 탭바 바로 아래**에 나타나는 검색 필드.
- **초기 상태**: 비어 있고 플레이스홀더 문구 표시 — *"Type ? to show the commands list; up or down to browse the command history"*.

### 9.2 상호작용 모델
- **입력 문법**:
  | 입력 | 동작 |
  |---|---|
  | 일반 텍스트 | 앱 내 모든 옵션을 이름 전체/일부로 검색(입력할수록 실시간 필터링) |
  | `?` | 사용 가능한 prefix/명령 목록 표시 |
  | `@` | **Action List** 모드(메뉴 트리 탐색·실행) |
  | `>` | **Expression Text** 모드(코드/다이내믹/조표/옥타브 기호/클레프/디렉션을 이름으로 직접 적용, 예: `>Cm`) |
  | `unset <Effect>` | 선택 영역에서 해당 이펙트의 모든 적용 제거(예: `unset Tie`) |
- **탐색/확정**: `↑`/`↓` = 제안 목록 탐색 + **명령 히스토리 브라우징**. 제안 목록은 **알파벳순**, **좌측 열 = prefix, 우측 열 = 설명**. 현재 선택 컨텍스트에서 사용 불가한 명령은 **회색 처리**(사용 가능은 흰색). **`Tab`/`Enter`/파란 제안 클릭** = 명령 완성(자동완성), **`Enter` 또는 최종 액션 클릭** = 명령 적용. 옵션 플레이스홀더(`bar-count`, `pattern` 등)는 **이탤릭**으로 표시. 추가 옵션 패널이 있으면 **탭 아이콘(↹)** 이 표시되고 `Tab`으로 다음 패널로 확장(반복 가능).
- **도움말 표시**: 현재 조립 중인 명령의 사용법이 **파란 바 아래**에 표시.
- **flow 모드**: 기본은 명령 적용 후 팔레트 자동 닫힘. `flow` prefix(Toggle flow mode)로 팔레트를 계속 열어둔 채 연속 편집.
- **로컬라이제이션**: prefix/명령은 **항상 영어로 표시**되지만, 내부 번역으로 **사용자 언어로 검색 가능**(예: 프랑스어 UI에서 "accord" 입력 → `chord` prefix 표시).
- **토글 재적용**: 이미 적용된 토글 효과(예: Palm Mute)를 다시 실행하면 **제거 여부 확인 프롬프트** 표시.

### 9.3 명령 유형
- **Quick commands(32개)**: prefix만으로 즉시 실행되는 토글/즉시 액션. 전체 목록: `anacrusis, crescendo, dead-note, dead-slapped, decrescendo, double-barline, double-simile, fade-in, fade-out, force-break-line, force-tuplet-bracket, free-time, hammer-on-pull-off, left-hand-tapping, legato, let-ring, multirest, multivoice, palm-mute, pedal, prevent-break-line, print, repeat-close, repeat-open, rest, show-string-number, simile, tapping, tie-beat, tie-note, timer, volume-sweel(원문 표기)`.
- **Advanced commands**: prefix + 옵션(들). 예: `add-bar 20` → "Add 20 bar(s)", `x`(repeat bars) + 횟수, `bend 2`, `clef`(현재 클레프 표시 + Tab으로 옵션 패널 진행), `4/4`(박자 직접 입력), `n:m`(커스텀 튜플렛). 내비게이션 관련: `$`(Go to section), `:`(Go to bar), `select-bars`, `select-section`, `view`(Change view), `zoom`(Zoom in/out), `voice`(활성 보이스 변경). 전체 목록은 약 68개(추출본 표 참조).
- **패턴 명령**: 문자 시퀀스를 선택 영역의 박에 순차 적용(입력 즉시 라이브 미리보기, 선택이 없으면 악보 끝까지 반복). 9종: `arpeggio, brush, golpe, left hand, picking, pickstroke, right hand, slap pop pattern, wah`. 예: `pickstroke dduddud`(d=다운, u=업, Space=빈 박).

### 9.4 하위 모드
- **Expression Text**: `Tools > Expression Text`, 단축키 `⇧⌘E`(Win 추정 `Shift+Ctrl+E`), 또는 팔레트에서 `>`. prefix 없이 항목 이름을 직접 입력 — 대상: **Chords, Dynamics, Key signatures, Octave signs, Clefs, Directions**.
- **Action List**: `Tools > Action List`, 단축키 `⌥⌘E`(Win 추정 `Alt+Ctrl+E`), 또는 팔레트에서 `@`. **앱의 모든 메뉴/서브메뉴를 탐색기처럼 탐색·실행**(12개 최상위 메뉴 전체). 부분 문자열 필터(예: "fretboard" 입력 → View > Show Fretboard View 액션 실행).
- **웹앱 시사점**: 단일 커맨드 레지스트리(메뉴·툴바·팔레트 공유) + 퍼지 검색 인덱스(다국어 별칭 포함) + 상태 머신(prefix → 옵션 패널 스택 → 적용). 패턴 명령은 keystroke마다 편집 트랜잭션 미리보기 → Enter 시 커밋, Esc 시 롤백.

## 10. 환경설정 (Preferences)

- **접근**: Windows `File > Preferences`, macOS `Guitar Pro 8 > Preferences`. **단축키 `Ctrl+,` / `⌘,`**.
- **구조**: **5개 탭 패널** — ① General ② Interface ③ Score errors ④ User information ⑤ Audio(/MIDI — 오디오 도메인에서 상세).
- **공통 동작**: 모든 탭의 설정은 **변경 즉시 적용 — OK/확인 버튼이 없다**(라이브 적용).
- **웹앱 시사점**: 설정 스토어(localStorage/서버 동기화)에 즉시 기록, 구독 컴포넌트 실시간 반영. 언어 변경만 예외적으로 재시작(웹에서는 리로드) 필요.

### 10.1 General 탭
| 설정 | 동작 / 수치·옵션 |
|---|---|
| **Default template** | 새 문서 기본값은 빈 문서(트랙 없음 → 트랙 생성 마법사 자동 오픈). 이 설정으로 새 문서를 **사전 구성 트랙이 담긴 템플릿**으로 생성(예: "Jazz Trio" = 재즈 기타 + 업라이트 베이스 + 드럼 3트랙) |
| **Force options** | 문서에 내장된 표시/재생 정보(**stylesheet, 페이지 레이아웃, zoom, notation, 재생 엔진**)를 문서 수정 없이 전역 강제 오버라이드. 예: 항상 **150% 줌** 강제, 항상 **MIDI 재생** 강제, **"Force stylesheet (when opening or creating a file)"** + 강제할 스타일 선택 |
| **Open last session documents** | 종료 시 열려 있던 탭을 다음 실행 시 자동 재오픈(토글) |
| **Embed audio files by default** | 오디오(백킹) 파일을 기본적으로 악보 파일에 임베드할지 여부(오디오 트랙 창의 Perform Tasks 메뉴에서도 접근 가능) |
| **Force playback with MIDI** | 모든 악보 열기/생성 시 전 트랙 MIDI 모드 강제 |
| **Updates 체크 주기** | 기본 **매주(weekly)**; 주기 변경 또는 **never** 설정 가능 |
| **Beta versions** | 베타 빌드 수신 옵트인 체크박스(해제 시 정식 채널 복귀) |

### 10.2 Interface 탭
| 그룹 | 설정 | 동작 / 옵션 |
|---|---|---|
| Score | **Playing cursor style** | 재생 커서 스타일 4종: **None**(표시 안 함) / **Smooth**(박 사이 연속 이동) / **Each note**(박마다 점프) / **Metronome**(메트로놈 틱에만 이동) |
| Score | **Highlight current played bar** | 재생 중인 마디를 **연노랑(pale yellow)** 으로 하이라이트(체크박스, 해제 가능) |
| 편집 | **Plus/minus keys behavior** | `+`/`-` 키의 음가 변경 방향 **반전** 옵션(기본: `+` = 더 긴 음가 방향) |
| 편집 | **Include chords and scale diagrams in copy paste** | 체크 시 단순 복사·붙여넣기에 코드/스케일 다이어그램 자동 포함; 해제 시 `Edit > Special Paste` 필요 |
| Toolbar | **Show Fretlight button** | 기본 숨김인 Fretlight 버튼을 툴바 우상단에 표시(툴바 구성 가변성) |
| Interface | **Language** | 다국어 지원. 변경은 즉시 적용되지 않고 **앱 재시작 필요** |
| Homepage | **mySongBook / Examples 항목 제거** | 홈페이지에서 해당 섹션 숨김 |

### 10.3 Score errors 탭
- **동작**: 작성 중 표기 오류를 **자동 감지**("워드프로세서의 맞춤법 검사처럼") — 감지된 오류는 악보에서 **빨간색**으로 표시. 대표 예: **불완전한 마디**(박자표 대비 길이 미달/초과; 단, 현재 편집 중인 마디와 Anacrusis 지정 첫/끝 마디는 제외). 이 탭에서 **오류 유형별 검사 활성/비활성** 토글을 구성.
- **웹앱 시사점**: 실시간 검증 엔진(마디 길이 등) + 렌더러의 오류 스타일 채널. 검사 항목별 boolean 설정 맵.

### 10.4 User information 탭
- **동작**: 새 파일 생성 시 곡 정보(작성자, 저작권 등)는 기본 공란 — 이 탭에 **기본 작성자/저작권 정보를 저장**해 두면 **모든 새 파일에 자동 프리필**된다.
- **웹앱 시사점**: 설정 필드를 새 문서 생성기의 Song information 초기값에 매핑.

### 10.5 Audio/MIDI 탭 (접점만)
- 오디오 장치/출력/입력(VU미터)/버퍼 크기, MIDI 4포트/레이턴시/캡처 감도 등 — **오디오 도메인 사양서 참조**. UI 관점: 5번째 탭으로 존재하며 동일한 라이브 적용 규칙을 따른다.

## 11. 다크 모드 (Dark Mode)

- **매뉴얼 근거 없음**: 334페이지 매뉴얼 추출 전체에서 다크 모드/테마 전환에 대한 언급이 발견되지 않았다(Preferences Interface 탭에도 테마 항목 없음). GP8 데스크톱 앱은 어두운 계열 고정 UI 테마 + 흰색 악보 용지 조합으로 알려져 있으나 매뉴얼은 이를 설정 항목으로 문서화하지 않음.
- **웹앱 권고**: 클론에서는 (a) 매뉴얼 충실 재현 시 테마 설정 없이 고정 다크 크롬 + 라이트 악보 캔버스, (b) 개선안으로 Preferences > Interface에 테마(Light/Dark/System) 항목 추가 — 단 이는 원본에 없는 확장임을 설계 문서에 명시.

4. **반응형 툴바 + 1024×768 최소 뷰포트** — 오버플로 시 버튼 숨김, 모든 기능은 메뉴/팔레트로 항상 도달 가능해야 한다는 원칙 유지.
